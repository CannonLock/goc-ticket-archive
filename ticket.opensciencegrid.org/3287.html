Hi,

As far as I can tell, MIT Tier2 OSG installation is restored.
I have recovered it from a backup done 12 hours ago. After that,
I went with "pacman -repair" over all packages with status "[X]",
then I ran the update I needed: for CA-Certificates, and finally
did a few more pacman -repairs. Now, as far as I can tell,
the installation is undamaged and functional (pacman -lc returns
[*] for all packages).

At this time, I am good except that on some packages I am not
up to date. I assume it is not worth the trouble fighting for
update of PPDG-Cert-Scripts (still fails, my trick with repairs
fails too), and vo-0.4.0 given that 0.6.0 upgrade is coming soon.

Let me mention once again: I strongly suspect that one
can not install OSG 0.4.0 from scratch at the moment due to
problems with the repository. Again, given the coming upgrade
to 0.6.0 I perhaps should not care, but it would be good if
after moving older versions out of the way someone checked
that installation of it is still possible, in the future.

Thanks,
Ilya
