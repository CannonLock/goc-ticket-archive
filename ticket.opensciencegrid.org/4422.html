    Deepak,

like ...
------------------------------------------------------------------------

127.0.0.1               localhost.localdomain localhost

------------------------------------------------------------------------
Also, please ensure that /etc/sysconfig/network has a HOSTNAME line that
contains your fully qualified hostname, viz:
------------------------------------------------------------------------

HOSTNAME=gratia01.fnal.gov

------------------------------------------------------------------------
rather than simply, "gratia01."

In order to have this take effect without a reboot, please execute (as
root):

------------------------------------------------------------------------

source /etc/sysconfig/network
hostname $HOSTNAME

------------------------------------------------------------------------

If this does not work, let me know as there is one other configuration
which *may* need to be changed.

    Hi Deepak,

Yes, you'll have to be part of **some VO** to run on the OSG.

to run on the OSG.

might also email the contact address on that document.
http://www.renci.org/~rynge/osg/Engage_New_User_Guide.pdf

There's also the OSG VO that you can consider joining if you're totally new to OSG,
and want to test the waters. More information here:
https://twiki.grid.iu.edu/twiki/bin/view/VO/OSG#Requesting_OSG_VO_Membership
