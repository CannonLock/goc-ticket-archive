    Nov 26, 2014 10:06 PM UTC by Marco Mambelli 
    I opened a Jira ticket but I'd prefer if this ticket is left open as well since some people working on this are not in Jira (e.g. Lisa).
https://jira.opensciencegrid.org/browse/GRATIA-162

Someone from OSG Software or Gratia development should be on the ticket. I assigned the ticket to Carl, maybe Edgar can take the ticket if he is following up with the developers.
If I understood correctly Lisa will start working on a modified version of the probe.
Please report progress or questions to this ticket.

To answer the questions:
1. If there is no format string in the log file the old format should be assumed
2. The old format should be the default (at least for now)

Ideally there should be also an option in the probe configuration (ProbeConfig) that allows to force one version of the log format, e.g.:
GEReportingFormat="auto"

With valid values "auto" (the probe detects the format form the file first line or uses the default old format), "old" (pre 8.2 format), "ge82" (new format since grid engine 8.2). old and ge82 would override defaults and the format recognized by reading the first line.

Thank you,
Marco
