    Oct 8, 2010 02:22 PM UTC by Thomas Lee 
    OK, the OSG CA package's "IGTF format" (my term) is really only an extension of the format required by Apache's SSLCACertificatePath directive, where each certificate is a separate file (or a symlink) with name <subject_hash>.<sequence_number>.  I think Apache may only need those files.  The additional .info, .signing_policy, .namespaces, .crl_url, etc. may not be necessary.  I hope they're not necessary, because they seem to be metadata not found in the certificates themselves, and the certificates are all I have.  If they're not necessary, it will be fairly easy to split up the ca-bundle.crt file into separate files.
    Oct 7, 2010 08:57 PM UTC by Thomas Lee 
    One of the stumbling blocks here is the fact that the certificate bundle that comes with RHEL is in one format (plain text), and the OSG CA package is in another (IGTF format).  There don't seem to be any ready-made tools to convert one to the other -- at least, none that I know of.
