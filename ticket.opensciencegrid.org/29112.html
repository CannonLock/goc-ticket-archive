    Hi Vlad,

GIP will complain if it can't advertise either a Storage Element or a GridFTP server. If you have a Storage Element, add an SE section to /etc/osg/config.d/30-gip.ini (look at the "SE CHANGEME" section in the comments for an example). If you have a GridFTP server, set "advertise_gsiftp" to True and "gsiftp_host" to the hostname of the GridFTP server.

If you don't have either, just set advertise_gsiftp to True and it will make the error go away. We'll probably remove those validation checks in a later release of GIP.

Once you've made the config changes, re-run "osg-configure -c" to apply the changes.

    Hi Vlad,

The priority needs to be set before any packages are installed. The
problem is, the EPEL repos have higher versions of the packages than
what OSG ships. If the two repos have equal priority, then yum will
prefer EPEL's packages because of the versions. If the packages from
EPEL are already installed, then even if you later prioritize the OSG
repos, a "yum update" won't change anything, because OSG's packages are
older.

Your options are:
- reinstall the entire machine, making sure the OSG repo has the correct
priorities before packages are installed
- "yum erase" the packages that you should have gotten from OSG, then
reinstall the CE packages

You can get a list of what packages are installed from a non-OSG repo
that should have been installed from an OSG repo by running
"osg-installed-versions" -- these packages will be under the header "NOT
FROM A RELEASE".

Let me know if you have any questions.

-Mat

    Apr 8, 2016 04:42 PM UTC by Matyas Selmeci 
    While I haven't pinpointed the exact cause of your error, I did notice that your osg repo (in /etc/yum.repos.d/osg.repo) does not have any priorities set. There are several RPMs that OSG ships that are also in EPEL, and you will get weird errors if you do not have the OSG version of those RPMs. Add the line "priority=98" to osg.repo.

Once you've done that, you'll need to replace the RPMs from EPEL with the same software from OSG. Considering how many you have, the easiest thing to do would be to reinstall the CE. If you can't do that, let me know and I'll come up with a recipe that will replace just those RPMs for you.
