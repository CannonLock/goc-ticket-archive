    OK--on my cluster I have the pbs logs in /var/spool/pbs
that is set in $VDT_LOCATION/osg/etc/config.ini,
in the PBS section
pbs_location = /var/spool/pbs

and then configure-osg takes care of making the correct
configuration file, which turns out to be
not ProbeConfig as I said before, but urCollector.conf.

pbsAcctLogDir = "/var/spool/pbs/server_priv/accounting"

I've never had to set this setting manually, configure-osg
took care of it.

So I think if you just set pbs_location in your config.ini
to be /var/torque
and run configure-osg -d -c
again, it should make the configuration change and it should just work.

    Nov 22, 2011 02:29 AM UTC 
    You need to nfs-export the following 2 directories from your PBS master to your CE:

/var/spool/pbs/server_priv/accounting
/var/spool/pbs/server_logs

Gratia needs in particular the first one of those.

If PBS is installed in your PBS master in some different directory
than /var/spool/pbs, then use that path in place of /var/spool/pbs

Gratia depends on reading the pbs server_logs directory to determine when
the job is complete and how much cpu the job used.  without that, you can't
make Gratia work for PBS.

You can check with the caltech cms tier2 which must be doing the same thing
because their gratia probe is working.
