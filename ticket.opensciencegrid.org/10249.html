    Firefox beta 4 shuts off ssl renegotiation by default, but
this doesn't work for the DOEGrids CA key generating, 2048 bit website
(or probably many others).  We are still investigating, but
we currently do not know that there is anything that can
be done about it on the server side (see 2nd link below).
(It is more likely that the next revision of DOEGrids CA
can deal with this.)

You can probably change the way the browser acts.

Read about it here
http://dotomaz.tumblr.com/post/786443743/firefox-4-0b1-and-ssl-renegotiation
https://wiki.mozilla.org/Security:;Renegotiation

Please refer problems like this to ESnet TTS # PKI19204.

Thanks, ==mwh
