    Jan 24, 2012 10:55 PM UTC by OSG-GOC 
    Greetings,

There seems to be two separate issues here, which I'm only just realizing.
There are errors of this kind

[SE][srmRm][] httpg://atlas.bu.edu:8443/srm/v2/server: CGSI-gSOAP running on
atlddm17.cern.ch reports Error reading token data header: Connection closed

which we've seen happen when our gatekeeper temporarily gets under high load.
These are brief episodes which seem to happen roughly once every few days. This
might be solved by upgrading our gatekeeper and/or moving where our Adler
checksum spreader is running.

Issue #2 are errors like this:

[SE][srmRm][] httpg://atlas.bu.edu:8443/srm/v2/server:
org.apache.axis.types.URI$MalformedURIException: Query string contains invalid
character:

which I hadn't notice before. This seems like a real protocol problem which
might be solved by upgrading our SRM software.

Our new gatekeeper has just arrived so we should be able to solve this soon
with a combination of gatekeeper and SRM upgrade. We've been told that this
isn't super-urgent as the deletions are retried when they fail and succeed
later. We'll keep this ticket open until we can get a fix in place.

- Saul & John
