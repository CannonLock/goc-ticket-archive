    Feb 13, 2012 01:27 AM UTC by OSG-GOC 
    [by Steve Timm]

There are new records coming from OU_OSCER_ATLAS, again reporting the
"ou2" VO.  From the JobUsageRecord information I can see
1) the "ReportableVOName" is DOSAR this time (previous reports had
NULL.
2) the LocalUserId is ou2012.  That means that your site has the same
bug that many others do--namely that it is lopping off the last three
digits of the user id and calling that the VO.

this was recently patched for PBS probe but not for the LSF probe.
I'll file a ticket in the OSG JIRA to make sure the patch gets
out for LSF as well and let you know when it is there.

In the meantime should we map this user to DOSAR VO, or
is it really non-osg usage that should be deleted?  IF the
latter you might want to use the option

SuppressNoDNRecords="1"

in your $VDT_LOCATION/gratia/probe/lsf/ProbeConfig

That will keep records like this from being sent.

Thanks

Steve Timm
for Gratia operations
