This is bizarre. Since the classdef *does* exist on your system, I would
have to venture a guess that tomcat is confused about whether a servlet
thread should be operating or not. This seems to be supported by the
message:

2008-12-12 22:18:59,532
org.apache.catalina.loader.WebappClassLoader(http-8880-Processor84)
[INFO]: Illegal access: this web application instance has been stopped
already.  Could not load
net.sf.gratia.util.TidiedDailyRollingFileAppender$DatedFileFilter.  The
eventual following stack trace is caused by an error thrown for
debugging purposes as well as to attempt to terminate the thread which
caused the illegal access, and has no functional impact.

I would suggest bouncing the collector *gently* with:

service tomcat-v55 restart 30

.. and see if you get the trouble again.

The collector did *not* receive the record -- it should have been saved
in gratia/var/tmp/gratiafiles on the sending CE. If not, that's a
probe-side problem we need to address.

Let me know what happens on the restart.

Thanks,

    Chris.
