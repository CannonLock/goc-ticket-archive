    Jul 30, 2010 01:21 AM UTC by Mariachi 
    Hi all,

I responded to a similar question on another ticket. Here's a copy:

--------------------

When upgrading GUMS using VDT, I believe you can specify the
OLD_VDT_LOCATION and the VDT install will automatically migrate your old
database contents (or retain the old DB and update the schema). The VDT
team should be able to clarify that this works, and how exactly to do it.

Another possibility, depending on how old the existing install is, is to:

1) dump the old db: . <old-vdt>/setup.sh ; mysqldump GUMS_1_3 > gums.sql
; exit

2) do a fresh VDT/GUMS install.

3) load the old db: . <new-vdt>/setup.sh ; mysql < gums.sql

There may be some additional command switches necessary, depending on
whether there is a root password set in MySQL. But you get the idea.

This works because our general approach is that database schema changes
only happen on major version number upgrades.

-----------------------------------

Cheers,

--john
