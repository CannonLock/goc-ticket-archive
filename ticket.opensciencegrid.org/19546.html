Hi all,

this is just a heads up that there's a possibility that the latest globus
updates we did from osg-testing at OU_OSCER_ATLAS may have ill effects
on the way that Condor-G (on a remote submit side) communicates with
the LSF jobmanager.

It's not clear to me whether there's a definite correlation, but we did
this update on January 31, and for the last week or so, the BNL ATLAS
pilot factory keeps losing track of the pilots that are running on
that site -- or rather, it thinks they're still submitted when they
have long since run and exited -- and that causes the site to drain,
since no jobs can start when there are no pilots.

This has happened occasionally in the past, but in the last few days,
the problem seems to have acceelerated dramatically, since we lose
all our jobs several times a day it seem like.

So, I'm not sure if you want to hold off on releasing these changes
tomorrow, I don't fully understand what's going on.

See below for all the RPMs we updated; maybe you can correlate on
what might be going on, I can't really ... :)

Thanks,
