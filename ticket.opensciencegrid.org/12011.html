    May 16, 2012 04:16 PM UTC by Matyas Selmeci 
    Yep, there's a line in the stop() function of the init script
$SU - $TOMCAT_USER -c "$TOMCAT_SCRIPT stop" >> $TOMCAT_LOG 2>&1
that you could change to
$SU $TOMCAT_USER -c "$TOMCAT_SCRIPT stop" >> $TOMCAT_LOG 2>&1

However, changing the init script isn't really the proper long-term solution, since the changes will not be preserved across upgrades.
What you should do is:
1. Make a directory for tomcat at /var/home/tomcat
2. Make sure tomcat owns and can write to that directory
3. Edit /etc/passwd and change the home directory for the tomcat user to /var/home/tomcat
4. "su - tomcat" and have the script that runs make the ssh keys for tomcat so it won't prompt next time (log out and su - again to make sure)
5. Do "yum reinstall tomcat5" to remove all the debugging changes that we added to the tomcat scripts
