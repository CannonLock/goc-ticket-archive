    Feb 10, 2014 03:59 PM UTC by Thomas Lee 
    I found out why the machine didn't start on boot: RHEL6 uses Upstart, not regular old initd.  The other two machines that are backed up with TSM (those being nukufetau/backup.grid and osg-xsede) are RHEL5, which still uses initd.  The RHEL5/initd method for starting TSM involves putting a line in /etc/initd, but Upstart ignores this line.  So TSM must be started either manually (which is what I did after installing the OS in December but forgot about this in my flurry of activity, but not after updating and rebooting dubois in January), or via an Upstart script.  I've written an Upstart script for this (called dsmcad.conf) and put it in /etc/init.  I've verified that dubois now starts TSM on boot, and I notified Keith about all of this, so backups are re-enabled.  I've verified that backups have been happening.
