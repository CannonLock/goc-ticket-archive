    Oct 2, 2017 08:38 PM UTC by OSG-GOC 
    Greetings,

I think there may be an issue with the gfal tools not requiring DNS reverse lookup verification when connecting to a resource. The issue was discovered while trying to debug an issue at the University of Maryland. bestman transfers starting failing on 9/20/2017. In testing the site, we discovered that their gridftp server (hepcms-gridftp.umd.edu) was misconfigured and did have the correct DNS entries for reverse lookup. While uberftp refused to connect to the gridftp server, gfal-copy and gfal-ls did connect. Bellow is some output from the tests. You  may not be able to verify this issue by using the same host as Maryland is in the process of correcting the DNS entries.

Thank you,
doug
