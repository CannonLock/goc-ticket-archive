    Hello.

Did you run the program to clean the SE/LFC? The new dump is already delivered
to your site ANL. (DSN=user10.HironoriIto.T3Dump.ANL.20101007_09_36_08) . For
time being, there is no automatic method for T3 admin to trigger the creation
of dump to avoid the overloading the LFC. Without some protections, repeated
requests from a single or multiple T3s within the short period can possibly
overload LFC database, making T3 LFC, which is a shared resource, unusable.
Therefore, until such a protection is in place, if T3s need a dump more often
the periodic delivery, just ask for the creation.
