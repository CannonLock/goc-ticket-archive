Hi John,
GridCat always use combination of mds-vo-name=local and port=2135.
So, in your SE input file to add SPRACE SE, you need to
use :
spdc00.if.usp.br giis spgrid.if.usp.br 2135 "ldapsearch" "-x -l 30 -b mds-vo-name=local,o=grid -h \
$giis_hostname -p \$giis_port"

Also you could use the way you did :
spdc00.if.usp.br giis spgrid.if.usp.br 2170 "ldapsearch" "-x -l 30 -b mds-vo-name=SPRACE,o=grid -
h \$giis_hostname -p \$giis_port"

Either way, you should get it.
Could you manually check :
ldapsearch -x -l 30 -b mds-vo-name=local,o=grid -h spgrid.if.usp.br -p 2135 '(objectclass=*)'
and
ldapsearch -x -l 30 -b mds-vo-name=SPRACE,o=grid -h spgrid.if.usp.br -p 2170 '(objectclass=*)'

?
If there is a timeout, it really means timeout.
I checked these from my lapton and I can see all the information from SPRACE.
