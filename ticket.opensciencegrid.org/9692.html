    Elizabeth,

I think I have 2 questions related to LLNL (now LC-glcc) registration both related to the same objective.

1. What additional steps need be taken so this new resource group is visible in MyOsg?

2. This one is likely a MyOSG/OIM question in terms of visibility and appears to be affecting the WLCG data transfer from the APEL database to the EGI accounting portal for those resource groups we send accounting data from Gratia.   We are currently sending data now for LC-glcc to APEL.

If I go to view the data for resource group LC-glcc here (http://tinyurl.com/29og3kh), I see nothing for the resource group.  Note: by default, the resource "Active Status" filter is set to "active".

If I remove the resource criteria for "Active Status", I then see that the resource group LC-glcc is registered (http://tinyurl.com/25smgc6) and the xml generated contains the resource group and shows it as being enabled.

I am guessing that the APEL accounting software that queries MyOSG (using the xml) for registered OSG resource groups has the filter for resource with "Active Status" set and therefore never sees the LC-glcc resource group in the xml and then never transfers LC-glcc data to the EGI accounting portal.

In MyOSG, I would think that when I do a default query on resource groups that I would at least see that the resource group does exist showing its status ('enabled' in this case) and also indicating no 'active' resources.

Based on how this is currently working, we need to get the LC-glcc resource set to active so we can start getting the LC-glcc data transferred to the EGI accounting portal so that when they register with LCG accounting their data will be available for MOU reporting.

So, back to the 1st question, what needs to be done further to get the resource set to 'active'.

Thanks
