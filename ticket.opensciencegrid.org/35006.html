    Sep 26, 2017 09:06 PM UTC by OSG-GOC 
    At FNAL we are currently setting up our own GRACC instance, for some internal accounting. Since we'll have our dCache probe (and possibly others) report to our GRACC, we'll need to forward records to the OSG GRACC. The plan when GRACC was designed was to do this by directly sending records to the RabbitMQ exchange. As such, we'll need an account on the ITB and Production RabbitMQ servers that can access the "gracc" vhost and send messages to the following exchanges:

gracc.osg.raw
gracc.osg-transfer.raw
gracc.osg-itb.raw
(or, generally, gracc.osg*.raw)

There may be some other permissions required, depending on how we implement the forwarding (e.g. if we use the gracc-request agent it will require some other ephemeral exchanges and queues). Let's start with the minimal permissions on ITB and tweak it from there.

Thanks,
Kevin
