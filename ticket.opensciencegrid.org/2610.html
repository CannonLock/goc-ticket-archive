    Oct 31, 2006 09:11 PM UTC by VDT 
    I've talked to Wayne and Leigh about this outside of this ticket,
though there has been no resolution yet.

The problem: You are not allowed to chown a file in AFS. The VDT
installation does a bunch of chowns as part of the installation. I
think that most of the chowns are in the VDT code, but Wayne noticed
some in Pacman too.

Wayne, my understanding is that you are installing the worker package
on AFS for use by many worker nodes. Is that correct?

I think that in the short-term, there is no way to eliminate the use
of chown when installing as root.

Given that you are installing the worker-node client on one computer,
for use by many computers, I think that you do not need to install as
root, but you can install as another user. Is that correct? If so, I
recommend that you do so, and you should avoid all chowns(), and the
installation shoudl succeed.

Is there any reason that this wouldn't be acceptable to you? If so,
let's talk further, perhaps by phone, so we can resolve your problems.

-alain
