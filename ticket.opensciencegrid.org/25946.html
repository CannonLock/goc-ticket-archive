    We think the problem has been located. This particular DaTRI request is being
mapped as user 'usatlas3' instead of 'usatlas1'. This shouldn't have mattered
accept for the fact that the /pnfs/aglt2.org/atlaslocalgroupdisk/rucio
directory (and sub-directories) was missing the group write protection. These
directories are being updated ('chmod -R g+w
/pnfs/aglt2.org/atlaslocalgroupdisk/rucio') and transfers for this particular
task are now succeeding. This was the first DaTRI request we have received
destined for AGLT2_LOCALGROUPDISK that wasn't mapped to 'usatlas1'.

If transfers complete soon we should be able to close this ticket.
