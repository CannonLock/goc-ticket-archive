    Aug 15, 2006 05:47 PM UTC by Footprints Remedy 
    Short Description=OSG 0.5.1 elimination MIS-CI

Status=Service Desk

During the pre-release phase of OSG 0.5.1 an attempt was made to eliminate
the use of MIS-CI in order to reduce the number of systems advertising grid
resources.

So MIS-CI was removed from the OSG 0.5.0-pre-release cache and we
determined that 2 systems were affected:
1. GridCat
2. GIP

Both systems go after the $VDT_LOCATION/MIS-CI/etc/grid-site-state-info to
get the site active bit settings.  The setting of active/inactive for a
site requires manually editting the file.

The grid-site-state-info file could be moved to the
$VDT_LOCATION/monitoring directory and GridCat and GIP modified to look
there instead.

Changes to both of these systems could be made and the MIS-CI package
eliminated.

Removing this file from the domain of MIS-CI, would also facillitate
modifications that are OSG related such as an inactive period (e.g. period
during	which a site is planned to be out of service.).  This is something
I thought I heard mentioned at the last ITB call (8/10 maybe).

As best I can tell, these are the only dependencies within the OSG cache of
software for MIS-CI.  This also eliminates a potential problem of VDT
distributed software being dependent on OSG distributed software. GIP is
distributed from VDT and MIS-CI from OSG and GIP is dependent on MIS-CI.  I
had earlier issued a VDT ticket 2001
(http://vdt.cs.wisc.edu/rt/Ticket/Display.html?user=guest&pass=guest&id=200

    1)  as a placeholder for this issue.  This issue would go away with the
elimination of MIS-CI.
