    Sep 10, 2015 03:47 PM UTC 
    Hi Vikas, Kevin,

I think I can clear some things up.

1) Bestman2 is a stateless SRM implementation; that means it delegates all actions to the underlying filesystem.  For example, when you do a "srm-mkdir", it's going to eventually fork/exec /bin/mkdir.
2) Hence, when you enable checksums, it's going to recompute the checksum from scratch each time (as no state = no cached checksum).
- Note that if you don't provide an executable, it has a built-in checksum calculator for md5/adler32/crc32.  So, you may not always see a fork'd process.
3) When you enabled checksums (showChecksumWhenListingFile=true), bestman2 started calculating checksums for each srmLs.
- This made FTS happy, but the length of time it took to list a single file went way up and DIRAC started seeing timeouts.
4) When you disabled checksums (showChecksumWhenListingFile=false), FTS will fallback to asking the corresponding *gridftp* server for the checksum.
5) Globus GridFTP only supports md5; in fact, it will return the md5sum for any checksum type requested (instead of throwing an error).
- This is probably what triggered the original issue (and maybe why Carl couldn't find the culprit in the bestman2 code): it wasn't actually bestman2 returning the md5sum.

So, that's where we are.

Now, how do we move forward?  There's a few options:

1) Some OSG sites have written their own script that wraps the adler32 command and caches the checksum (typically in an extended attribute if the filesystem supports that or in a different directory).
- This means that the checksum will be instantly available for DIRAC -- no timeouts!
- I'm sure Carl can help find one of these sample scripts; I think it'd be a reasonable change for us to package it in future releases, especially if extattrs are an option.
2) HDFS sites run a gridftp plugin which correctly implements the checksums.
- I would be interested in having OSG work with the Globus team to port the GridFTP plugin to the base Globus GridFTP server.  It's been a big irritation for me that they have this broken out-of-the-box.  This would take a week or so, of course, before we could put a patch in your hand.
