    So I see two problems here: one is that the 32-bit Python libraries did not make it into our 64-bit repositories, even though they should have been.
That's been fixed in our master repository now, but it might take a while for the fix to propagate to all of our mirrors.
You might want to edit /etc/yum.repos.d/osg.repo, comment out the first mirrorlist line and uncomment the first baseurl line so the [osg] entry would look like this:

[osg]
name=OSG Software for Enterprise Linux 5 - $basearch
baseurl=http://repo.grid.iu.edu/osg-release/$basearch
#mirrorlist=http://repo.grid.iu.edu/mirror/osg-release/$basearch
failovermethod=priority
priority=98
enabled=1
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-OSG
consider_as_osg=yes

which will let you get the update quicker. You may also wish to clear the yum cache by doing: "yum clean all; yum clean expire-cache"

The second problem is the other dependencies you are missing, such as some of the globus libs and libvomsapi.
Those are very definitely in our osg repositories, yet from your yum install command it looks like yum is only searching in the "Rocks 5.3" repository for them.
Can you make sure that you have the yum-priorities plugin installed? ("rpm -q yum-priorities")
