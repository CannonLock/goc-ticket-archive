    Sep 11, 2015 01:40 AM UTC 
    Hi,

Just FYI - this is almost definitely a HTCondor configuration problem.

If the worker node has a UID_DOMAIN different from the schedd, HTCondor will launch the job under the 'nobody' account (a few variations of this depending on worker node configuration).  Since the environment variables are set by the CE, it's not possible for it to know a priori what the eventual worker node will do.

Jose - I would definitely recommend keeping the tweaks in the ATLAS pilot though.  I don't know if this particular configuration at Caltech is on purpose, but I know UCSD uses different usernames between the CE and worker node on purpose.  So, unless UCSD tweaks the environment variables, it's quite possible you'll see this one again.

Brian
