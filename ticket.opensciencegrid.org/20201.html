    Hi,

I have an issue I need to deal with today, so just couple answers and more later today.
I'll tidy up 30-gip.ini - I just wanted to see some data flowing through.
The rpms that are showing are only for the base image. The image creation process is quite involved,
but in short, we create a base image, then we add to it with yum install as documented
on OSG pages whatever to create a CE, or SE or rsv monitoring or other services
we support on the cluster. We then create a diff between the base and the full image
and store it separately. During boot we first deploy the image and then unroll add-on on top.
We have not worked out yet how to list rpms for add-ons to make it completely transparent,
there is a seperate file with a list.
That list is created when the add-on is assembled and packed with the add-on.
I will upload it a little bit later today.

I do not do upgrades for that reason. Each time I want to deploy a new version I do a fresh build
of an image and the add-ons. One cannot modify without much hacking once the image is built and we don't.
Images together with all the addons are under version control.
One could argue with this approach, but it is working really well at the center for lots of aspects of what we do.
You can easily tell what software was installed on a node from the day it was booted to the day it was shut
down and you can pull out that image from the repository and reboot a server to it if only the hardware still
exists.
Oh, we use xcat  to manage our clusters and image building and packing etc is done within this framework.

I appreciate your help,

Iwona
