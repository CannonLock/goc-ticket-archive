    Nov 29, 2016 08:15 PM UTC by OSG-GOC 
    Hi Vince, all,

We've identified the source of these errors we were seeing. One of the data servers is getting very poor performance on one of two iSCSI links, and this causes serious back-ups / slowdowns in the SRM service. The result is the "Communication error on send...Connection closed" types of errors. A while back we set the problematic storage server to read-only, and since that time the rate of file transfer / deletion errors has improved considerably.
We're still debugging the iSCSI issue, but given the improvement in overall system performance it should be o.k. to close this ticket. Since this ticket is now >one month old, if we see a recurrence of the storage system errors a new ticket can be opened as needed at that time.

Regards,
-- Mark Sosebee
