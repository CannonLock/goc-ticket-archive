New Work Log Entry : The following was e-mailed to the Requester:

weigand@.... sent this Notes To Requester:
I don't know what release(s) are affected here, but in the 1.8+
version of VDT in ITB 0.7.0, you should be able to just run the
configure_gums script and setting the VDT_GUMS_HOST variable

source $VDT_LOCATION/setup.sh
export VDT_GUMS_HOST=cmssrv08.fnal.gov
$VDT_LOCATION/vdt/setup/configure_gums -client

which is the way it was supposed to be done in the first place.

There is another issue.  It appears the gums-host-cron cron service is
not enabled in configure_gums script even with the -client option.
This appears to be a bug.  The gums-host-cron service needs to be
enabled, then turned on if it is not already.

source $VDT_LOCATION/setup.sh
$VDT_LOCATION/vdt/sbin/vdt-register-service --name gums-host-cron
--type cron --disable
vdt-control --on gums-host-cron
