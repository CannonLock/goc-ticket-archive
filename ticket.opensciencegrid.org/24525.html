    I can verify that we had two nodes with cvmfs issues:

c0521.crane.hcc.unl.edu
c0801.crane.hcc.unl.edu

In both cases it seems sss didn't come up with our LDAP cvmfs user/group info and puppet "ensure => present" added the user/group to the local account databases of these nodes.  This caused permission issues with the cvmfs cache directory since it had files owned by the LDAP account.  I've removed a local tweak that I believe is responsible for sss not returning the LDAP info in time.

All 458 crane nodes are looking good presently:
