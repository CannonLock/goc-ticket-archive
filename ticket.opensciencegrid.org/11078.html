    Oct 3, 2011 04:22 PM UTC by OSG-GOC 
    It's mostlikely the client issue in srmcp.
What I see is that the srmcp uses SRM v1 protocol rather than v2 protocol, and the error messages are about that.  "faultString: No such operation 'advisoryDelete'".

Either the version that you have in srmcp does not support v2 protocol or somehow the RSV probe path to srmcp got messed up pulling the old version.

Recommendation: check the srmcp version that your RSV probe uses and test the command manually:

srmcp -srm_protocol_version=2  \
file:////sharesoft/osg/ce/osg-rsv/bin/probes/storage-probe-test-file \
srm://cms-se.hep.uprm.edu:8443/srm/v2/server?SFN=/mnt/hadoop/tmphdfs/test4

If the command loops for checking status again and again, it's not doing the right thing. bestman server returns the transfer information right away, so there should not be such waiting loop.

Another recommendation: OSG is transitioning all bestman servers to the latest bestman2. It should be upgraded to the latest bestman2-2.1.x for continuing support.
Also, it seems that RSV probe is out-dated too. We recommend an upgrade on RSV probe as well.

-- Alex
