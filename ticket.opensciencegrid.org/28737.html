    Dear all,

Dennis is fortunately back and we met and can give you the following information:

1. How many cores per node do your resources behind this gatekeeper have?

2x8-core machines

2. What is the available memory per core limit on these resources?

4 GB / core

3. What is the wall clock limit imposed on jobs in your local batch system?

at the moment, there is only a hard limit of 12 days; however, we asked the users to stay below 48 hours if possible.

4. What ARC-CE/nordugird RSLs do you expect us to explicitly specify at the factory when glideins are submitted?

not sure what format you need exactly. The information we can give you is:

CE endpoint: 	arc-ce.pleiades.uni-wuppertal.de:2811
queue: fugg.q

so probably "arc-ce.pleiades.uni-wuppertal.de:2811/nordugrid-SGE-fugg.q" is what you need.

We use VOMS authentication, so every user which is in the VOMS server should be authorized to run:

[vo]
id="icecube"
vo="icecube"
source="vomss://grid-voms.desy.de:8443/voms/icecube"
