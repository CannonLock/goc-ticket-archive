    Hi,

when querying for GratiaWeb service status when the service is actually status, the exit code returned by the /etc/init.d/GratiaWeb script is not following the standard (http://fedoraproject.org/wiki/Packaging:;SysVInitScript#Exit_Codes_for_the_Status_Action). The exact behaviour that's causing me some trouble is that when the service is stopped, a service status returns 0:

[root@gr10x0 ~]# /etc/init.d/GratiaWeb status; echo $?
gratia_web is stopped
0

The way to reproduce this is to stop the GratiaWeb service (service GratiaWeb stop) and then query GratiaWeb status (service GratiaWeb status).

I'm deploying and maintaining several instances of the GratiaWeb software using puppet and having this bug fixed would be really helpful; now puppet can not properly detect that the service is stopped, so it never gets started.

Thanks,
Gerard
