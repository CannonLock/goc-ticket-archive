Please be aware that the Globus installation in $OSG_LOCATION/globus
(currently /opt/osg-0.4.1/globus) does not work on WNs because of the
missing certificate dir.

Anyway that installation is in OSG_LOCATION and does not need to work on
worker nodes because it
is the server installation (client programs MUST NOT use/rely on it, it
may chanche to suite system needs).
According to OSG documents (e.g.
http://osg.ivdgl.org/twiki/bin/view/ReleaseDocumentation/LocalStorageUse
)
clients should use the client installation in OSG_GRID. There was already
a client installation visible on all WNs (/share/app/wn_client) but the
information in osg-attributes.conf was wrong, pointing to the wrong
place and has been updated.
Now the correct path is also in GridCat.
The ticket is resolved.
