    Apr 2, 2015 12:45 PM UTC by OSG-GOC 
    Hi Igor,

the ticket was already updated with the permanent solution; FNAL's SNOW and GGUS are in sync.

I'll paste a few more details on the root cause of the issue:

1. The /pnfs was mounted without noac on cmssrv228 (cmsphedex-tape replacement). The mountpoint used to be mounted with the 'noac' option but on Mon Feb 9 it was removed as part of some recommended performance improvement tweaks.

2. FileDownloadVerify script relies on "ls" to see if the file is there. Due to caching it did not see the file in time and PhEDEx considered the transfer failed and issued deletion. The problem repeats itself in retries.

3. After some testing, on 2015-04-01, the 'noac' option was reintroduced in the Phedex servers.

Double checking the status I see that the Phedex server 'cmsphedex-disk.fnal.gov' does not have the 'noac' option in the pnfs mounts yet, I'll follow up on that.

Gerard

    Mar 30, 2015 11:46 PM UTC by OSG-GOC 
    As one can see from the log in the previous post, the following happens:

1) FileDownloadVerify-chimera  called before the transfer ( argument "pre")  finds that file does not exist
2) FileDownloadDelete called before the transfer (argument "pre") does nothing
3) FileDownloadVerify-chimera  executed after the successful transfer ( fts status "0" passed  as an argument ) returns 1. i.e.  file does not exist . The check is done using "ls" on the NFS mounted namspace.
4) FileDownloadDelete "post" is executed after verify step fails.  We run srmls inside FileDownloadDelete to check the current status,  and as you see the file is there with the correct size.

In the last three days this  particular file has been deleted/re-transferred/deleted 23 times.

We did not have any changes in the plugins since 2013, except for location of the logs.
However there was recent change in dcache nfs mount options deployed during our large shutdown on Feb 11th.
Particularly "noac" option used to suppress attributes and data caching has been removed [*].

We will check with dCache developers (added Dmitry to the watch list ) if this could contribute into the problem.

- Natalia.

- added "rsize=65536,wsize=65536"
