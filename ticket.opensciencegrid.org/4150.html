Hi,

I am working on squid validation for osg-0.7.0. I ran configure-osg.sh
to provide the  squid information, but configure_squid failed. We have
the squid  installation separately on another installation directory

installation  directory?

This is one idea that the configure-osg.sh script hadn't considered.
If the squid hostname entered is on the same machine name
as your gatekeeper host, then it assumes that it is within
the same VDT installation too and tries to configure it.

If a different hostname is specified, it doesn't try to configure
the squid.  This should be fed back to the maintainers of the
configure-osg.sh script who I believe are at the GOC.

Steve
