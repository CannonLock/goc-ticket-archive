    Thanks Kevin,

DocDB does not use the osg-ca-cert rpm.
It was decided in the past that DocDB should not
accept all the certs in that bundle. The certs DocDB
accepts are: KCA, Digicert, CERN and CILogon.

So auto-installing the RPMs will not take care of it.
It sounds like I should either:

1) Check the osg-ca-cert release notes once a month
to see if any of the needed certs have changed.
(What is the URL for that and/or is there an e-mail
notification for that?)

2) Contact the vendors (Digicert, CILogon) to see if
they can provide e-mail notification. I'm assuming
the Fermilab security folks will announce it when
they make a change for KCA certs.

Mine advised against #2 because she said you folks
do some checking before adding the new CA. So I'm
leaning towards #1 unless you have any better ideas.

    Jan 14, 2015 07:42 PM UTC by Kevin Hill 
    Hi Laura,
There isn't currently a mail list for CA cert updates. The OSG CA cert bundles are integrated with the rest of the OSG software releases, so generally there will only be at most monthly updates. They will be included in the OSG software release notes. The easiest way would be to have some sort of automated yum update install the osg-ca-cert rpm from the OSG repo and keep it up to date. If that's not possible, let me know and we can see what other options are available.

thanks,
-kevin
