    Sep 18, 2006 11:26 PM UTC by leighg 
    I would like to report a bug in the gris init script
that is installed as /etc/init.d/gris. This script comes
with OSG installation as far as I understand.

The beginning of this script has a mix of different
formats with directives. It looks as follows:

# chkconfig: 345 99 99
# description: Grid Resource Information Service
### BEGIN INIT INFO
# Provides: gris
# Required-Start: $network
# Required-Stop:
# Default-Start: 3 4 5
# Default-Stop: 1 2 6
# Description: Grid Resource Information Service
### END INIT INFO

Note that first #chkconfig specifies the run levels, and then
they are later specified as well. When I use chkconfig
to install and switch on this service, it gets confused by
the mixed directives and fails to set the priority right.
The first line above specifies that the priority for startup
and shutdown are both 99. The chkconfig utility fails
to understand that and sets them to 50 both.

Can you make the set of directives to adhere one
format?

As a side note, can you change the startup
and shutdown priorities to something more reasonable?
(well, at least it seems to me :-)) I'd suggest startup 90
and shutdown 10, or if you want to keep 99, then
maybe startup 99 and shutdown 1? There are other
services that depend on gris and have to be started after,

    such as the bdii service.

thanks,
Ilya.
