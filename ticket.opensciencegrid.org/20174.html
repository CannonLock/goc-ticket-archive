    Hi Brad,

This package has several problems:

First, your package has no dist tag -- this means the Release field does not have the string %{?dist} at the end of it.
%dist is a macro whose value is based on the repo and distro version that the build is for. For example, if you are doing an el5 build in trunk, %dist will be ".osg32.el5".
This is necessary to avoid NVR (Name-Version-Release) collisions between the builds you do for different repo/distro version combinations.
That is why you got the "Build already in progress" error for your el6 build. So you need to add %{?dist} back to the end of the Release field.

Second, your el5 build failed because the setuptools Python library was not installed at build time. setuptools comes in the "python-setuptools" RPM, so you should add that to your BuildRequires line.

Third, you have the line "%define _tmppath /var/tmp" in your spec file. You should take that out -- osg-build and koji define _tmppath such that package builds will not collide with each other, so you should not be overriding that in the spec file.

Hope this helps,

-Mat
