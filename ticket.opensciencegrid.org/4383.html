Jeff,

I think this is an easy fix. I could just make the fix, but since you
are maintaining the SGE job manager right now, I thought I should tell
you how to make the fix.

In globus_scheduler_event_generator_sge_setup-1.1/setup-seg-sge.pl,
there is some code that looks like this:

my $path                = "$root/$cell/common/reporting";
my $help        = 0;

GetOptions('path|p=s' => \$path,
'help|h' => \$help);

&usage if $help;    # [dwm] This call does not return.

# [dwm] Check that our reporting file exists and is usable.
unless (-e $path && -r $path) {
die "'$path' is not readable or does not exist! Aborting!";
}

I think we just have to change the die to a print, like this:

unless (-e $path && -r $path) {
print "Warning: '$path' is not readable or does not exist!";
print "         Make sure that SGE's reporting is enabled before
submitting GRAM 4 jobs";
}

Do you think you can make this change before you next send me an update?
