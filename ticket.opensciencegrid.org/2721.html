    Hello,

Apparently, LIGO users have been assigned a different user ID at STAR_BNL.
LIGO apps rely on $OSG_DATA workdir for placing all input GWFs. So,
whoever has created this directory, other LIGO users won't be able to
access it.
We will modify our configuration script so each user name will be
appended at the end of $OSG_DATA to distinguish access privileges.

You can close this ticket and thank you very much.

    Nov 14, 2006 04:47 PM UTC by Wayne Betts 
    It appears that the immediate problem is that

/star/data08/OSG/DATA/workdir/

does not have group write permission:

[root@stargrid02 root]# ls -ld /star/data08/OSG/DATA/workdir
drwxr-sr-x   33 grid0715 stargrid     8192 Nov  1 18:36
/star/data08/OSG/DATA/workdir

It was created by a different user (David Meyers in all likelihood):

[root@stargrid02 root]# grep Samidi /etc/grid-security/grid-mapfile
"/DC=org/DC=doegrids/OU=People/CN=Michael Samidi 288621" grid0759
[root@stargrid02 root]# id grid0759
uid=70759(grid0759) gid=69902(ligo) groups=69902(ligo),8775(stargrid)
[root@stargrid02 root]# grep grid0715 /etc/grid-security/grid-mapfile
"/DC=org/DC=doegrids/OU=People/CN=David Meyers 54839" grid0715
[root@stargrid02 root]#

Note that each LIGO user gets mapped to a different grid account (and
furthermore, it is possible, though not common, that a cert's user
mapping could change to a different user at some point)

I'm not sure that I can or should go poking about somebody else's
directory permissions, such as by adding group write permission.

As a general question for anybody who wants to chip in, should site
admins (like me) "fix" things like this on demand, or expect the users
to look after themselves, after gently pointing out the problem?

(Or, and I shudder to consider this possibility :-(, have I
misunderstood or misdiagnosed the problem?!?)

-Wayne
