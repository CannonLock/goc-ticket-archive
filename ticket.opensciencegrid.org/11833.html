    Hi all,

we think we just fixed the problem. It seems to have arisen because
a recent OSG RPM update put back the 30-gpi.ini GPI configuration file
template which I had removed as part of the initial install/configuration
process, and therefore the default 'SE CHANGEME' started reporting
the template x Billion CPUs. I removed this template file again,
and that seems to have fixed the GIP numbers.

And maybe it'll still take a little more time for this info to propagate
itself through all the various LDAP servers? Let's give it an hour or two,
and then could you please check again to make sure everything looks correct
everywhere now?
