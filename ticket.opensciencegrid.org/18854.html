    Jan 22, 2014 09:41 PM UTC by Matyas Selmeci 
    Ok, good to know that everything's fine.

This is the history:
When we rebuilt all the Java packages we ship against Java 7, we put the 'java7' dependency into them so they would pull in the proper version of the JDK when installed. At the time, all of the Java 7 JDK packages provided by Redhat/CentOS/SL provided the 'java7' dependency instead of the 'java' dependency, so that's what we had to do.

At one point, Redhat decided that Java 7 was stable enough to make it the default. They added the 'java' dependency to their Java 7 JDK packages, and removed the 'java7' dependency, which caused install failures when people tried to install our packages. We provided a shim package that provided the 'java7' dependency to fix the install failures, and also complained on Redhat's bugzilla about the change, got them to add the 'java7' dependency back, resulting in the change you see in this errata. Now the JDKs they ship provide both 'java' and 'java7'. This makes our shim package unnecessary (though it's harmless to leave around).

Hope that helps,
-Mat
