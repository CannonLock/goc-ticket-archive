    Dec 2, 2016 07:38 PM UTC by Brian Lin 
    David,

They do get transferred back to the SPOOL dir on the submit host. It
goes something like this:

1. Job is running on a worker node, creates files, and gets evicted
2. Job goes back to idle
3. Created files are transferred back to the submit node to the job's
SPOOL directory because of your 'when_to_transfer_output = ON_EXIT_OR_EVICT'
4. When the job starts running again, the submit node transfers those
files along with all of your other input and executable files to the new
worker node

Does that help?

- Brian

    Dec 1, 2016 10:06 PM UTC by Brian Lin 
    David,

That submit file looks fine to me and according to the HTCondor manual
<http://research.cs.wisc.edu/htcondor/manual/v8.4/condor_submit.html#90746>;,
any files that are produced as output should be transferred along with
your other input files when the job starts running again:

"The /ON_EXIT_OR_EVICT/ option is intended for fault tolerant jobs which
periodically save their own state and can restart where they left off.
In this case, files are spooled to the submit machine any time the job
leaves a remote site, either because it exited on its own, or was
evicted by the HTCondor system for any reason prior to job completion.
The files spooled back are placed in a directory defined by the value of
the SPOOL configuration variable. Any output files transferred back to
the submit machine are automatically sent back out again as input files
if the job restarts."

You shouldn't have to worry about the location of the input files on the
submit machine if a job gets evicted. Are you looking for those specific
'checkpoint' files in a DAG post-script?

Thanks,
Brian
