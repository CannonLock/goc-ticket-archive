GOCTX Source: BNL Ticket: Queue=GLTier2 ID=24585
    Jul 10, 2014 05:31 PM UTC by OSG-GOC 
    Not all AGLT2 hosts are affected right by transfer failures, I suspect a subnet MTU mismatch causing large packets to drop in the AGLT2 network.

Only 192.41.236.59 and 192.41.239.58 are affected, other hosts seem to be transfering okay. Suspect that these hosts have MTU 9000 set, while the router has MTU 1500 set. Please set router-router links to something like 9192 or 9170 (same on both ends of the link), move storage nodes to MTU 9000 and confirm connectivity.

From these nodes, try:
ping -M do -s 8000 oliebol-04.nikhef.nl <- should give either ICMP fragmentation needed or reply, over LHCONE it should work without fragmentation, but no reply is invalid configuration.
ping -M dont -s 8000 oliebol-04.nikhef.nl <- should always reply
traceroute --mtu oliebol-04.nikhef.nl <- should always show full path with F=9000 or when going over internet F=1500
