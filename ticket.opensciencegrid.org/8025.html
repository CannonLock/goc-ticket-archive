    Please assign to vdt-support, this affects both production and integration.

The VDT is shipping two different and incompatible versions of curl
in vdt-2.0.0.  One of them, curl-7.18.2 supports the --retry
option and is the one that comes in the curl package.

The other, curl 7.11.1, is included as part of the PRIMA package
and does not support the --retry option.

This would not normally be obvious to anyone who installs an OSG-CE
because the OSG-CE does not include the curl package by default.
Nor would it be obvious to anyone who installs just a worker node client
because the worker node client package doesn't include prima
by default.  But when you have wnclient + glexec then the two
versions of curl end up together.  Likewise both are present
in a client+gridftp installation.

root@fg0x7 bin]# cd /usr/local/grid/prima/bin
[root@fg0x7 bin]# ./curl --version
