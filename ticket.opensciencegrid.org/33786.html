    May 22, 2017 08:03 PM UTC by Derek Weitzel 
    You could always reconfigure condor on the worker nodes to point to a directory under /tmp, if you would like.  For example, you could create a directory under /tmp called, condor_execute, so it would look like: /tmp/condor_execute.  Then, you could change your worker node's condor configuration to set EXECUTE to that directory:

(any condor configuration file on the worker node)
EXECUTE = /tmp/condor_execute

Then, all of the execute (as well as the $TMP, $TEMP, and now $OSG_WN_TMP) directories will be set to something under /tmp.  This is completely a site decision.  I just don't want jobs to overrun your worker nodes.

    May 22, 2017 04:25 PM UTC by Derek Weitzel 
    Hi,

OSG_WN_TMP is configured on the CE in the file /etc/osg/config.d/10-storage.ini.  For a condor cluster, like yours appears to be, I would recommend setting it to "$TMP" (without quotes).  The documentation for this variable is here:
https://twiki.grid.iu.edu/bin/view/Documentation/Release3/IniConfigurationOptions#Storage

The $TMP variable would be evaluated on the remote worker node, which should point to Condor's scratch directory.
