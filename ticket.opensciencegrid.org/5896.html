    Hi Tim,
Softenv is a system used to build the user's environment. Each user has a ".soft" file in which they specify groups of applications that they're
interested in. Softenv reads a central database when necessary to update the user's PATH, MANPATH and other variables.
Some more information:
http://www.ncsa.uiuc.edu/UserInfo/Resources/Hardware/CommonDoc/softenv-intro.html
http://www.teragrid.org/userinfo/softenv/ <http://www-new.mcs.anl.gov/systems/software/softenv/softenv-1.6.2.tar.bz2>;
