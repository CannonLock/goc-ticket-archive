    Hello,

so here is how this problem may be tested:

* create a simple bash script that periodically outputs to a log file

[robert@osg-ldg scripts]$ cat /tmp/ligo.test.sh
#!/bin/bash
for i in {1..100}
do
/bin/date > /lustre/hep/osg/ligo/rengel/ligo.test.log
sleep 6
done

* create a job description RSL v1.0 that uses the bash script

[robert@osg-ldg scripts]$ cat /tmp/ligo.test.sh
#!/bin/bash
for i in {1..100}
do
/bin/date > /lustre/hep/osg/ligo/rengel/ligo.test.log
sleep 6
done
[robert@osg-ldg scripts]$ cat /tmp/ligo.test.rsl
(* RSL v1.0 created by /DC=org/DC=doegrids/OU=People/CN=Robert Engel
392994 on osg-ldg.ligo.caltech.edu at Thu Jan 13 09:37:39 2011 *)
&
(count = 1)
(executable=/bin/bash)
(directory=/lustre/hep/osg/ligo/rengel)
(* arguments given to the application *)
(arguments=
/lustre/hep/osg/ligo/rengel/ligo.test.sh
)
(* location of stdout and stderr *)

    (stdout=/lustre/hep/osg/ligo/rengel/ligo.test.out)
(stderr=/lustre/hep/osg/ligo/rengel/ligo.test.err)
(* arguments used by GRAM *)
(maxWallTime=750)
(maxMemory=512)
(minMemory=256)
(jobType=single)

* copy the script to the resource

[robert@osg-ldg scripts]$ globus-url-copy file:///tmp/ligo.test.sh
gsiftp://antaeus.hpcc.ttu.edu/lustre/hep/osg/ligo/rengel/ligo.test.sh

* submit the job to GRAM using the RSL v1.0

[robert@osg-ldg scripts]$ globusrun -r
antaeus.hpcc.ttu.edu:2119/jobmanager-sge -f /tmp/ligo.test.rsl -F

.. this will return a job contact http:...

* query status of job periodically

[robert@osg-ldg scripts]$ globusrun -status
https://antaeus.hpcc.ttu.edu:40974/15998/1294941947/
PENDING

* watch out for the log file at the same time

[robert@osg-ldg scripts]$ globus-job-run
antaeus.hpcc.ttu.edu:2119/jobmanager-fork /bin/ls -al
/lustre/hep/osg/ligo/rengel/ligo.test.out
-rw------- 1 ligo tigre 1855 Jan 13 12:14
/lustre/hep/osg/ligo/rengel/ligo.test.out

This shows that the job is running, while GRAM marks it as PENDING. A
delay between the job starting and the job being flagged RUNNING by GRAM
is normal (~1min), but on your resource the PENDING never changes.

Greetings,
Robert
