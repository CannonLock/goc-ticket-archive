    Jun 8, 2017 09:40 PM UTC by Dave Dykstra 
    I'm reopening this ticket.  None of the osgstorage.org repositories have been added to the BNL stratum 1 yet, so everything is not fine.  From the user perspective it works because they read from FNAL or GOC, but every other repository is replicated to all three stratum 1s for redundancy and performance, and the osgstorage.org repositories should be replicated too.   BNL missed the requests for most of them originally because their cvmfs-server version was too old, but that is no longer the case.

In addition, John, note that you will probably find that the osgstorage.org repositories need garbage collection enabled.  If you have plenty of disk space, you can defer it until cvmfs-server-2.4 is released (in a couple of months) and you upgrade, because that version has a handy 'cvmfs_server gc -a' option to garbage collect all collectable repositories which you can run from cron once a day.  The FNAL & GOC stratum 1s have their own scripts to do that currently with cvmfs-server-2.3.3 or later (since 2.3.3 added a lock function to prevent garbage collection from happening at the same time as a snapshot).

The list of missing repositories is still the same:

des.osgstorage.org
ligo.osgstorage.org
mu2e.osgstorage.org
nova.osgstorage.org
stash.osgstorage.org
uboone.osgstorage.org

Don't worry about the initial space.  The garbage-collected space for all of them together currently adds up to only 17G.   I don't know how fast they grow without garbage collection but you could always run cvmfs_server gc for each one by hand as well.
