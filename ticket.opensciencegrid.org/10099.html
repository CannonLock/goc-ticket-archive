    May 2, 2011 05:15 PM UTC 
    For updates check the campfire transcript: https://integrationcloud.campfirenow.com/room/192189/transcript/2011/04/29

The next problem was to point to a different smtp port for the notification server.

In the file:
glite/var/etc/voms-admin/<VONAME>/voms.service.properties
e.g.
$VDT_LOCATION/glite/var/etc/voms-admin/GridColombia/voms.service.properties

The line
voms.notification.smtp-server =
sets only the server hostname
voms.notification.smtp-server.port =
sets the port you want to use

Diego will set the port and let know if it works.

    I tried to access gc-voms.javeriana.edu.co (200.3.153.50) but
I cannot ping it and it is not answering telnet on port 8443.
That means it must not have port 8443 open through the firewall.

I have seen errors like this before but cannot debug them
without browser access

Try https://gc-voms.javeriana.edu.co:8443/voms/GridColombia/services/VOMSAdmin?method=ListMembers&VoGroup=/GridColombia
That might give an error too but if it does, then "view source" on the web page and you
will see the XML.  One common cause of this is a member who has some illegal
character (tab or control characters) embedded in the DN.

If even that doesn't work then try

https://gc-voms.javeriana.edu.co:8443/

that should show the top level vdt test page, if it
doesn't then apache is broken and you should look at apache logs
$VDT_LOCATION/apache/logs

Also in browser https://gc-voms.javeriana.edu.co:8443/voms/GridColombia

should bring up top level voms-admin page, if that doesn't happen
then tomcat or voms-admin is broken, look in the tomcat logs.
