I talked with Mat about this -- apparently that 'probes' option is only for selecting the "jobmanager" and/or "gridftp" _categories_ of probes, but it does not let you select which jobmanager probes to enable, if multiple are installed together.

    Dec 14, 2016 08:21 PM UTC by Carl Edquist 
    Hi Dan, sorry for the delay,

Those two cron files are for separate probes, pbs-lsf and slurm.  Is there a reason why both probes are installed?  (Instead of just the one for the batch system in use?)

If you want to leave both the pbs-lsf probe and the slurm probe installed, only one of them should be enabled in /etc/gratia/*/ProbeConfig.  If a probe is not enabled in its ProbeConfig, the cron_check should fail, causing the probe itself not to run.

If you do not need both probes installed, you can yum remove gratia-probe-X for the one that's no longer needed.  This will also remove the corresponding cron file.
