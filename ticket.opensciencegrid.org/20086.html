    Hello experts,

I am having trouble renewing the host certificate for a small grid Storage Element (SE) at my university (UC Santa Barbara).  The SE is used by CMS experiment members.  Previously, the host certificates were obtained from the DOEGrids CA, so this is my first time trying to get a certificate through OSG.  To do so, I went to https://oim.grid.iu.edu and logged in with my CERN personal certificate.  Then I went to the host certificate request page.  After generating a CSR with the following command:
umask 077; openssl req -new -newkey rsa:2048 -nodes -keyout hostkey.pem_new -subj "/CN=cms6.physics.ucsb.edu"
I copied and pasted the certificate request into the appropriate box in the web interface, which resulted in the following error message:
Failed to find any GridAdmin who can approve certificates for CN: cms6.physics.ucsb.edu

The hostname of the SE is cms6.physics.ucsb.edu.  I have never requested a host certificate in this way before, so I'm not sure what I am doing wrong.  Is there a problem with my argument to the -subj argument of the openssl command, or is it a problem that I have logged in with the CERN personal certificate?  Please let me know if there is any additional information that I can provide.  I would really appreciate any help with resolving this issue.
