This seems to indicate either a network issue, or a software problem. Do you by
chance have any supplementary security software installed, such as Trusteer
Rapport? If so, try disabling the software and see whether you can connect.

If not, you can try clearing the browser cache, restarting the browser, and
then retrying the site.

Oddly enough, I've read that reducing the MTU value of a wireless connection
has helped users with this problem as well. For a wireless connection, this
would be:
- System Preferences -> Network -> (Airport on left) Advanced -> Ethernet (or
Hardware) -> Configure: Manually -> MTU: Custom, and a value such as 1300.

Let us know how things go.

Thanks,

    Hi Shih-Chieh,

Can you give a bit more information on what occurs during the the
failed connection? What is the error message you receive?

If you see an error indicating that the browser can't verify the
identity of the VOMS web site, as in the attached screen shot, you may
need to change the "trust" settings of the CERN CA chain files,
including the CERN Root CA, the CERN Trusted Certificate Authority,
and the lcg-voms certificate. When you receive the error, click Show
Certificate, and then for each of the certificates in the chain, click
Trust, and change the value of "When using this certificate" from "Use
System Defaults" to "Always Trust".  Alternatively, you may be able to
click a check-box for 'Always trust "CERN Root CA"' if one appears.
After changing the value for all certificates in the chain, click
Continue.

If this is not the issue, please provide additional detail.

Thanks,
