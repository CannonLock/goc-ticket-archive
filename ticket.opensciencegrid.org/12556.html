    Sep 21, 2012 07:31 PM UTC by Marco Mambelli 
    Some more details.

RSV should use fork to run the probes.
Fork, also managed-fork should run on the gatekeeper normally, please check the configuration if you think it is not. You may also try to run some jobs with jobmanager-fork.
As further tool, you can check where jobs sent to jobmanager-fork-poll are going and you can change what RSV is using.

I think in your version of RSV you would have to tinker the script files to change the jobmanager.
In osg-rsv/bin/probes/central/RSVProbeBase.pm there is a line where you can change the jobmanager used:
$o{'globusjobCmd'} = "$o{'globusjobrunCmd'} $o{'hostName'}/jobmanager-fork $extra_rsl $cmd ";
    Sep 21, 2012 06:36 PM UTC by Marco Mambelli 
    Hi David,
the jobmanager fork should run jobs on your gatekeeper.
If you installed managed fork this will be through a condor queue but still local.
To bypass managed fork and run real fork jobs for your RSV probes you can use "fork-poll" as jobmanager, that should bypass managed fork.
Best,
Marco
