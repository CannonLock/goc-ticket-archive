    May 17, 2007 09:16 PM UTC by Arvind Gopu 
    Hi Bill,

Sorry about the delay in getting back ... YEs, the change you made in that
osg-vo-user-map.txt file apparently gets over-written as we'd discussed on the phone
call 2 days back.

John Rosheck in the GOC did some testing and believes he's come up with a solution to
fix this "Engage" vs. "engage" issue: upgragind vo-0.6. Can you give this a whirl,
and let me know how it goes? John's still not sure it works for GUMS but if you would
not mind testing it, that might be helpful. Let me know.

Here are the steps suggested by John:

####
I ran the update for the vo package on tb10 using the following commands :

# . /grid/mypacmandir/setup.sh
(I used the same pacman I initially installed with)
# cd $VDT_LOCATION
# cp setup.sh /root/my-ce-setup-date.sh
(because I'm paranoid and loosing this file is very bad)
# pacman -allow save-setup
(supposed to make pacman save it too)
# pacman -update OSG:vo-0.6
(actually update the CE installation)
# edg-mkgridmap
(this will recreate the files in $VDT_LOCATION/monitoring, etc)

This should fix any of the vo name problems that some sites seem to have (UCR-HEP?).
Which VO's a site supports is now controlled by commenting out the actual "group
voms://" lines for each VO in the file $VDT_LOCATION/edg/etc/edg-mkgridmap.conf.  I
know this works for edg-mkgridmap but I haven't tested with GUMS.
####

Thanks & Cheers,

    Arvind
