    Hi,
I suspect that with old OSG users needed to source /osg/grid/setup.sh that would set all env variables. However with OSG3 everything installed in standard linux locations, so users should be able to execute commands as is with standard PATH of course. The decision was not to fake /osg/gri/setup.sh into doing this job to force users to switch to standard OSG3.
