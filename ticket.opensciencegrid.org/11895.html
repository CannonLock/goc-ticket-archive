Subject: no error message in voms-admin CLI

Hi,

We would like an error message to be produced that displays the DN of the certificate that the voms-admin CLI is attempting to use during a failure to access a VOMS server.

The problem we are experiencing is that voms-admin looks in several places to determine which certificate to use when establishing a connection to the voms server.  When voms-admin fails to estabilish a connection because it chose the wrong cert, it fails without producing a useful error message containing the DN of the cert it chose.  Simply putting the DN in the error message would help a user a lot.

Thanks,
Dan
