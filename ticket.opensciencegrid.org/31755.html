    Dear IHEP admins,

your site appears to be blocking AFS callbacks from the CERN AFS servers. These callbacks are integral part of the AFSv2 protocol; blocking them decreases AFS client performance and may contribute to server-side overloads.

Example lines from our logs (for Oct 2016, times are UTC+1):
afs287.cern.ch:Mon Oct 31 03:22:37 2016 CheckHost_r: Probing all interfaces of host 202.122.33.232:7001 failed, code -1
afs287.cern.ch:Mon Oct 31 06:46:37 2016 CheckHost_r: Probing all interfaces of host 202.122.33.236:7001 failed, code -1
afs290.cern.ch:Mon Oct 31 13:24:53 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7002 failed, code -1
afs290.cern.ch:Mon Oct 31 15:07:43 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7001 failed, code -1
afs290.cern.ch:Mon Oct 31 15:09:45 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7002 failed, code -1
afs290.cern.ch:Mon Oct 31 19:08:14 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7002 failed, code -1
afs290.cern.ch:Mon Oct 31 21:17:09 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7001 failed, code -1
afs290.cern.ch:Mon Oct 31 21:26:05 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7002 failed, code -1
afs290.cern.ch:Mon Oct 31 21:40:04 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7004 failed, code -1
afs290.cern.ch:Mon Oct 31 21:42:06 2016 CheckHost_r: Probing all interfaces of host 202.122.36.3:7003 failed, code -1

Client machines seem to be distributed over the following subnets:
202.122.33 , 202.122.36, (most from 202.122.36.3)

Short-term:
* Please review the usage of CERN AFS from these machines, as it might point to a misconfiguration by the site or the experiment. In particular, LHC experiments should be using CVMFS to access experiment software, but might fall through to AFS.

* Please enable incoming UDP to port 7001 (perhaps restricted to source ports 7000-7005) to all machines running the AFS client and contacting the CERN AFS cell. Use "vos listaddrs -cell cern.ch" in case you'd like to further restrict by source IP, this set will change over time.

Medium-term:
* CERN is phasing out AFS, please ensure that your local experiment coordinators (and affected users) are aware of this. Please escalate any issue, preferably via the experiments. All active CERN experiments have nominated AFS phaseout coordinators to make sure that all use cases are covered.

Thanks in advance,
jan
