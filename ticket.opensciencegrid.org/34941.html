    Sep 14, 2017 06:00 PM UTC by OSG-GOC 
    With the new LCMAPS VOMS authentication, I had a question about order of entries in a specific file like voms-mapfile.  The example [1] seems to hint that order does not matter but I'm curious how order in the file works or if the account mapping picked is based on order of more specific to less specific in terms of the DN.

For clarify the example [1] seems to say that glow is a fallback for things not matching glow1 and glow2 DN.  Is this the case?  Does order matter in the file or is order defined by some other conditions?

Thanks,
- Trey

[1]:
# map GLOW jobs to the 'glow' Unix account.
"/GLOW/*" glow
# map GLOW jobs in the chtc group to the 'glow1' Unix account.
"/GLOW/chtc/*" glow1
# map GLOW jobs with the htpc role to the 'glow2' Unix account.
"/GLOW/Role=htpc/*" glow2
