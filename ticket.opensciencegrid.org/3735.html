    Alain,

The current set up does break things in the OSG. This is from a
ticket opened by Burt with CMS.

Hello friends,

"ASGCCA has two CA certs:

a692434d Taiwan - ASGCCA http://ca.grid.sinica.edu.tw/ IGTF
efdf6f74 Taiwan - ASGCCA-2007 http://ca.grid.sinica.edu.tw/ IGTF

The problem is that the crl_url file points to the same CRL for both
CA certs, but it is only signed by the first CA cert. The middleware
(fetch-crl) drops the cert for the second CA.

ASGCCA needs to issue separate CRLs for each CA cert."

So I think a new package is necessary.

Thanks,
Rob
