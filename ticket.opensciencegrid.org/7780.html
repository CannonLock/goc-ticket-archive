    On Dec 2, 2009, at 1:54 PM, Open Science Grid FootPrints wrote:
> I assume this is different than Software Tools.  What will Software
> cover?

Software Tools covers internal software development. Software covers the breadth of software we distribute and support for our users. The names are confusing, but I didn't pick them. :)

Things I want in "Software"
* The Software requirements from stakeholders (currently in Software Tools, doesn't belong there).
* Internal communication about software being tested. For example, keeping track of what versions of Bestman are in which pre-releases of the VDT and which have or have not passed testing.
* A description of how we'll maintain the RSV probes (interactions with the RSV probe developers)
* Other communication about software

I don't think it's a good place for end-user software documentation--we already have the VDT web site and the ReleaseDocumentation twiki.

-alain
