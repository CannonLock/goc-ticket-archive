    Nov 1, 2016 02:25 PM UTC by OSG-GOC 
    Catalin,

The important point is not where the client configuration is, but which set of stratum 1s the clients read from. This ticket is about importing an OSG repository to the EGI stratum 1s; how the client configuration is done is not relevant.

Currently the default configuration rpm from the CVMFS project will read *.opensciencegrid.org repositories from the EGI stratum 1s. So this will work fine for RAL and the other EGI stratum 1s to import xenon.opensciencegrid.org and make it available at EGI sites. For the new releases of cvmfs client, EGI sites will instead be asked to install the EGI cvmfs configuration rpm, which will set CVMFS_CONFIG_REPOSITORY=config-egi.egi.eu, and then that config repository will define *.opensciencegrid.org also to read from EGI stratum 1s. It doesn't make a difference where the client configuration comes from, because in both cases *.opensciencegrid.org repositories will come from the EGI stratum 1s.

In OSG the client configurations are set to read from the OSG stratum 1s, either directly from rpm or via the OSG's configuration repository.

Dave
