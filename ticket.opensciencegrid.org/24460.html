    Hi,

An update from the new nova.opensciencegrid.org repository has been running to the FNAL and other CVMFS stratum 1s from the GOC stratum 1 for the last two hours.  Could the manager of nova.opensciencegrid.org please be contacted to ask that he publish code only in relatively small chunks, with some time in between, until the updates show up on a client mounting from the FNAL or BNL stratum 1?  Also please make sure no flux data files are published?

Also it concerns me that I see no .cvmfsdirtab at the top level.  That's going to be very bad performance if everything is published in one big catalog.  The manager should see
http://cernvm.cern.ch/portal/filesystem/maintain-repositories
and also look at the nova entries in /cvmfs/oasis.opensciencegrid.org/.cvmfsdirtab.

Dave
