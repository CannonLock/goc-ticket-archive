    All,

Most of the dialogue/resolution has occurred on a thread outside of this ticket.
Here is a summary of what has transpired.

The cause of the problem
---------------------------------
Back in September 2013 with the latest version of Condor, cpu and wall were getting reported
with scientific location and the gratia-condor probes were parsing this data incorrectly.  A patch
was done locally.   These patches have not yet made it to the latest OSG release.  In February 2014,
an upgrade was performed at Nebraska and the latest gratia-condor probe (w/o the patches for
scientific notation) was installed thus causing the under-reporting to re-occur.
This was not noticed until the February Tier 2 MOU report was issued on March 11.

Why no updates were sent from Gratia to APEL/WLCG
---------------------------------------------------------------------
Nebraska re-installed the patched probe and forwarded the corrected data to Gratia at that time.
However, the Gratia-APEL interface for a previous month (in this case February) only runs up until
the 5th day of the current month (March 5th) as explained here:
https://twiki.grid.iu.edu/bin/view/Accounting/GratiaInterfacesApelLcg#Frequency
This ticket was not issued until March 11 and, unfortunately, there was no cc on the ticket.

Additional issue
-------------------
During this same time frame, the DOE certificate used to communicate between Gratia and APEL
was expired and a new Digicert certificate did not get updated correctly with the interface brokers
on the APEL side.  Apparently, OSG Gratia has to be treated differently than other "sites" reporting
to APEL and we have not, as yet, figured out how that is done.  Understand that the DOE certificate
was set up several years ago and not changed since. So, we have been unable to send any data
(February or March) to APEL since 3/17.  This issue is being actively worked by APEL support.

To resolve the under-reporting
---------------------------------------
The interface for February has been re-run (w/o being able to send/update APEL/WLCG and the
Nebraska numbers look like this....

    Published Tier 2 MOU report and EGI portal view
Jobs - 207,362
Normalized CPU - 1,210,112
Corrected (as in, will be in the EGI portal view once the interface works again)
Jobs - 371,568
Normalized CPU - 8,255,080

I think that covers it.  If I have missed/mis-stated anything, please advise.
This ticket should be kept open until we here from Cath in lcg accounting.

John Weigand

    We discovered that the patch which allowed the condor gratia probe to parse Condor's scientific notation wallclock output got overwritten locally by a new OSG version of the probe. This happened to us as well last September when the Condor output first changed -- apparently that patch only recently made it into the official OSG release as per https://jira.opensciencegrid.org/browse/GRATIA-123

In theory this shouldn't happen again as it's now (finally) patched for everyone.

We are currently replaying what we can from condor_history, but some records are indeed gone for good.
