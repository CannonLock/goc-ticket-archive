    Jan 2, 2013 05:39 PM UTC by Soichi Hayashi 
    Mine,

I have 2 questions..

1. In order to associate another DN with their user account, user needs to enter new DN in a text field - since browser doesn't allow user to present both existing and new certificate simultaneously. This means that, user can enter any DNs including someone else's. We can validate so that user can't enter DN that are already associated with other OIM users, but if they are not registered yet, they can register any DNs. Will this be a security risk? If it is, then we can give user a special URL with a token and ask user to close down the browser, then re-open the browser with the special URL using the new certificate to associate.

2. Currently, OIM maintains authentication roles based on which DN they are using to log in. When a user associate a new certificate (DN), we currently give "OSG End User" role but none others. There is a Jira ticket to *possibly* modify this behavior such that when a user requests for a new certificate, the new DN will given authentication roles that are of aggregate of all authentication roles given to that user from all other DNs. I personally don't like this idea, but what do we need to do in case of user associating new DNs with their accounts themselves?

Thanks,
Soichi
    Jan 2, 2013 05:13 PM UTC by OSG-GOC 
    This request is about the process of registering multiple certificates for a OSG user in OIM. An OSG user who is already registered in OIM with a DOEGrids certificate has the ability to add additional certificates. Currently, the process for adding additional certificates requires manual approval by the OIM staff. The security team would like to work with GOC team  to review and modify this process to make it easier and faster for the end user. The security team's initial proposal is to remove the manual approval process for those users who are already registered in OIM with an existing certificate. In terms of security, we do not see any risk involved with this change.

Thank you

by Mine Altunay (guest)
