    Jul 27, 2011 04:21 PM UTC by aaronew@.... 
    Hi John,

The application I will be running is the 21cmFast simulation code that can
be found at http://www.astro.princeton.edu/~mesinger/DexM___21cmFAST.html.
It is a semi numerical modeling tool to simulate the 21 cm emission of
Hydrogen in the very early universe. A program, init.c is first run to
create mass density fields (governed by weakly interacting dark matter so it
can be performed independently of simulations of other physical quantities
of interest) and than Ts.c generates temperature fields from these initial
conditions at a particular redshift (time).

My goal is to simulate signal that fills a field of view for the MWA radio
telescope which would require a (.5 Gpc)^3 (approximately (1.5 billion light
years) ^3)  simulation box. However I could just as easily run say  one
hundred 5 Mpc^3 simulations and stitch the results together to form the
field of view. That said, my intent is to simply call 100 copies of the same
serial program on a relatively small data set in parallel. Data will not
need to be shared between each simulation and there is no input (save what I
enter into some parameter files). The output of each simulation should be on
the order of hundreds of megabytes for a total output on the order of tens
of gigabytes.

thanks,

-Aaron
