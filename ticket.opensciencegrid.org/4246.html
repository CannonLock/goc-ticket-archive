    Oct 23, 2007 06:23 PM UTC by Neha Sharma 
    Unofficially, you can do wget http://cmsdcam2.fnal.gov/dcache/ecrc
then chmod +x ecrc
ecrc -a <filename>
Check the 1-seeded crc with the value stored in layer2 and
see if they match.

Note that you have 'on write' set to be true, so your receiving pool
is telling you that it can't store the file because the crc is wrong.
You don't have it set on reading, so your client doesn't detect it.

Jon
