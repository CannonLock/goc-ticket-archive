    Jun 25, 2015 08:28 PM UTC by Carl Edquist 
    Hi Bob,

I'm not sure how the paths in your ProbeConfig got changed to

<     DataFolder="/var/log/gratia/rsv/data"
<     WorkingFolder="/var/log/gratia/rsv/tmp"
<     LogFolder="/var/log/gratia/rsv/logs"

but it looks like the defaults did not change between OSG 3.2.22 and 3.2.24 (that is, between gratia-probe 1.13.31-1.1 and 1.14.2-4).

Maybe it was originally configured that way by osg-configure?  (Before you updated to 3.2.24, I mean.)  The defaults in the .rpmnew version are the "standard" locations, but it should still work if configured to other paths.  So, if it is working for you now after you went back to your original paths (as pasted above), then really you don't need to fix anything.

But if you _do_ want to use the paths in the .rpmnew instead, you may have to manually create some directories if they are missing (as the warning message seems to indicate), or else doing a "yum remove" and then "yum install" again on the package in question (gratia-probe-metric) should create any missing directories and save any modified config files with a .rpmsave extension.

I'm including Mat in case he has any additional comments about rsv/osg-configure aspects of this.

Thanks,
Carl
