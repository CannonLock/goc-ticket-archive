    May 2, 2007 10:05 PM UTC by anand-padmanabhan-1@.... 
    Hi Alan,

There are two separate issue here,
1.) How to get SE publish GIP information independent of any CE, i.e.
have GIP installed on SE and have SE publish information directly.
2.) How to publish from a CE, information about what SEs that may be on
a separate box.

The first part is something that is not implemented yet and would
possibly need more than just changes to GIP package to get it up and
running. The second issue is not a problem, since GIP can be configured
to publish this information correctly.
During the GIP configuration questions, you should be able to
appropriately configure your CE to publish any standalone SE
information. This is achieved by answering  the following 2 questions in
the srm section of the CE configuration:

What is the registered OSG  sitename of your SRM Storage Element? (e.g.
UIOWA-ITB): [UNAVAILABLE]
What is the hostname of your SRM Storage Element? (e.g.
rsgrid3.its.uiowa.edu) : [UNAVAILABLE]

I don't know, if you had a chance to look at the following GIP page
https://twiki.grid.iu.edu/twiki/bin/view/Integration/ITB_0_5/GenericInformationProvidersText.
I hope it address some of your concerns about documentation holes. This
page has been updated relatively recently, and we will appreciate
feedback on making it better.

Thanks
Anand

The general problem I am trying to solve can be summarized in the
question "What is the equivalent step on a SE to the configure-osg.sh
that is run on a CE to set up its monitoring Glue schema, etc., for
proper reporting?"  This is the only problem that remains, since you
answered the question about how to register an SE.

As you mention, no consensus seems to exist on how to do this, but I
think as a matter of opinion that there should be a configure-osg-
se.sh or something like that to set up the SE information for proper
reporting to be used by VORS, etc.  Note in general there may be one
or more SE's at a site that are not directly part of the CE(s) there,
so configuring an SE for reporting to the OSG monitoring systems
cannot be assumed to be part of the SE install steps, although
allowing specification of a separate SE is already part of the CE
configure-osg.sh steps.

Hope this helps,

    Apr 10, 2007 07:05 PM UTC by John Rosheck 
    Hello,

Some basic procedures for setting up SE's can be found at :
https://twiki.grid.iu.edu/twiki/bin/view/ReleaseDocumentation/StorageElementAdmins

I've noted that this SE is separate from the CE clusters at your site but it will
still need something that can report GIP information.  I'm not an expert on these
matters but it seems likely that the easiest way to accomplish this would be to load
the standard CE package but without enabling the gatekeeper, jobmanager, etc.

For monitoring to function, we'll need this resouce registered as a storage element
with a valid shortname and the SE endpoint to use.  Other information that we will
need that currently isn't captured by the registration process is the host name and
port of the CEMon/BDII/LDAP server and the applicable search string one would need to
use to get the information specific to the SE.  Most of the
time this will be mds-vo-name=local,o=grid assuming that you aren't reporting to an
aggregator.

Please let me know if you have any other questions.

Thanks,
John
