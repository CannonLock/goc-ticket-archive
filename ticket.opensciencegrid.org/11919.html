    [by Marco Mambelli]

Background:
UTD just updated the CE and RSV to OSG 3.0.
The Se is still OSG 1.2 because they cannot update to Xroot 3.1.1 (because of needed python binding) and are afraid that updating to the OSG 3 SE would require a newer Xroot.

Errors:
Running the RSV probes that were running against the old SE they had 2 problems:
1. All the probes were failing because Bestman was not recognizing the new format of the Globus proxy.
In the bestman log file there are java errors.
It looks to me like the GSI code doesn't understand the RFC 3820 proxy certificate extension (1.3.6.1.5.5.7.1.14).

2. After solving 1 with a workaround
The srm-copy was failing because the root destination directory was "None"

Discussion:
1. grid-proxy-init makes RFC 3820 proxies.
To make old style proxies you need to add "-old". There is no way to specify this in a RSV configuration using a service certificate.
The workaround is to change the configuration to use a user certificate:
user_proxy = /tmp/myrsvproxy

And add to the crontab of the rsv user something like:
grid-proxy-init -old -cert /etc/grid-security/rsv/rsvcert.pem -key /etc/grid-security/rsv/rsvkey.pem -out /tmp/myrsvproxy -valid 72:0
to run once a day.
I will add this to the RSV documentation.

It would be nice to have the possibility to let RSV use the old proxies when creating them with the system cert (it could even be the default)

2. The workaround for 2 is to add the srm_dir entry (not tested yet).

; The srm_dir options gives the directory on the srm servers that the RSV SRM
; probes should try to write and read from.  This should be a comma separated
; list containing the same number of entries as srm_hosts (the first entry
; in srm_hosts corresponds to the first entry in srm_dir, etc)
srm_dir = DEFAULT

    
Anyway either the default should provide a valid directory or osg-configure should raise an error. Or at least the probe should.
The value "None" should never appear as directory name.
I don't know if the error is in osg-configure, RSV or rsv configuration.

Best,
Marco
