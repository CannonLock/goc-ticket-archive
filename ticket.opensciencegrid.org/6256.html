Hi Joel,

I am not able to duplicate the problem. Is it on-going?

Changes in our storage set up are creating an issue with dZero files,
and may be contributing to this problem.  The situation is this:
Previously, all our files were stored in non-reserved (non-space-token).
Now the vast majority of our storage is converted to reserved
(space-token), and dZero is the only serious user of non-reserved space.
The non-reserved space is all being served from 1 pool.  When many dZero
jobs start at once, this pool is overwhelmed.  For a long-term solution,
I'd like to move dZero to reserved (space-token) space, so we can spread
the file-serving load over all 44 pools.

--Sarah
