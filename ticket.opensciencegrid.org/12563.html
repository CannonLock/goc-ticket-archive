    Hello,

We suspect the problem was caused by a large gratia queue backlog. The files that should have been compressed were open, compression continued until the files closed and the system then returned to normal. This explains all we observed, the timing on the queue size and inode problems is exact and self correction is explained.

I am resolving this ticket now. We can revisit it later if the problem persists. It might be worth considering some sort of modification to the compression plan but that is a call for the developers.
