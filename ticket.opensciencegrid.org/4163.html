    Oct 23, 2007 03:24 PM UTC by CIGI 
    Hi,

Gabriele, forwarded this problem to me. So I did investigate this
problem and here is my take on it.

1. As for the df errors you are seeing, if you answered that you dont
have an SRM it will assume that you have a classicSE, in which case
$OSG_DATA is assumed to be shared. So it tries to do a df. If it fails
the only effect you would see due to these errors is that GlueSE
attributes like Free and Total size will be set to 0.
2. As for the error you see with GROUP_PRIO_FACTOR_group_ligo, I am not
sure why it would occur. The only way I see this error occurring is if
%condor_config_val GROUP_NAMES returns ligo in its list of VO, but there
is no corresponding query %condor_config_val
GROUP_PRIO_FACTOR_group_ligo doesnot return a valid entry. This can be
easily fixed, and I will get it fixed in the new release.

Please correct me if I am wrong, but I am still not convinced that this
will cause the CEMon and ReSS issue you are seeing. The only effect you
would see is that the dynamic values reported by condor will be set to
0, but your site should still be reporting to ReSS.

Thanks,
Anand
