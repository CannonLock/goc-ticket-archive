    Oct 1, 2015 05:16 PM UTC by Suchandra Thapa 
    Hi Nate,

The issue is that gratia is trying to query for PER_JOB_HISTORY_DIR to make sure things are configured correctly.  If your submit node is heavily overloaded, then the invocation ( condor_config_val -schedd PER_JOB_HISTORY_DIR or something substantial similar) may sometimes fail.  You can check for this by running the command a few times to see if you get a non-zero exit code.   If that's the case, then you can probably ignore this safely provided that the job records aren't piling up in PER_JOB_HISTORY_DIR or the gratia directories.

Suchandra
