    Jun 29, 2015 09:48 PM UTC by OSG-GOC 
    Thanks for digging this out. This is exactly the info we needed to debug the REBUS reporting issue in another email thread.

In OSG CE configuration, we used to set both resource name and resource group name to WT2. When we migrated the CE to SL6 2-3 years ago, we were asked to keep the resource name and change the resource group name to WT2_SL6. The fact the your ldap search asked for GlueSiteUniqueID=WT2 but get GlueSiteUniqueID=WT2_SL6 in the return probably indicate a bug in OSG BDII (GIP) somewhere.

I will report this issue to OSG. In the mean time,  I will change the resource group back to WT2 to see if this fix the REBUS reporting problem. We want to resolve this issue quickly.Changed CC to mernst@....
