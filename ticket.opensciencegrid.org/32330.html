    Feb 13, 2017 05:22 PM UTC by OSG-GOC 
    Hi Marian,

It was an annoying bug/race condition. Before sending the proxy from
the CE to the worker node, someone was making a limited copy of the
proxy and setting the permissions to 600 to only be readable by the
glidein user. If that file existed when Condor noticed the job ended,
it would try to send it back to the factory, but since the permissions
on the file were restrictive, it would fail, giving the obscure TOOL
permission error.

It works with the stock BLAH libraries because they tack on a manual
"rm" at the end of the script to remove the limited proxy. I made a
small change to the slurm implementation to keep from having to have a
shared filesystem between the CE and worker nodes, and I missed
changing that one line. With the little fix I added, things get
cleaned up properly and I can have a CE without a shared filesystem
again.

Lemme know if you need anything else,
