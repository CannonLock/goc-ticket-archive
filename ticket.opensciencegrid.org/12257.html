    Hi Alain,

You are correct about the osg-ca-scripts, I was thinking wrong before.  osg-ca-scripts should be an example of functionality that we need to add.

osg-ca-certs is the only RPM that sites HAVE to update when there is a release, or RSV will go into warning (and eventually error).  I think there is little understanding of that.

In Pacman, it was automatically updated, without any interaction necessary from the administrator.  This is certainly a loss of functionality from the Pacman distribution.  I believe we should provide some method to add this functionality to the RPM distribution.

Also, large sites have their own copy of the RPM repo, and therefore will need to update the repo before running the yum update.
