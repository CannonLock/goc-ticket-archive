    Sep 22, 2009 08:02 PM UTC by Tanya Levshina 
    The following information has been received from dcache developer(Dmitry
Litvintsev) who has helped to resolve this ticket:
***********************************************************
Solution was:

specify

-streams_num=1

option to srmcp.

In a firewall situation, when reading in passive mode (which is default) one
has to specify number of streams to 1. The default is 10.

The latest client RPM (which is not yet available from dcache.org) has a
workaround to the effect that is

-server_mode=passive

specified, then number of streams is set to 1 automatically.

I showed Jim how to create srm config file and set number of streams in this
config file to 1 so no need to specify it as command line argument.
****************************************************************************
