New Work Log Entry :
Wei/Ilya/MIT_CMS Admins,

MIT_CMS has stopped uploading RSV records to our collector. I suspect you forgot to
turn osg-rsv back on after your maintenance period.

If that's the case, then please start osg-rsv, and skip the rest of this note.

If indeed you started osg-rsv but something is not working right, then read on ....
we're trying to dig into these problems with a profiler script -- can you run the
following profiler script, send us that tarball, and then restart RSV? Thanks!

Download Profiler:

wget http://vdt.cs.wisc.edu/scot/rsv-profiler
. rsv-profiler
# provide access to tarball to rsv-dev by email or via web

To Restart:

cd $VDT_LOCATION
vdt-control --off osg-rsv condor-cron
vdt-control --on condor-cron osg-rsv

Hopefully that'll resume the RSV  client. To view RSV probe and consumer jobs ...

condor_cron_q

And you can look at:

$VDT_LOCATION/osg-rsv/logs/consumers/*.log
(and   $VDT_LOCATION/osg-rsv/logs/probes/*.log)


    to see there are any indications of any errors.

Let me know if you have any questions.

Thanks,
Arvind
