It seems to be a bug in the way GUMS is generating the list of ranges -- when you have 00-99,100-999,1000-7999, it seems to try to sort the ranges in ascii order instead of numerically, and then it ends up with a mess of small ranges.  You might be able to fix this for the time being by removing the 100-999 range and replacing it with 0100-0999, etc.  But hopefully we'll have a proper fix for the next GUMS release.
