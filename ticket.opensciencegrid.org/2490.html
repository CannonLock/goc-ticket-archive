    I'm not sure why the content got cut out.  Suffice to say, the Globus
security patch requires VDT_PRETEND_32 to be set if you're patching a
32-bit installation on a 64-bit platform.  If the upgrade is
attempted without this set, it will install 64-bit binaries which
look for 64-bit libraries (which don't exist).
