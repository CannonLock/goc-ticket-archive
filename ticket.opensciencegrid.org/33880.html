Trying to understand the suggestion
Are you suggesting to use a placeholder in the values in the config file that then is replaced at runtime with the hostname?
Or to eliminate it some other way?

Currently the host name replacement is used for variables (or part of them) that use the hostname normally but could be configured to do otherwise,
like the URL of the factory files, condor identity of the frontend, DNs of the certificates.

Things to consider:
- the URL is by default on the host but may be also somewhere else (proxy or file server) (here we could use a relative URL assuming the local host if
not present)
- HTCondor collector and schedd may be on a different machine
