    Jun 21, 2011 05:27 PM UTC by Ian Stokes-Rees 
    Hi Elizabeth,

Sorry for not getting back to you sooner.  Here are my comments:

1. A single link to the ESNet CA would be nice.

2. Firefox procedure is different depending on platform, I believe --
certainly this is the case for Chrome and I believe Safari.

3. Chrome and Safari procedures should be included too (and maybe Opera).

4. Current web-based procedures are marked as "Failed", which isn't very
encouraging.

5. IE instructions are very cursory and do not outline procedure for
cert request, import, or export.

6. The Firefox instructions are pretty good, but a few short notes on this:
* optional steps should be clearly marked as such.  Extra work will
only frustrate the user.
* I would not have "OSG" as the example VO selection -- users who
read this page are bound to think this is directing them to put the same
information in both drop down boxes.
* it should be made clear that "When your certificate request has
been processed and a certificate issued" is a process that requires a
human being to review the request and could take 1-2 business days, plus
a pointer on who to contact if nothing happens.
* the file should be called "usercred.p12" since this can be used
directly from ~/.globus.

7. There are no instructions on how to import a key.

8. There are no notes on how to renew a cert (before and after expiry).

9. There are no pointers on where to put an exported cert for command

    line use.

Regards,

Ian

> You should have received an email with a URL that will allow you to retrieve
> your certificate. You need to use the same browser and computer that you used
> when you made the initial certificate request. I have added you to the WeNMR
