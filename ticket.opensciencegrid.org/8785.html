On Jun 18, 2010, at 8:27 AM, David Lesny wrote:

Neha,

I was able to look into this last night and think I figured it out....

The upgrade saved a new Probeconfig in the gratia areas as Probeconfig.rpmnew,
and leaves the previous release as Probeconfig.

However, the older config file does not appear to be compatible with the new gratia probe.

What I did was take the new config from .rpmnew, make the appropriate changes
and make it the active copy.  That seems to have solved the problem below.
I have not received any additional errors and the reports seem to be going to the collector.

You might want add this information to the upgrade document.

One other item, though.  You create a cron job

/etc/cron.d/gratia-probe-dcache-storage.cron

with the line

15 * * * * root "/opt/d-cache/gratia/probe/dCache-storage/dCache-storage_meter.cron.sh"

The problem with this entry is that it generates an email every hour with the message
