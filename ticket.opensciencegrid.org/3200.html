    Mar 13, 2007 04:21 PM UTC by VDT 
    Sorry for the slow response.

We sent email to the GOC yesterday (Monday, 12-Mar-2007) with details of
how to address this problem. I copy it below for your reference. Please
let us know if you need more assistance.

Hi folks,

For the VDT, there are two possible software updates for the recent
Daylight Savings Time update.

1) An update for MySQL. We looked into this, and it seems we do not need
it for the VDT. (It updates tables that are not used for any VDT
application we are aware of.) So you can ignore this one.

2) An update for Java. We are not sure if this is required for any of
the Java programs in the VDT, but we tested it with the Java in VDT
1.6.1, and it was a safe, easy update. Because we trust Sun, it should
be equally safe for older versions of the VDT. You can read about the
update from Java, and apply it directly with their updater:
http://java.sun.com/developer/technicalArticles/Intl/tzupdatertool.html

A summary of the update procedure:

a) Download the updater

b) Make sure there is a Java in your installation
cd $VDT_LOCATION
. setup.sh   (or source setup.csh)
which java  (does it show a Java in your $VDT_LOCATION?)
If Java isn't installed in the VDT, there is no need to update.

c) Stop running services
VDT 1.6.0 and later:
vdt-control --off

    VDT 1.3.10/OSG 0.4.0:
Stop MonaLisa, Tomcat, jClarens and/or the Globus container if they
running.

d) Run it with:
java -jar tzupdater.jar -f -bc
