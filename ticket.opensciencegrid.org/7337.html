Arvind Gopu wrote:
With all the re-locations of caches that have taken place, I am
wondering if this is still the location of the latest official OSG
gums.config template.
http://software.grid.iu.edu/pacman/tarballs/vo-version/gums.template

Yes. Each time we release a new VO package, we deploy:
- Two tarballs for vo-package and vo-client in both OSG-1.2
(http://software.grid.iu.edu/osg-1.2/tarballs) and OSG 1.0
(http://software.grid.iu.edu/pacman/tarballs/)
- untar the tarball in
http://software.grid.iu.edu/pacman/tarballs/vo-version

We were not aware that the gums.template link you have linked above was
hard-coded in a script somewhere but had a sneaky suspicion someone was
using the untarred content. So we decided to retain it with every
vo-package release.

Ideally, given the OSG software cache structure, the gums-create-config
script would download the pacman file
http://software.grid.iu.edu/pacman/vo-package.pacman -- get the tarball
URL, download it, untar it to find the template, instead of depending on
an untarred directory content.

The last notification of a change pointed us to an svn repository.

The GOC has an agreement with VTB in which the latter will test changes
to the vo-package before a production release. These could come from
anywhere, including a subversion trunk link (even though ideally, the
BTB would provide us a way to deploy it in the VTB cache, and then VTB
testers would update their VO-package using known mechanisms -- for this
to happen, first the VTB would (ideally) be hosted on the same server as
the production and ITB caches.

Cheers,
-ag
