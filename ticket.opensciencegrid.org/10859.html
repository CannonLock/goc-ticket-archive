    Hi David,

The metric is complaining about certificates, not CRLs - so look at
the files that end in '.0' (instead of '.r0', which are the CRLs).
Running vdt-update-certs was the right step to fix the problem, but it
appears that it wasn't running correctly from cron since you hadn't
received an update since 1.17.

Is vdt-update-certs running in cron?  You can examine root's crontab
and look for 'vdt-update-certs-wrapper'.  If it's not, you can turn it
on by:
. VDT_LOCATION/setup.sh
vdt-control --on vdt-update-certs

If it was turned on check its log file at
VDT_LOCATION/logs/vdt-update-certs.log and
vdt-update-certs.log.system_output to look for problems.

The second part of your answer - how to run RSV metrics by hand.  For
this metric, you can do the following (after sourcing setup.sh)

rsv-control --run --host umiss001.hep.olemiss.edu
org.osg.certificates.cacert-expiry

There is more documentation on rsv-control here:
https://twiki.grid.iu.edu/bin/view/MonitoringInformation/RsvControl

Let me know if you have further problems.

Thanks,
scot
