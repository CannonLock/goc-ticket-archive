On Apr 16, 2010, at 11:54 AM, McKee, Shawn wrote:

Hi Everyone,

We have been seeing an intermittent problem (likely correlated with high loads on our infrastructure) where successfully transferred files don’t have
their dCache “level-2” metadata ready when queried.  With the current ATLAS infrastructure this results in job failures for these relatively rare events.

We are running dCache 1.9.5-15 with Chimera on SL 5.4 64-bit systems.

Some quick questions:

1)      Is this a known ‘timing’ issue in dCache?
2)      How long should it be once a file is transferred before the level-2 metadata ( ‘.(use)(2)(filename)’) is ready?
3)      Any tuning that could help reduce this interval?

The pilot code could possibly recognize this situation and retry after a short interval.

Thanks,

Shawn
