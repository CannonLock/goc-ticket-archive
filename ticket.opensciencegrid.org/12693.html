    Nov 2, 2012 10:05 PM UTC by Matyas Selmeci 
    Brian investigated this and informed me (off-ticket) of the following finding:

The site's /etc/globus/globus-gram-job-manager.conf had the line
-cache-location /var/lib/globus/gass_cache
which should have been
-cache-location /var/lib/globus/gass_cache/$(LOGNAME)

This caused multiple users to attempt to upload their files into the same gass_cache directory. The first user "won" and denied write access to the other users. This caused the staging errors.

The site's still red in RSV though. Was this fix made?
