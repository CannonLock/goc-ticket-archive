Dear OSG support:

I have posted directly to Mats Rynge but since it’s very early yet in California, I thought to send it to you too.

My jobs have been running well for several days.  I made a change in how a common data file is fetched yesterday afternoon.  Maybe it’s unrelated but here it is. I had been having it fetched separately to every client.  That is still happening but now I’m using http in hopes that the file will sometimes get cached locally and so reduce the network load on the frontend from which the files are staged.  That has been working well since yesterday afternoon – several 10’s of 1000’s of jobs.  Here’s the http stuff in the script I run on each client:

source ${OSG_GRID}/setup.csh
if ( "${OSG_SQUID_LOCATION}" != "UNAVAILABLE" ) then
setenv http_proxy ${OSG_SQUID_LOCATION}
wget -q -O Data${argv[3]}.tar http://osg-xsede.grid.iu.edu/scratch/donkri/${Pkg}${argv[3]}.tar
set Check = `ls -s Data${argv[3]}.tar`
if ( ${Check[1]} == 0 ) then
unsetenv http_proxy ${OSG_SQUID_LOCATION}
wget -q -O Data${argv[3]}.tar http://osg-xsede.grid.iu.edu/scratch/donkri/1${Pkg}${argv[3]}.tar
endif
else
wget -q -O Data${argv[3]}.tar http://osg-xsede.grid.iu.edu/scratch/donkri/${Pkg}${argv[3]}.tar
endif
set Check = `ls -s Data${argv[3]}.tar`
if ( ${Check[1]} == 0 ) then
exit
endif
