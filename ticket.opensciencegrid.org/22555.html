    Oct 1, 2014 01:42 PM UTC by OSG-GOC 
    As a part of the transition of the running of the Gratia-APEL interface from FNAL to the
OSG GOC that occurred back on 8/22/14, the change described below is required.

The gratiaweb WLCG Reporting pages ...
http://gratiaweb.grid.iu.edu/gratia/wlcg_reporting
http://gratiaweb.grid.iu.edu/gratia/jot_reporting
get the data they display from this url...
http://gratia-apel-wlcg.opensciencegrid.org:8319/gratia-apel/
which is created by the interface module.

This needs to be changed to ...
http://reports.grid.iu.edu/gratia-apel/
as the gratia-apel interface has been running on reports.grid.iu.edu since 8/22/14.

The 'gratia-apel-wlcg.opensciencegrid.org' is actually an alias for gr13x6.fnal.gov
and that node is being taken out of service this Monday 10/6.
I have been running the interface in parallel (w/o updates to APEL) on gr13x6 in
order to retain visibility to these pages.

It is my understanding (and may be incorrect) that there is a configuration file..
/usr/lib/python2.4/site-packages/gratia/config/urls_list.conf
than contains these entries and they look like this based on the source code I've seen.
ApelUrl  http://gratia-apel-wlcg.opensciencegrid.org:8319/gratia-apel/{year}-{month}.summary.dat
ApelUrl2 http://gr13x6.fnal.gov:8319/gratia-apel/{year}-{month}.summary.dat

So, it may be just a simple matter of modifying that on the production node and then making the
changes to the source files so a future upgrade does not cause them to revert.

Please advise if there are any questions and just what steps were done so I can document it for
future reference.

Thanks
