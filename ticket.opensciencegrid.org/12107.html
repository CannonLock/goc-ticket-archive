    May 17, 2012 08:17 PM UTC by Bockjoo Kim 
    Hi Jack,

Following your gip_debug and Looking line 54 in /usr/lib/python2.4/site-packages/gip_cese_bind.py
, I notice job_manager (line 27 of the same python) is wrongly set as 'default'.

On brux3, the following command confirms job_manager value is wrongly set as 'DEFAULT':
/usr/bin/gip_print_config | grep job_manager

[ce]
job_manager               : DEFAULT

So, the value capital DEFAULT is set for the job_manager.

By grepping DEFAULT against all config files,
grep DEFAULT /etc/osg/config.d/*  | grep -v \~ | grep -v \; | sort -u
/etc/osg/config.d/01-squid.ini:cache_size = DEFAULT
/etc/osg/config.d/01-squid.ini:location = DEFAULT
/etc/osg/config.d/01-squid.ini:memory_size = DEFAULT
/etc/osg/config.d/01-squid.ini:policy = DEFAULT
/etc/osg/config.d/30-cemon.ini:bdii_servers = DEFAULT
/etc/osg/config.d/30-cemon.ini:ress_servers = DEFAULT
/etc/osg/config.d/30-gip.ini:batch = DEFAULT
/etc/osg/config.d/30-gip.ini:gsiftp_host = DEFAULT
/etc/osg/config.d/30-gratia.ini:probes = DEFAULT
/etc/osg/config.d/30-gratia.ini:resource = DEFAULT
/etc/osg/config.d/30-rsv.ini:service_proxy = DEFAULT

I think /etc/osg/config.d/30-gip.ini:batch = DEFAULT
is the origin of the error.
You need to change from batch = DEFAULT to batch = condor
and execute the gip_info again to see if you still have another error.
Bockjoo
