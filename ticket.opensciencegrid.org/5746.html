    Hi Tim,
I figured out that it was not yet updated by looking at your pacman
command history, which is included in the system profiler (I've pasted
it below).

VDT 1.10.1i (with the change to CA-Certificates) was released on Sept
11th.  The first updates after that (on October 6th)  do not depend on
CA-Certificates (CA-Certificates-Updater is a dependency of
CA-Certificates).  So the first time CA-Certificates would be updated
is on 9/09.

We have been releasing a lot of lettered updates to the VDT
(increasingly because stable versions are expected to be around longer
than before) and we realize it is hard for admins to know what has
been upgraded.  In 1.10.1k, we rewrote parts of our vdt-version tool
to help admins determine what software has been updated.  vdt-version
now attempts to determine if an update is available for a package, so
you can more easily determine if anything is missing instead of trying
to go through all the VDTs release notes. Unfortunately, we cannot do
this for all packages immediately.  In order to determine if an update
is available, we need to edit the pacman file, which makes pacman
think an update is available.  So we will be phasing this in for more
packages with each lettered release.  If you have a chance to try it
out, please feel free to give us feedback, we would like to improve
this so that admins will find it to be a useful tool.

Thanks,
scot

    Oct 9, 2008 09:10 PM UTC by VDT 
    Hi Tim,
The CA-Certificates package on your CE was never updated since we
released VDT 1.10.1i (on September 11).  The package was updated for you
today because you updated Apache, which relies on CA-Certificates, and
pacman also looks for updates for all dependencies.

The first time after updating the CA-Certificates package from 1.10.1i,
it is necessary to select a certificate distribution (by editing
vdt-update-certs.conf) and to run vdt-setup-ca-certificates to properly
re-install the certificates.

Unless the VDT team has another update to the CA-Certificates package,
future updates will not require you to perform these steps.
