our code uses Condor-G and only requires one job-manager to run on

of jobs ( ~800 ) the load on the gatekeeper remained low for two days.

related to LIGO. I would expect a larger than usual load only to occur
on the file server hosting $OSG_DATA in a situation when many of our
jobs are finishing at the same time and write out data to the disk. On
the gatekeeper we only run the globus job manager and hourly a cleanup
job ( 1 thread ) that is unlikely to cause any problem. At least we
haven't observed such yet.

$OSG_DATA if you suspect that would solve the problem.

Please note, that I have observed very high load issues on your
gatekeeper before ( load > 100 ) at times when we were not running at
your side!
