    Gregory,

Hello.  You will need to move your certificates into a directory called .globus.  (the period before globus is important, as it is part of the name of the directory)  Are you doing this on your Mac or on a Linux box that you ssh into?

With regards to the Simon database, you may need to contact Ian Fisk, who is our point of contact for the Simon Center.  I can add him to this ticket if you would like.

With regards to the OSG Client installation, I have been researching this a bit to clear up some confusion.  The software development team is fixing the document with the four broken links for you.  That being said, installation of the OSG Client (or any of the OSG Software stack) on MacOS X has not been supported in quite a few years.  According to https://twiki.grid.iu.edu/bin/view/Documentation/Release3/SupportedOperatingSystems , the only supported systems are:

CentOS 5 (3.2 only)
CentOS 6 (3.2 and 3.3)
CentOS 7 (3.3 only)
Red Hat Enterprise Linux 5 (3.2 only)
Red Hat Enterprise Linux 6 (3.2 and 3.3)
Red Hat Enterprise Linux 7 (3.3 only)
Scientific Linux 5 (3.2 only)
Scientific Linux 6 (3.2 and 3.3)
Scientific Linux 7 (3.3 only)

Did you read somewhere in the documentation set that MacOS X was supported for OSG Client installation?

Hopefully this points you in the right direction.  Please let me know if you need more assistance.

Hello Gregory,

On the site you want to add, right-click the red lock icon in the
address bar.

Click the tab labeled Connection, then click Certificate Information

Click the Details tab, the click the button Copy to File.... This will
open the Certificate Export Wizard, click Next to get to the Export
File Format screen.

Choose DER encoded binary X.509 (.CER), click Next

Click Browse... and save the file to your computer. Name it something
descriptive. Click Next, then click Finish.

Open Chrome settings, scroll to the bottom, and click Show advanced
settings...

Under HTTPS/SSL, click Manage certificates...

Click the Trusted Root Certification Authorities tab, then click the
Import... button. This opens the Certificate Import Wizard. Click Next
to get to the File to Import screen.

Click Browse... and select the certificate file you saved earlier, then
click Next.

Select Place all certificates in the following store. The selected

    store should be Trusted Root Certification Authorities. If it isn't,
click Browse... and select it. Click Next and Finish

Click Yes on the security warning.
