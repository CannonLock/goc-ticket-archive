    Aug 15, 2008 07:21 PM UTC by VDT 
    > I don't see ldapsearch in my $VDT_LOCATION/globus/bin area on either
> the CE or wn-client installations  for OSG 1.0.
>
> Is there a package I need to install to get this?  Is it in the OSG
> (non-worker-node) client package instead?

I'm not aware of any requests from VOs to have globus/bin/ldapsearch, so
we've made no effort to guarantee it's there.

Historically we've had it in Globus because it comes with MDS and will
still install MDS. I don't know why we install it--no one uses MDS and I
expect it will be dropped in the future.

MDS doesn't build on 64-bit platforms, so the 64-bit VDT don't have
ldapsearch from Globus. My strong suspicion is that this is your
problem. With OSG 0.8 you probably still got MDS even though it doesn't
build on 64-bit platforms because we tossed in the 32-bit version, but
we no longer even install the 32-bit version on 64-bit platforms due to
problems linking against the system OpenSSL.

We incidentally happen to have the OpenLDAP 2.3.39 client in the VDT to
support lcg-utils. I think the OSG worker node has lcg-utils, so you'll
get it there, or you can install the "OpenLDAP" package to get it. This
is what I recommend you install. It won't be in the Globus directory,
but in the openldap/bin directory, and I think your PATH won't include it.

Until I have a request to ensure that ldapsearch is included, I can't
guarantee it will continue in it's present form. The packaging might be
restructured, or the version might be changed to work with new versions
of lcg-utils.

-alain
