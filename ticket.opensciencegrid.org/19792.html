    Hi all,

Sorry I dropped this.

Yes, banning users is correctly described here:
https://twiki.grid.iu.edu/bin/view/Documentation/Release3/BanningUsersAtSite
No special or direct manipulation with the DB is needed.  Just follow those instructions and it should work through the web UI, i.e.
1) create a manual user group 'bannedUserGroup',
2) add DN/FQAN definitions to it, and
3) under 'Configuration' select the newly-created bannedUserGroup under "Banned User Groups".

Yes, wildcard should work as listed in that page that was referred to.

To ban by CA, one would do, e.g.
//DC=org/DC=doegrids/OU=People/CN=*, .*
to ban all doegrids user certs, regardless of VO.

If fine-grained banning is meant to be supported, then OSG should do more testing to be sure this really works as it is supposed to. And in general, if more sophisticated banning is needed then SAZ should be brought to the fore again. GUMS isn't a good place to be doing this.

Cheers,

--john
