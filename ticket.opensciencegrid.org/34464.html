    Greetings,

If I understand this correctly, Baylor is now running CentOS7 on their worker nodes and wants to run CMSSW via Singularity.

The first issue is accessing local files inside the container. Although rather terse, this is mentioned in the documentation:

https://twiki.cern.ch/twiki/bin/view/Main/CmsSingularity

under the section: Configure POSIX Storage Elements.

If CMSSW reads files at your site through the POSIX APIs (i.e., file:// protocol), then the POSIX storage must also be accessible inside the container as the pilot user.
Two potential changes will be needed:
Change the permissions for CMS files such that the CMS pilot user can read CMS data.
Bind mount the storage to /cms and update storage.xml accordingly. To be visible, a bind mount must be used (symlinks will not suffice).

I realize the above is obtuse, but I think they have to bind mount /data under /cms/data and then add a rule to storage.xml to map file://data to file://cms/data.

I think the bigger stumbling block is the "#s in file names" from the BLAHP.  As I recall, Baylor uses PBS. If so, I believe this is still an outstanding issue. The Singularity failure due to #s is in their SAM tests; the isolation test falls back to glexec.

Thank you,
doug
