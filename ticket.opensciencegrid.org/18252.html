This is related to https://ticket.grid.iu.edu/goc/17984.  I did not notice your question
on 11/21 because the ticket had already been closed.  There is something not right
in how the Fermi ServiceNow and goc ticket system interact such that ServiceNow
automatically closes the doc ticket.  And the goc system does not send email out after
a ticket is closed (the most annoying part).

ANYWAY... your question was...
"Why can't the site be Crane?  It is certainly the name of the cluster.
Crane-gw1 is just the hostname of the gatekeeper.
Maybe I do not understand the site / probe relationship."

In your /etc/osg/config.d/40-siteinfo.ini file, you should be populating the
resource attribute with the resource name in OIM as the comments in the file
indicate...
; The resource setting should be set to the same value as used in the OIM
; registration at the goc
;
; YOU WILL NEED TO CHANGE THIS
resource = UNAVAILABLE

When you run "configure-osg -c", it populates the SiteName attribute of
the /etc/gratia/condor/ProbeConfig with the value of the resource.
In this way, there is some correlation between Gratia and MyOSG/OIM when
trying to troubleshoot and for reporting purposes.  You should note that goc
generated tickets for resources, not resource groups.

Also, you are referring to the cluster as "Crane-gw1" which appears to be part of
the FQDN and is not relevant to anything in Gratia or OIM in terms of the
hierarchy.

Make sense?
If so, can you fix it in the 40-siteinfo.ini file and then run 'configure-osg -c'?
