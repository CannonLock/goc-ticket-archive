Hey all,


    Based on my understanding from the respective experts, there is no
cheap/easy fix for the race condition which Charles Waldman point
out.  This is, to some extent, a design-level problem due to the
default setting of the GridFTP querying PNFS for information (on the
other hand, the suggested fix has GridFTP querying PnfsManager).

Please use the following setting in the Setup files of any gridftp
door:

PermissionHandlerDataSource
=diskCacheV111.services.PnfsManagerFileMetaDataSource

Brian

    We have uncovered a bug related to creation of directories
using the dCache gridftp door.   This bug is reproducible and
only affects the gridftp door - the other doors do not seem to
exhibit this behavior.

This was uncovered in the course of troubleshooting an
issue with RSV probes, but is a dCache bug.  There is a
document with full details at

http://twiki.mwt2.org/bin/view/Main/GridFTPDirectoryBug

This writeup refers to the context of the RSV probe
problem, but has all required details about the dCache
problem, including a script which provokes the error.

Hey Charles,

A couple of comments:

- The "sleep" commands are present because, I understand, XrootD has an arbitrary 5 second delay between file
creation and when the file is safe to use.  If that wasn't the reason, then it probably was to
- Add the following line:

PermissionHandlerDataSource=diskCacheV111.services.PnfsManagerFileMetaDataSource

to /opt/d-cache/config/gridftpdoorSetup (assuming you're running a sufficiently recent dCache) and restart the
GridFTP door.

Does that solve your problem?  I could recreate your problem and confirm that change solves it for me.

Brian

I CC'd Gerd and Tigran, the relevant folks who would be interested in this bug.  Folks, the context of this
message is here:

    
http://twiki.mwt2.org/bin/view/Main/GridFTPDirectoryBug

This problem is completely solved if you switch the PermissionHandlerDataSource for the GridFTP door, which
suggests something might be wrong in PnfsFileMetaDataSource or PnfsFile.  More below.
For the record, Brian's suggestion  to add

PermissionHandlerDataSource=diskCacheV111.services.PnfsManagerFileMetaDataSource

to gridftpDoorSetup seems to have cleared up the problem at UC - my tests
ran for 200 passes without triggering the error.  Thanks for the
timely response and the helpful suggestion.

- Charles
