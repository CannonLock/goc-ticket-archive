    Aug 27, 2015 05:48 PM UTC by Thomas Lee 
    Hiro,

If you are attempting to obtain a DigiCert-Grid certificate with a SAN (Subject Alternative Name) field, it is just like obtaining one without: generate the CSR and enter it in the form at https://oim.opensciencegrid.org/oim/certificaterequesthost .  The difficulty is in generating a CSR with a SAN field.

(If you are attempting to obtain a certificate from some other CA, the process, and indeed the entire question of whether a SAN is acceptable, will depend entirely on that CA and its policies.  Here we are assuming you wish to obtain a certificate from the DigiCert-Grid CA.)

I know of two ways to create such a CSR.  One way is using an OpenSSL command, but in order to do so you would have to create (or copy and modify) an OpenSSL configuration file, as this is the only way to tell OpenSSL about your SAN data.

For example, you would need to create a file – let's call it openssl_with_san.cnf – in which the [ v3_req ] section contains

keyUsage = digitalSignature, keyEncipherment, dataEncipherment
subjectAltName=DNS:altname1.bnl.gov,DNS:altname2.bnl.gov

You must also make sure that the [ req ] section contains this line:

req_extensions = v3_req

Then, you would need to refer to that configuration file (which must contain all the other configuration directives necessary for creating the certificate, which is why I suggest you copy it from an existing openssl.cnf file such as the one in /etc/pki/tls found in Red Hat type distros) in the OpenSSL command line that creates the CSR:

$ umask 077; openssl req -new -newkey rsa:2048 -nodes -keyout /path/to/hostkey.pem -subj "/CN=fts03.usatlas.bnl.gov" -config openssl_with_san.cnf > /path/to/hostcsr.pem

Another way to create the CSR: I have had some success with the Perl module Crypt::OpenSSL::PKCS10, using a script somewhat like this:

#!/usr/bin/perl
use strict;
use warnings;
use Crypt::OpenSSL::PKCS10;
my $hostname = 'fts03.usatlas.bnl.gov';
my @alts = qw( DNS:altname1.bnl.gov DNS:altname2.bnl.gov );
my $req = Crypt::OpenSSL::PKCS10->new(2048);
my $dn = "/DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=$hostname";
$req->set_subject($dn);
$req->add_ext(Crypt::OpenSSL::PKCS10::NID_key_usage, 'nonRepudiation,digitalSignature,keyEncipherment');
$req->add_ext(Crypt::OpenSSL::PKCS10::NID_subject_alt_name, join(',', @alts));
$req->add_ext_final;

    $req->sign;
$req->write_pem_req('/path/to/hostcsr.pem');
$req->write_pem_pk('/path/to/hostkey.pem');

Both of these methods create the CSR in a file, /path/to/hostcsr.pem (vary the path to taste), as well as the key, hostkey.pem.  I suspect there are many other ways to do it (there is almost certainly a way to do it in Python, and I'm fairly certain that someone has found a way to do it with Ruby).  To verify that it has the correct information before submitting it, use the command

$ openssl req -text -noout -in /path/to/hostcsr.pem

Once you are satisfied with the content of the hostcsr.pem file, paste its contents to the form in the URL listed above.  I hope this helps.
