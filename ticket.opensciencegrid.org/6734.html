    Apr 23, 2009 05:36 PM UTC by Christopher Pipes 
    Hello,

I'm conducting an audit of VO registration information for the Security
Team, and I notice some problems with the data returned by
<https://oim.grid.iu.edu/pub/vo/show.php?format=plain-text>;.

First, the vo_long_name for the ALICE and JDEM VOs contains a comma,
breaking the CSV formatting.

Second, some of the URLs are invalid:

http://osg.ccr.buffalo.edu/grase/
http://osg.ccr.buffalo.edu/contact/grid_help.php
Connecting to osg.ccr.buffalo.edu|128.205.41.178|:80... failed: Network
is unreachable.

http://www.its.uiowa.edu/research/grow
http://www.its.uiowa.edu/research/grow/osg/
HTTP request sent, awaiting response... 404 Not Found

https://fermigrid2.fnal.gov:8443/vomrs/i2u2/vomrs
Connecting to fermigrid2.fnal.gov|131.225.107.46|:8443... failed:
Connection refused.

http://jdem.fnal.gov/
unable to resolve host address `jdem.fnal.gov'

http://www.ivdgl.org/ivdgl/mis-vo-privilege.html
http://www.ivdgl.org/ivdgl/ops-vo-privilege.html
http://www.ivdgl.org/ivdgl/osgedu-vo-privilege
HTTP request sent, awaiting response... 404 Not Found

https://www-mariachi.physics.sunysb.edu/voms/mariachi/
HTTP request sent, awaiting response... 404 Not Found


    http://www.compbiogrid.org/support
HTTP request sent, awaiting response... 404 Not Found

Third, the voms_url values listed for ATLAS, CIGI, and IceCube do not
point to VOMS servers.

Fourth, many of the support_url values are: "Support URL missing!"

I see at
<https://twiki.grid.iu.edu/bin/view/Operations/OIMRegistrationInstructions>;
that the Support URL is optional, so maybe we should just leave the
support_url value empty if none is given?

Fifth, in most cases the aup_url does not point to an AUP. I can find no
AUPs at the URLs specified for ATLAS, CDF, CIGI, CMS, CompBioGrid, DES,
DOSAR, DZero, Geant4, GLOW, GPN, i2u2, NWICG, NYSGRID, SBGrid, SDSS, or
STAR. The URLs given for GRASE, GROW, mariachi, MIS, OPS, and OSGEDU are
unreachable or do not exist (see above).

Engage has <https://osg-engage.renci.org:8443/vomrs/Engage/vomrs>; listed
which references the OSG AUP with the
<http://osg-docdb.opensciencegrid.org/>; URL where the AUP is not easy to
find. It'd be better if a direct link to
<http://osg-docdb.opensciencegrid.org/cgi-bin/ShowDocument?docid=86>; was
given.

Fermilab has <http://fermigrid.fnal.gov/>; which has a link to
<http://fermigrid.fnal.gov/policy.html>;, but it'd be better if
<http://fermigrid.fnal.gov/policy.html>; was listed here.

nanoHUB has <http://www.nanohub.org>; which links to
<http://nanohub.org/legal/abuse>; which looks like an AUP. It'd be better
if the direct URL was listed here.

Thanks,
Jim
