And indeed, now when I look for that Sender Policy Framework record (for those not in the know, there's no actual DNS record type designated "SPF"; SPF records are just specially-formatted records of type TXT, much like Hesiod), here it is:

$ dig +short -t TXT support.opensciencegrid.org
"v=spf1 ip4:email.freshdesk.com a ~all"

This should basically mean, "This is an SPF version 1 record stating that if email arrives with a 'from' address at support.opensciencegrid.org but that was relayed via email.freshdesk.com, or via support.opensciencegrid.org itself, it's OK.  Otherwise, it's a SOFTFAIL, which is usually accepted but tagged."

It should mean that, but I don't think it does.  I think the "ip4:" requires an IP address, and I asked the DNS admin about this in my request and provided alternate versions of the record, one with the hostname and one with the IP, and they used the hostname version, but the SPF tester at http://www.openspf.org/Tools rejects this SPF as invalid because "Invalid IP address: email.freshdesk.com".  I'm consulting with the DNS admin about this.

    Hi,

Can you create an SPF record in the DNS entry for support.opensciencegrid.org?

This is so that email sent from freshdesk is not flagged as spam.

And here's how you do it:
Go to your domain registrar's website.
Login to the domain control panel.
Open your DNS configuration settings.
Edit your ZONE File and look for TXT or SPF record.
Specify the following value: v=spf1 include:email.freshdesk.com ~all .
Save your changes.
But what if an SPF record already exists?

You can just include the SPF in the same line as your existing record
v=spf1 include:spf1.domain.com include:spf2.domain2.com include:email.freshdesk.com ~all

Ref: https://support.freshdesk.com/support/solutions/articles/43170-creating-an-spf-record-to-ensure-proper-email
