    Oct 21, 2012 03:21 AM UTC by Marco Mambelli 
    Hi Terry,
in your version of OSG the probe is in perl. You found and ran the correct one.

Looking at the directory I think the problem may be that the probe and the programs run against the content of the TRUSTED_CA directory while the update script is updating the content of the certificates* directories. TRUSTED_CA should have been a link, I don't know how it became a directory. I'm expecting it to be now out of sync.

Can you check if the content of TRUSTED_CA, see if it is the same of certificates then and do something like:
cd /opt/osg-ce/globus/
mv TRUSTED_CA TRUSTED_CA.old
ln -s certificates  TRUSTED_CA

Then try again to run the probe.
Thank you,
Marco

PS About the slowness of the host. Sometime there are files accumulating due to Globus not cleaning up after the jobs that may clog the system. The osg user home should not contain much. There are some cleanup scripts in OSG: https://twiki.grid.iu.edu/bin/view/ReleaseDocumentation/VdtCleanupScripts
Another way, if you use the OSG directory only for gread jobs is to wipe it out clean during your next planned downtime/maintenance
