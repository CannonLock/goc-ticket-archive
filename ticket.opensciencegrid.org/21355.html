Soichi,

Order

We have discovered what the issue was that prevented the service_name from being added to the common name on the certificate. It looked like a random issue when we first looked at it but we found that when our system matches the common name with a previously issued certificate with the same common name and it will expire soon we call code for a "Renewal" of the certificate. The function for renewing a certificate had not been updated with the ability to pass the service_name through to the common name.

Our developers are working on the resolution now. I will let you know when the fix has gone through our QA process. I will need to have approval through your change control process before we push this bug-fix live.

Alain,

It's called "grid_request_host_cert". The "service_name" parameter that we are setting seems to be ignored.

Following is how I am making this API call.

HttpClient cl = createHttpClient();

PostMethod post = new PostMethod("https://www.digicert.com/enterprise/api/?action=grid_request_host_cert";);

post.addParameter("customer_name", StaticConfig.conf.getProperty("digicert.customer_name"));

post.setParameter("customer_api_key", StaticConfig.conf.getProperty("digicert.customer_api_key"));

post.setParameter("response_type", "xml");

post.setParameter("validity", "1"); //security by obscurity -- from the DigiCert dev team

post.setParameter("common_name", cn);

//post.setParameter("sans", "a.digicert.com,b.digicert.com,c.digicert.com");

if(service_name != null) {

post.setParameter("service_name", service_name);

}

post.setParameter("csr", csr);

post.setParameter("hash", StaticConfig.conf.getProperty("digicert.hash"));

post.getParams().setCookiePolicy(CookiePolicy.IGNORE_COOKIES);

Thanks,

Soichi
