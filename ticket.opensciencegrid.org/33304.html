    Unfortunately this file is not present on the MWT2 SE. And will need to be replicated to the SE.

I am not certain why there was an error 54 on this job failure, but when I go to the same node
and try the transfer I get a file not found (error 2) returned as it should.

Yesterday, there was a 3-4 hour period where DNS was failing at the UChicago site.
DNS reverse lookups were failing to resolv to host names.

As a result, transfers to/from the MWT2 SE were failing due to authentication failures (host name mismatch)

At the time, UChicago networking was reconfiguring how DNS worked on campus.
The caches on the DNS servers needed to be cleared to reflect the changes as to how to connect to the AWS DNS servers.

Once the caches were cleared, reverse lookups starting working again and transfers to the SE succeeded.

It is possible that this DNS issue caused some lingering issues on the workers etc, which cleared up on cache timeouts etc.
