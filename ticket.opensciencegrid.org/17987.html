    In OIM, it appears that the new Purdue-Conte resource group will be
reporting Gratia accounting data to WLCG/EGI.  This is based on the InteropAccounting
flag in the WLCGInformation of OIM.

If this is not true (not to be reported to WLCG), please reset the InteropAccounting
flag to False.

If true, then these issue need to be addressed
(also see https://twiki.grid.iu.edu/bin/view/Interoperability/AccountingAndWLCG)

1. the WLCGInformation AccountingName should be changed to T2_US_Purdue

2. you will need to register this new resource group (EGI site) with LCG accounting
(LCG.office@....) so it is added to the Rebus topology.

3. you will need to provide me with a Normalization Factor for the cluster
https://twiki.grid.iu.edu/bin/view/Accounting/GratiaInterfacesApelLcg#Normalization_Factor

As of this date, I see that you are not reporting any data as yet to Gratia.
When you do, be sure that the site-info ini files contain the correct resource and resource group
names.

Until you are actually ready for this resource to go live, it might also be beneficial to mark this
resource as down for maintenance or as inactive.  By doing so the Gratia-APEL interface will
not generate any exception messages for you.

I know this is a lot of info.  If you have any questions, please ask.

John Weigand
