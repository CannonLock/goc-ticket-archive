    Dec 16, 2015 07:23 PM UTC by Kyle Gross 
    Dear FNAL team,
in WLCG we have been running a campaign this autumn to get SRM certificates
ready for an upcoming change in the Globus algorithm used by clients to verify
if a service certificate is acceptable.  More details are provided here:

https://twiki.cern.ch/twiki/bin/view/LCG/WLCGBaselineVersions#Globus_GSSAPI_Name_compatibility

ATM there are at least 2 SRM hosts that would be affected by that change:

-----------------------------------------------------------------------------
=== cmssrm.fnal.gov:8443 HYBRID
server name: /DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=cmssrmtemp.fnal.gov
authenticated server name is acceptable
OK
=== cmssrm.fnal.gov:8443 STRICT_RFC2818
server name: /DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=cmssrmtemp.fnal.gov
Error authenticating: authenticated peer name does not match
FAIL
-----------------------------------------------------------------------------
=== fndca1.fnal.gov:8443 HYBRID
server name: /DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=fndca4a.fnal.gov
authenticated server name is acceptable
OK
=== fndca1.fnal.gov:8443 STRICT_RFC2818
server name: /DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=fndca4a.fnal.gov
Error authenticating: authenticated peer name does not match
FAIL
-----------------------------------------------------------------------------

Please look into replacing the affected certificates with compliant versions,
to avoid bad surprises when the Globus update starts getting used.

Affected Site: USCMS-FNAL-WC1
