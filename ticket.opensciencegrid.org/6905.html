So, a few comments:

o	you shouldn't need the "noac" option.  That was required by PNFS,
but with Chimera it shouldn't be needed any more.

dropped on test node
o	you seem to have all caching set to zero (acregmin, acregmax,
etc).  This is probably a side-effect from setting "noac", but it
would be worth verifying that omitting the "noac" option results
in non-zero ac* values.

o	you also have "sync" set.  This, again, is probably a side-effect
of specifying "noac"; but, please check it goes away when omitting
the "noac" option.

o	finally, just to point out that you explicitly set the "intr"
option.  This option is deprecated (after kernel v2.6.25) and
is ignored in recent kernels.

dropped on test node.  Should I also drop "hard"?
These may improve performance of Chimera by reducing the load that the clients place on the server.
