    Aug 28, 2014 12:59 PM UTC by Christopher Pipes 
    YI.

GOC ticket instances (ticket1/2/itb)'s ticket view pages were broken since 8/28 3:30 UTC (around 8/27 midnight).

GOC ticket generates a XML containing a list of similar tickets every day at 3:30 UTC. Last night, there was some kind of issue at FP, or connectivity to FP which prevented the generation of this XML. The cron job was not designed to handle this failure, and it generated the broken group list XML which then broke the ticket view page.

2014-08-28T03:30:06+00:00 ERR (3): 32534@4.303 Error Message ---------------------------------------
Couldn't contact IU Footprint server : looks like we got no XML document

I have done following.

1) Updated the cron task so that it will only overwrite the group XML if the new content is valid.
2) Made ticket2 to run the cron task at 6:30 UTC instead of 3:30 for ticket1 to minimize the chance that both ticket1/2 will have outdated XML due to FP issue.
