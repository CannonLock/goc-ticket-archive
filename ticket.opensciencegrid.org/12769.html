    Sorry, the recursive option is not supported for srmmkdir.  It will work for srmrmdir but will not create parent directories for you using srmmkdir.
I think that the behavior for srmcp is a little different based on what SRM server you are using.  I think that dCache will create parent directories for you, but BeStMan SRM does not.

If you are willing to use the BeStMan srm client, then the "-mkdir" optiion for srm-copy will create the parent directories for you when you copy files.  However, even the BeStMan srm-mkdir command has no similar option.

Sorry for the inconvenience, but I don't think there is the relevant option for you.
