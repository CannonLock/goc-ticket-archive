    Steve,

sonofsam ran out of disk space, and the mysql server on it choked. I've removed some old log files to free up some space, and restarted mysql. I also did "mysqladmin flush-hosts" since the OIM connection was black-listed on sonofsam due to repeated connection error. The OIM page now seems to function again.

We have various issues that we need to solve.

1) The URL https://oim.grid.iu.edu/pub/resource/show.php?format=plain-text was deprecated 2 years ago. I am amazed to find that this URL still existed, and even more amazing is that someone is still using this..

Pleas switch to use following URL instead

http://myosg.grid.iu.edu/rgsummary/xml?datasource=summary&summary_attrs_showhierarchy=on&summary_attrs_showwlcg=on&summary_attrs_showservice=on&summary_attrs_showfqdn=on&gip_status_attrs_showtestresults=on&downtime_attrs_showpast=&account_type=cumulative_hours&ce_account_type=gip_vo&se_account_type=vo_transfer_volume&bdiitree_type=total_jobs&start_type=7daysago&start_date=05%2F29%2F2011&end_type=now&end_date=05%2F29%2F2011&all_resources=on&facility_10009=on&gridtype=on&gridtype_1=on&service_1=on&service_5=on&service_2=on&service_3=on&service_central_value=0&service_hidden_value=0&active=on&active_value=1&disable_value=1

2) Although the OIM page is long deprecated, it shouldn't be using OIM DB stored in sonofsam. sonofsam is not scheduled to be re-tired. OIM scripts should be using its own DB.
3) We have several munin graphs that don't have correct alert threshold set where we aren't getting any alerts. sonofsam seems to be one of them. I've been trying to discuss this internally to audit each of our services and make sure that correct alert thresholds are set, but we haven't found a time do this yet. We need to raise priority on this item.
4) We need to add more tests on rsv.grid's service monitor so that if it fails to connect to DB (sonofsam), it will send an alert. I have not received any goc-alert from rsv.grid, and I didn't know that rsv-sam was failing due to sonofsam mysql being off line (as well as this deprecated OIM script)
5) Find the root cause of disk space ran out on sonofsam and fit it (although sonofsam is soon to be re-tired..)

Thank you for the report,
