    Hi Samir,

Sorry, I misspoke a bit about the proxy_url setting, and should clarify a bit. Although we could set proxy_url on the factory side to the local site's proxy URL, that would become rather tedious and difficult to maintain. Instead, the glideinWMS developers included an option to set proxy_url="OSG", which is what we have set for Caltech (and for a lot of other entries). Setting it to OSG means that the job will automatically discover the local web proxy at the site. However, due to this bug in glideinWMS, jobs are being sent with proxy_url="None" instead, so they don't even attempt to use a proxy. Of course, that doesn't explain why glideins are failing on wget only some of the time.

We could try temporarily setting the proxy_url manually to Caltech's local proxy until the bug has been fixed. Of course, I don't know if that bug only applies to proxy_url="OSG", or if proxy_url is set to any value, so manually setting it to Caltech's may still not result in it being passed along.
