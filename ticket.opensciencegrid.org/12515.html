    Dec 5, 2012 07:43 PM UTC by Anand Padmanabhan 
    Could you tell me which version of the script you are using (are you getting it from OSG 3.0 or earlier versions of OSG).

I don't want to hold you up from requesting a certificate, so I would suggest you use the doegrids website as an alternative approach. Go to http://pki1.doegrids.org/ca/  and click on Grid or SSL Server.

To generate the request that you should put into to PKCS #10 Request do the following steps on your linux command line
% umask 077; openssl req -new -newkey rsa:2048 -nodes -keyout hostkey.pem -subj "/CN=rhic20.physics.wayne.edu"
(This will write a hostkey.pem in the current working directory, so please take care not to overwrite your existing certificate)
The command will also output a CSR that would be something like
-----BEGIN CERTIFICATE REQUEST----
............................................................
-----END CERTIFICATE REQUEST-----
This whole block including the being and end should be place in the PKCS #10 Request text box on the doegrid page. Fill out all the other information (for host and service certificate for OSG services leave the optional passwd blank) and submit the request.

Once it has been approved you will get information on how to retrieve the public key (Note the private key was generated by openssl command and is already on your machine). You can then move the public and private key to the appropriate location.

Let me know if you have any questions
