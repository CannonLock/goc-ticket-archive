    Dec 10, 2007 11:02 PM UTC by Thomas Wang 
    fGOC,

Can you please contact the admins at UFlorida-IHEPA regarding incorrect GIP attributes?
UFlorida_IHEPA: Inbound Network Connectivity: ReSS--No, SysAdmin--Yes

Anand has provided instructions below on how to correct this (osg version specific).
Please let us know if you have any questions.
Thanks.

regards,
Tom

-----------------------------------
For a 0.8.0 site:
The Inbound and Outbound connectivity is in gip-attributes.conf, but
assigning it by hand may be tedious. Ask the sites to do
1. source setup.sh
2. Run $VDT_LOCATION/monitoring/configure-osg.sh or
$VDT_LOCATION/monitoring/configure-osg-gip.sh
3. For each sub cluster associated with the site answer the following
two questions correctly
a. Is there Inbound connectivity to these nodes? (i.e. TRUE, FALSE)
b. Is there Outbound connectivity to these nodes? (i.e. TRUE, FALSE)
4. Run VDT_LOCATION/lcg/libexec/osg-info-wrapper to verify the GIP output.

For an 0.6.0 site:
There are no attributes in gip-attributes.conf associated with inbound
and outbound connectivity. The default values are False and True
respectively. Sites will have to manually override these values using
the method identified in
https://twiki.grid.iu.edu/twiki/bin/view/ReleaseDocumentation/GenericInformationProviders#Overwriting_attribu
tes_published
