    It looks like globus-job-manager was "stucked" and no OSG job was sent in our local batch (globus-job-manager kept /var/lib/globus/gram_job_state/nova/novastore/fork.1b8fa43f.lock see log file at the end) - I killed this process and now glidein jobs again successfully starts in our local batch.

Meanwhile I also created "qdel" wrapper that return exit code 0 in case of attempt to delete non-existing job (exit code 153), because we repeatedly received requests to delete jobs that were killed on 9.11. (because of issues with air-conditioning) and (all?) non-zero qdel exit codes are probably evaluated as an error removing running job from local pbs batch.
