    Sep 11, 2015 07:29 PM UTC by Matyas Selmeci 
    Hi Tony,

You can get a list yourself by installing the yum-utils rpm and running
repotrack --urls --repoid osg,epel osg-ce
(replace osg-ce with whatever osg package you are interested in).

That will give you a list of URLs of RPMs.  If you leave off --urls, it will download the RPMs as well.

-Mat

    Hi Tony,

There is a difference between the globus gsi rpms not getting installed versus the wrong versions getting installed, and I'd like to know which one is actually happening on your machines.

Are you using the UW condor rpm?     Routing to the software team:

Not sure how to tag this ticket.  I'd like it to be routed to the OSG Software team.

Please provide a dependency map of the OSG rpms.

We are running into a situation here at FNAL where yum priorities don't seem to be taking effect.  I say that because much of the globus gsi libs are not being installed when we provision a machine.  We end up having to come back and manually install the rpms.  I know that in the past this has always been traced to installing EPEL rpms rather than the OSG rpms, however, everything that we can test points to the fact that priorities are indeed installed and set up prior to installing the OSG software.  We would like to be able to pull from OSG a list of the rpms that get installed as dependencies .  We can do this as a one-off by manually installing each component and scraping /var/log/messages, but that won't help the next time OSG introduces a new dependency.  We intend on creating a single yum repo with all the required rpms rather than relying on a mechanism that a) never played nice with Puppet, and b) doesn't seem to work well for us.

Specifically components that are affected:

HTCondor-CE
osg-wn-client
osg-wn-client-glexec

Thanks,

Anthony Tiradani
