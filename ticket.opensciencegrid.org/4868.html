Hi.  This ticket can be closed as the situation is now resolved.

We had 2 major issues with this.  Alain Roy suggested the following
would work, and he was correct:
pacman -get OSG:Globus-Condor-Setup

This successfully installed all the missing pieces and got our Condor
factory on the air following a globus-ws restart.

The second major issue is, apparently, a race condition between the
startup of mysql and globus-ws during a system boot.  As I understand
the document I found (below) mysql must be completely started before
globus-ws is started.  The symptom asserted itself as shown here (this
was actually a problem on gate01.aglt2.org, not gate02.grid.umich.edu,
but perhaps we were just lucky that it did not affect gate02):

[ball@gate02:~]$ globusrun-ws -submit -F gate01.aglt2.org:9443 -S -s -c
/bin/hostname
Delegating user credentials...Done.
Submitting job...Done.
Job ID: uuid:8ca895d2-2798-11dd-ae7b-0007e9334ae2
Termination time: 05/23/2008 00:46 GMT
Current job state: Active
Current job state: CleanUp-Hold
gate01.aglt2.org
Current job state: CleanUp

    Current job state: Failed                <----------- Note failure here
Destroying job...Done.
Cleaning up any delegated credentials...Done.
globusrun-ws: Job failed: Staging error for RSL element fileCleanUp.

If globus-ws is stopped then restarted, this problem goes away.  The
solution URL and details of this are shown here:
--------------------------
found this:
http://www.bestgrid.org/index.php/Vladimir%27s_grid_notes#RFT_staging_fails

Fixing startup order

To fix the startup order to avoid the above problem, run the following
commands:

sed '/^# chkconfig:/c # chkconfig: 345 97 09' --in-place=.ORI
/etc/rc.d/init.d/mysql
sed '/^# chkconfig:/c # chkconfig: 345 98 04' --in-place=.ORI
/etc/rc.d/init.d/globus-ws
chkconfig mysql reset
chkconfig globus-ws reset
-------------------------------------

Is this sufficient information to update the OSG TWiki?  If not, I will
gladly answer any questions you may have.

Thanks for all of your help.
bob
