    We have adjusted the original OSG production network services running on psmad.grid.iu.edu and psomd.grid.iu.edu from their "replicated" DNS alias setup.  Originally both were DNS round-robin pointers to perfsonar1.grid.iu.edu and perfsonar2.grid.iu.edu for the MaDDash (psmad) and OMD/check_mk (psomd) services.

For these services such a setup caused confusion and  problems.   Setting via the DNS VIP would only be applied to one of  the two running instances.  Future connections to the VIP would sometimes see the updated setting and sometimes not.

Around March 7 we decided to change the DNS aliases as follows:

psomd.grid.iu.edu would only point to perfsonar1.grid.iu.edu (This is the OMD/check_mk service)
psmad.grid.iu.edu would only point to perfsonar2.grid.iu.edu (This is  the MaDDash service)

We should now finalize this change as follows:

1) Verify that OMD/check_mk is setup correctly on perfsonar1.grid.iu.edu and that psomd.grid.iu.edu points only to it.
2) Verify that MaDDash is setup correctly on perfsonar2.grid.iu.edu and that psmad.grid.iu.edu points only to it.
3) Remove any of the MaDDash services and configuration from perfsonar1.grid.iu.edu  (free up CPU, memory and disk space)
4) Remove any of the OMD/check_mk services and configuration from perfsonar2.grid.iu.edu  (free up CPU, memory and disk space)

See our "Living Document" for OSG network services at https://docs.google.com/document/d/1l144BSo-88M0cLMMjKcKMIE-Q5s21X-w3lYl-0Pn_08/edit

This document should provide the details regarding OMD/check_mk and MaDDash.

5)  Update the operations guide for the OSG network services to match the new configuration of the services.

Thanks,

Shawn
