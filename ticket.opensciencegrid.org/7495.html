    It looks like a problem with the VDT-supplied sge.pm module.  Looking
in sge,pm, shell is forced to /bin/sh with "-S".

Fromn the script:

# Writing script header
#
$sge_job_script->print("#!/bin/sh\n");
$sge_job_script->print("# Grid Engine batch job script built by ");
$sge_job_script->print("Globus job manager\n");
$sge_job_script->print("\n");
$sge_job_script->print("#\$ -S /bin/sh\n");

If Dzero really requires the shell to be bash, they will have to
modify their submission scripts to use bash.  As a helpful step, I
have altered the SGE setup here to include bash as an allowed shell.

Can Joel try a test hob that sets the shell to bash to see if it works
for him?
