    Thanks for the access. I noticed a few things already without sudo access:

1) I couldn't generate a GLOW proxy because it couldn't contact the GLOW VO server. This looks like it's because you're on an older version of our vo-client, the newest version is vo-client-49-1.osg31.el5.noarch.

2) I was also able to srm-ping with a VOMS proxy and a 1024 bit proxy created by grid-proxy-init but had some issues when I didn't specify the number of bits with grid-proxy-init. With the most recent Java 7 update, there have been issues with proxies created with less than 1024 bits because of a change to the java security file:

/usr/lib/jvm/java-1.7.0-openjdk-1.7.0.45.x86_64/jre/lib/security/java.security

You will want to change the line (on your client machines and the BeStMan2 server):

jdk.certpath.disabledAlgorithms=MD2, RSA keySize < 1024
to:
jdk.certpath.disabledAlgorithms=MD2, RSA keySize < 512

Normally, we were seeing this problem only with srm-copy but it looks like srm-ping is affected too. I believe you should have gotten an e-mail about it last week if you're subscribed to the osg-sites mailing list. If you aren't, e-mail goc AT opensciencegrid.org, mention that you're an admin and your e-mail will be added to the appropriate lists.
