    Hi Trey,

IMO, it was a mistake to mark it as a config file in the first place. As the comment states, it's not supposed to be modified. I'll fix that in the spec file.

If you want to override CONDOR_IDS, I suggest creating a file called /etc/condor-cron/config.d/condor_ids_override. I'll add something to that effect into the comment.

(I don't want to change the spec file to not modify the condor_ids file, because that might break things for people that aren't using Puppet.)

    Jul 13, 2017 02:56 PM UTC by OSG-GOC 
    The docs for RSV [1] users mention ensuring the CONDOR_IDS value in /etc/condor-cron/config.d/condor_ids matches what's in /etc/passwd.  As such I manage this file with Puppet to ensure the UID/GID match what Puppet adds for the user/group. I just upgraded my CE and noticed the condor_ids file was replaced.  The value for CONDOR_IDS didn't change but the comment did since I put a comment at the top that Puppet manages the file.  I noticed with "rpm -qc condor-cron" that this file is flagged as a config, but the output from "rpm -q --scripts condor-cron" shows that this file is likely getting modified during preinstall script [2].  Maybe some change needs to be made to RPM spec to avoid modifying the file?  If the spec is publicly accessible maybe on github, I can make the change myself.

Thanks,
- Trey

# Need to put the uid/gid of cndrcron into the config file as CONDOR_IDS
CC_UID=`/usr/bin/id -u cndrcron`
CC_GID=`/usr/bin/id -g cndrcron`
echo "# This file is automatically generated.  Do not edit." > /etc/condor-cron/config.d/condor_ids
echo "CONDOR_IDS = $CC_UID.$CC_GID" >> /etc/condor-cron/config.d/condor_ids
