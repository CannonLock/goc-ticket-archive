    Oct 14, 2016 05:01 PM UTC by OSG-GOC 
    CILogon team made the following change request:

"We will be updating all of the various services provided by CILogon
since the OAuth For MyProxy (OA4MP) libraries upon which CILogon depends
have been updated. This includes osg.cilogon.org. In addition to better
MyProxy server fail-over handling, osg.cilogon.org now checks for and
removes duplicate SubjectAltNames (so that only 1 of each domain is in
the SAN). This probably doesn't affect anything with your web client
which may already do this check, but it's a nice check to have on the
server side.

There should be zero downtime since we will temporarily swap to the
secondary production server when updating the primary production server.
This is the current standard for all CILogon services. This also allows
us to swap back from the new code to the old code in case we discover a
bug when the new service is deployed to production. After a suitable
period of no bugs on the production server (typically a week), we will
then update the secondary production server with the new service code.

With your approval, we would like to deploy the updated code to
production on the morning of Wednesday October 19, 2016."

I see no problem with the requested change and approve it from the security standpoint. I also informed Cilogon team of my decision via email. If you have operational and other concerns please document on this ticket.
