Hello Neha,

Unfortunately we have decided to close the CompBioGrid VO.  Over the years, it has been mostly a placeholder created to meet unexpected needs from our users, which remained minimal, as growing local and cloud resources have been able to meet the demand.  We have turned off our CE and website over a year ago, and now we do not see a reason to maintain the VO anymore.

Therefore, the transition will not be necessary.  Please let us know how to proceed with shutting down the VO.  We expect to remain involved with the OSG in the future, but likely through the services of other VOs (such as SBGrid, for example).

Subject: Upcoming CompBioGrid VO transition to CILogon OSG CA

Hi,

CompBioGrid VO has been scheduled to transition to CILogon OSG CA on May 10th.
Based on contents of vomses file (vo-client package provided by OSG Software team),
VOMS server for CompBioGrid is currently using a certificate issued by Digicert CA
"CompBioGrid" "voms.cam.uchc.edu" "15001" "/DC=com/DC=DigiCert-Grid/O=Open Science Grid/OU=Services/CN=voms/voms.cam.uchc.edu" "CompBioGrid"

It will need to be switched to CILogon OSG CA. Please request a new host cert (for your VOMS server) now via http://oim-itb.grid.iu.edu/oim/certificate and send me the DN (I'll need to coordinate with GOC on changes to vo-client package)

In addition, there may be other services/software that VO uses that do get affected by this backend OSG CA change (from Digicert -> CILogon HSM) and you probably already have plans to ensure they keep working smoothly after the transition.

Two things that come to mind that should be done  -

1) Upload new CILogon DNs for all users that currently have Digicert CA issued certificates

This step involves running a script (provided by OSG Security) that will look up all existing (user/host/service) Digicert DNs in VOMS DB and add their CILogon OSG CA equivalent. Digicert DN entries will NOT be affected.

Sole purpose here is user convenience. Users will still need to request new CILogon OSG CA certificates
( when their current Digicert CA cert expires or is about to expire), but they won't have to request membership
to VO with their new DN (because it would have uploaded in VOMS DB already)

Please let me know if you plan to do this, so I can email you the script and instructions on how to use it.

2) Communicate with users/VO members and admins

They must be aware of when the transition is happening and how it affects them; what they need to do to prepare for it.

Users -- should know that current Digicert CA certs will continue to work until expiration;
the procedure to request/renew certs will remain same and it is only the backend CA that is changing;
post transition date, certs will be issued by CILogon OSG CA; new DNs will be already uploaded in VOMS DB for their convenience; anything else you think is necessary

Admins -- should install CILogon OSG CA cert on all relevant nodes ( upgrade osg-ca-certs to atleast 1.50-1) before May 10. vo-client package will also need to be upgraded (on all nodes that are used to generate voms credentials) to latest version ( that is released on May 10th)

If you have any questions/concerns, please let me know.

Thanks,
- Neha
