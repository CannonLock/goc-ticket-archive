    Hey Mike,

Personally, I would just clean out all the replicas from the companion DB via postgres, then run "pnfs register"
on each pool via the admin interface.

The Postgres command is one line of SQL and if you don't have a "cluster-fork" equivalent for dCache, let me
know.

Brian Bockelman wrote:
Hey Michael,

I would recommend that you use "rep rm" on the files via the admin
interface, not use "rm" on the RAID - this way, your pool stays consistent.

Brian

On Sep 22, 2008, at 3:11 PM, Michael Thomas wrote:

While doing some cleanup in the Caltech dCache system, I noticed that
there are a number of files where the file size on disk does not match
the file size in pnfs.  Sometimes the file size on disk is almost 50%
smaller than the file size in pnfs.  Due to our file replication, there
are often multiple copies of these files, and the other copies have the
correct size on disk.

What is the correct way to clean up the replicas with the incorrect
size?  Can I simply 'rm' the offending file on disk, or is there some
command I should use in the dCache admin interface to properly remove
such replicas?

--Mike
