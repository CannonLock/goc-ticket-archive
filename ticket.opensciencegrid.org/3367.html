Hi Fred,
I've slightly changed the pilot installation script to avoid performing
(by mistake) a df or du to the disk root. Unfortunately this is the
result of a site misconfiguration, since I never do a df or du of /. The
exact commands I do in the pilot job for the installations is df
$VO_ATLAS_SW_DIR or du -sk $VO_ATLAS_SW_DIR/*. Evidently if
$VO_ATLAS_SW_DIR is not set, I perform a full disk scan. This is now
corrected and if $VO_ATLAS_SW_DIR is not set I skip all the further
checks. Anyway, the sites should never leave $VO_ATLAS_SW_DIR unset if
they want to support the ATLAS VO: if they do not provide an area where
to install the software, we cannot deploy it. Could this be fixed,
please? If they cannot provide an area for the software installation
I'll exclude the sites from the deployment procedures, in order to
prevent further problems; just let me know how to proceed.
Thanks,

Alessandro
