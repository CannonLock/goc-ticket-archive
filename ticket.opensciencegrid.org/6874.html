    May 19, 2009 06:00 PM UTC by Kyle Gross 
    Patrick,

Have you tried restarting the services?

/etc/init.d/bdii-osg stop
/etc/init.d/bdii-wlcg-interop stop
/etc/init.d/cemon-consumer stop

/etc/init.d/cemon-consumer start
/etc/init.d/bdii-osg start
/etc/init.d/bdii-wlcg-interop start

Note: It is important to stop both the OSG and the WLCG-Interop BDIIs before starting
them back. The restart option for the two BDII services does not work reliably
because of the way BDII is programmed -- one stop kills all slapd daemons, and so
starting one, then stopping the other will kill the former's slapd daemons!

Then....

Restart Apache, execute /etc/init.d/httpd restart.

And there's also a note about the Translation Scripts....

The translation script(s) run(s) off root's crontab. To stop/start it, use crontab
[-l, -e] to list/edit those settings. By default, the scripts run once a minute.

# EDITOR=emacs
crontab -l
crontab -e

Let me know if any of this works.
