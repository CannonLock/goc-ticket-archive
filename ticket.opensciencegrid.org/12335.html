    This ticket can be closed.

After discussing this with Brian and Brad in a call today,  it was decided
that the maintenance of the Subcluster table is no longer required.

A little background on what it was used for in the event there are questions...

Long long time ago, when the Gratia-WLCG interface was pretty much in
its infancy, there was a need to calculate an SI2K/HS06 based  normalization factor (NF)
to be applied to CPU and Wall Gratia accounting data sent to WLCG for
MoU purposes.   The thought was to have a consistent way of determining these
factors within OSG.   By collecting cluster configuration data (eg. preocessors, speed, cores, etc)
from BDII and populating a Subcluster table in the Nebraska Gratia collector,
a normalization factor could be calculated for all OSG sites.

The original concept was good but the BDII data required some manual updates
by the site admins which was not always forthcoming or forthcoming in a timely manner.
The other issue was that the feed from BDII was unreliable or some sites did not
publish to BDII. Reliance on that source of data was iffy at best.   So Tier 1/2 sites started
calculating their own NFs.  Long term, the use of BDII is being deprecated.

Considering the last update was Feb 2012 and the need is no longer there,
there is no point expending any effort on its resurrection.
