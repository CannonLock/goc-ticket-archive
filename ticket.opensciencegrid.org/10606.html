On Thu, 23 Jun 2011, Open Science Grid FootPrints wrote:

> [Duplicate message snipped]
    Jun 23, 2011 01:17 PM UTC by Soichi Hayashi 
    For Chrome on Mac issue, you have opened another ticket (https://ticket.grid.iu.edu/goc/viewer?id=10569) and in it I have requested you to verify your CA certs which seems to be invalid. As far as I know, Chrome on Mac should be able to access twiki with or without a certificate.

For Safari on Mac issue, I believe the issue is caused by the latest changes made to Mac's keychain feature. Please open a ticket with Mac's support forum.

For Safari on Windows issue, this is clearly a browser issue where it doesn't handle "SSLVertifyClient optional" apache directive correctly. I have tried to workaround this but could not find solution that has a side effect on other browsers. If you are seeing a similar issue on your Mac, one thing you might want to do is to open a ticket with Safari support asking to allow Safari to continue with no certificate when you access https:// pages and click "Cancel" button on the certificate selection dialog.

Also, the reason I needed to forward http to https is that, twiki will not allow user to edit pages simply by clicking on Edit button if the user is currently in http:// session even if the user has valid certificate - among other authentication related reasons. Simply forwarding from http to https eliminated various other issues that majority of users (FF/Chrome on Mac/PC) would find very annoying.

I have spent many hours on getting twiki to work on as many browser, OS, and on various scenarios combined with peculiarity with how twiki handles user authentication, and I believe what we have currently is unfortunately the best I can offer. Please remember that x509 certificate is not yet very mature technology for many browsers & web servers and it is extremely difficult to *workaround* all bugs that exists (or what each of them call "standard".) I am willing to revisit this as more and more bugs are squished on each browsers and their behavior to become more uniform.

Thanks,
Soichi
