    Jun 17, 2014 02:55 AM UTC by Thomas Lee 
    The new drive is now installed, and the RAID controller is resilvering it.  This machine is older (it predates my hire) and has an unusual drive situation: two drives in a RAID-1 array and four drives in a RAID-5 array.  The failed drive is one of the RAID-1 array's two.  This is an older Dell model and also a RAID-1 array, so I'm not sure whether this behavior is standard for Dell hardware RAID-1 arrays of this era, but apparently the drive was inaccessible when running in degraded mode with one drive down.  It's going to take some time for it to rebuild, but when it's done, I hope the data is recovered and the machine can boot normally.  If it doesn't, I will have to reinstall — the data is backed up two different ways and in any case is stored on the array that didn't fail, so I'm not worried about having lost the data, but reinstalling the system will take time and effort.  I do have a script that can install and configure the software, based on the script that installed the secondary server (a VM), and I'm glad I wrote that last year, but it's never been tested on the primary server or on a physical machine — I've never had to rebuild it.  And I hope I don't have to rebuild it now.

It's taken about 15 minutes for the RAID controller to rebuild about 10% of the drive, so I have to estimate that it will take a little about 2 hours 15 minutes longer, and at that point we'll boot and see whether the RAID-1 array behaves normally and contains the data.  If it does, we'll be back to normal as of that moment.  If not, it will take more work.
    Jun 16, 2014 11:45 PM UTC by Thomas Lee 
    A drive has failed on our primary internal LDAP/DNS server, and as it turns out the VM that normally acts as a backup for this internal-only service was only partially configured (internal LDAP was working, internal DNS not so much).  As a result, inter-server SSH/SCP is slow (attempts to contact the primary server must failover to the backup) and DNS lookups of local-only hostnames were not working.  I have configured the internal DNS portion now, so internal DNS lookups should work again.

I have contacted SEI, and they are locating a replacement drive for this; unfortunately it is in Louisville.  They expect they will be able to have it in hand by 1 hour 45 minutes from now.
