    Mar 28, 2016 07:17 PM UTC 
    The way this is suppose to work is:

The probes run on both of the CE's, parsing the same PBS log (usually NFS mounted from the head node).  The certinfo files only exist on the CE that received the job.  They should be in some directory, /var/lib/gratia/data ?  This directory should be unique to each CE.  When the PBS probe runs, it parses the jobid from the PBS log file, then attempts to find a certinfo file from the gratia directory.  If it is not able to find a certinfo file (for example, if it's a locally submitted job or a grid job from the other CE), then it does not have a DN associated with the job.  The ProbeConfig should have SuppressNoDNRecords = "1" which will suppress all records which do not belong to that specific CE.

Debug checklist:
1. Check your ProbeConfig to confirm that SuppressNoDNRecords is set to "1"
2. Make sure that the /var/lib/gratia/data directory is not shared between the CE's.
