    Mar 14, 2011 08:05 PM UTC by Bobby Ball 
    At AGLT2 all appropriate values as  specified by these formulae are correct.

GlueSubClusterPhysicalCPUs = node_count * cpus_per_node
GlueSubClusterLogicalCPUs = node_count * cores_per_node

However, we should note that there may be an inconsistency in how we are
reporting.  For example, our [Subcluster AGLT2-Dell3] has the following
settings:
node_count = 81
cpu_model = Intel(R) Xeon(R) CPU           E5520
cpus_per_node = 2
cores_per_node = 12

So, this machine is using Hyper-Threaded (Logical) cores, but we allow
only 12 job slots for Condor on this machine, not the 16 that one would
expect from a dual-quad-core machine.  The HS06 measurement is based
upon 12 such threads running in this configuration, and thus our total
reported capacity is consistent.
