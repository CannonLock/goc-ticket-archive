    The Fermi/Remedy ticket has been updated.

Work Info Type* : General Information
Date+           :
Source          :
Summary*        : Probes are operating but with information loss.
Notes           : The probes are now operating once more. However, there is some information loss with the condor probe because you do not have PER_JOB_HISTORY_DIR set in condor_config as described in the OSG CE install guide. If you set this (and restart condor) then the classAd for just-completed jobs will be placed in $VDT_LOCATION/var/data, removing the need to invoke condor_history for each completed job. As things are right now however, the condor history has rotated away and many of the older managed fork jobs will therefore not be recorded.

Let me know if you have any trouble with the condor_config setting.

Thanks,
Chris.
