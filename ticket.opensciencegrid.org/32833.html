    Hi Mike,

The problem was not really the protections, but a missing entry in
/etc/group.

On m002, you have:

[root@m002 ~]# ls -al /bin/fusermount
-rwsr-xr-x 1 root 412 32368 May 11  2016 /bin/fusermount
[root@m002 ~]# grep cvmfs /etc/group
cvmfs:x:18:

Where you should have:

coltrane> ls -al /bin/fusermount
-rwsr-x--- 1 root fuse 32368 May 10  2016 /bin/fusermount
coltrane> grep cvmfs /etc/group
fuse:x:494:cvmfs
cvmfs:x:491:

Your fuse group id should be 412. it might a security issue allowing other
execute for /bin/fusermount.

    Feb 28, 2017 05:15 PM UTC by Dave Dykstra 
    The "external location for configuration files does not exist" errors are because config-osg.opensciencegrid.org failed to mount.  We experienced those secondary errors periodically at Fermilab, so I added a feature to cvmfs-2.3.3 which prevents those secondary errors from happening and causing a long term problem -- the feature just refuses to mount the next repository so it can be tried again later.  I suggest upgrading to it by doing 'yum install --enablerepo=osg=development osg-oasis'

As for your primary error on the config-osg repo, "failed to create Fuse channel", the only times I have seen that before was when /bin/fusermount was not executable by the cvmfs user for some reason, usually because of an  incorrect group or because the cvmfs user is not in the 'fuse' group. Could something like that be happening in your case?  Otherwise something else is causing it and I'd like to find out what it is. If you get a machine in a condition where that mount of config-osg failsreliably, let's investigate it more closely to find out why.  If you could give me a temporary login on such a machine I'd be happy to do the investigation, otherwise maybe you could investigate it further yourself.
