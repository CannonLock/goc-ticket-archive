    Jul 25, 2017 03:25 PM UTC by Derek Weitzel 
    Hi Bockjoo,

I think all of the CE's should have "SuppressNoDNRecords = "1"".  Otherwise we will have double counting, which we addressed maybe a year ago?  Yup, https://ticket.opensciencegrid.org/28852

So if you have SuppressNoDNRecords = "0", then you will get duplicate records with both CE's reporting the same jobs.

As for the slurm_meter stop reporting issue.  I don't see anything obvious in the logs.  So you just ran it manually and it started working?  Or did you change that SuppressNoDNRecords, then ran it, and suddenly you see jobs?  If you changed that configuration variable, then it's likely it just picked up any jobs it sees in the slurm log and sends them to GRACC.
