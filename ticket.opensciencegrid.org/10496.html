    Hello.

The message in DDM is not really error in DQ2. It is just a indication/warning
in DQ2 that the transfers seems to be going slow. Now. in US, we run the
throughput test (http://www.usatlas.bnl.gov/dq2/throughput) daily. If one looks
at it, you can easily see that the peroformance from LYON to US T2s have
deteriorated over last few days. (see attached plots. You can see the same
plots from the link above. Just chose the appropriate destination site.)
However, from LYON to BNL is not suffering as much (although it is also
slower.) One should recognize that the network path between LYON's to US T2s
(general IP) are different from that of LYON to BNL (OPN). So, I am guessing
that it is the combination of the bad network traffic as well as LYON's
storage. There is nothing wrong in AGLT2. One should contact LYON.
