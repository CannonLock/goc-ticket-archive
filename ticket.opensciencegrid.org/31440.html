    Hi,

We now understand what's happening. It turns out that the FNAL ITB frontend is not in fact configured identically to our production frontend when it comes to glexec, though I thought it was. It appears that the ITB jobs were not in fact running glexec at all, which explains why they worked.

The reason glexec was failing in production was due to the user proxy. The one submitted in the initial test was signed by CILogon Basic CA, which is by default not accepted by sites that have the EGI configuration, as JINR is. There is some code coming out soon to deal with this case but it's not in production yet. Sure enough, when I fired off some tests with a user proxy based the Nova production service cert, which is signed by the CILogon OSG CA, everything was fine. I'm confident we understand all the issues now.

Since the new HTCondorCE at JINR is about to start testing, I propose that we disable the CREAM CEs for NOvA in the production factory, and focus on getting the new HTCondorCE working.
