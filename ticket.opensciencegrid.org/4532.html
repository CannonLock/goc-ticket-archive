    Hi John,
You can see the Condor-Devel queue by sourcing
$VDT_LOCATION/vdt/etc/condor-devel-env.sh

Then run condor_q to see Condor-Devel's jobs.  The queue that you see
is not related to which condor_q binary you run, but rather to what
your CONDOR_CONFIG environment variable is set to.

Can you do the following prior to sourcing condor-devel-env.sh

echo $CONDOR_CONFIG
condor_q
. $VDT_LOCATION/vdt/etc/condor-devel-env.sh
echo $CONDOR_CONFIG
condor_q

This should show us both queues.  Nothing should run in your
production Condor, it should all be self contained in Condor-Devel.

Thanks,
Scot
