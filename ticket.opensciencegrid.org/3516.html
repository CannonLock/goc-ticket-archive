Subject: Request better diagnostics on DOEGrid Cert renewal import page.
Date-Sent: Friday, May 11, 2007 4:47 PM -0500

Hi,

Currently the DOEGrids procedure for renewing an existing cert involves clicking on a
JS button to import the new certificate directly into the browser. If this operation
fails for some reason such as the user's browser not being able to verify the full
signing chain, there is no opportunity in firefox to view the chain to see what it
was unhappy with -- were one to import the cert from a file on the desktop this
facility would be available.

I would be grateful if the new and renew certificate pages could provide a link to
the certificate for download to the desktop to allow the user to try a manual import
in the event of import failure. I would also request that the web pages suggest that
one might want to do this in the event of an import failure; simple instructions on
how to interrogate the certificate under IE and FF / TB; what to look for; and the
procedure for fixing the most common problems (ie provide the download links for the
ESNET and DOEGrids CA certs).

Finally: the CA cert links do not have the correct extension (.cer, I think) to allow
easy import into FF even though they have the correct format. This requires the user
to select "all files" to see it on the import page. This seems like a trivial task
for the tech-savvy, but yet another annoyance in a long line for a non-expert who
just wants to get his cert, sign his email and submit Grid jobs.

Thanks for your consideration and help,
Chris.
