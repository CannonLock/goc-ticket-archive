From Abhishek,

A good way is to create symbolic links such as:
[1]. On server nodes:

/etc/init.d/dcache-`hostname -s`-services -> /opt/d-cache/bin/dcache-core

Then one can simply start or stop all servers with a script as:

for REMOTE_HOST in A B C D E F; do ssh -i /var/ssh/id_rsa $REMOTE_HOST 'hostname; /etc/init.d/dcache-
`hostname -s`-services stop'

for REMOTE_HOST in F E D C B A; do ssh -i /var/ssh/id_rsa $REMOTE_HOST 'hostname; /etc/init.d/dcache-
`hostname -s`-services stART'

[2]. On pools:

/etc/init.d/dcache-pool -> /opt/d-cache/bin/dcache-pool

[3]. On PNFS server:

    
An additional script which wraps both dcache-core and pnfs server:

case "$1" in
start)
echo "....... Starting dCache PNFS services ....... "
/opt/pnfs/bin/pnfs start
/opt/d-cache/bin/dcache-core start
;;

stop)
echo "....... Stopping dCache PNFS services ....... "
/opt/d-cache/bin/dcache-core stop
/opt/pnfs/bin/pnfs stop
/opt/d-cache/log/back-up-logs.sh
;;

restart|reload)
$0 stop
$0 start
;;
*)
echo "Usage: $0 {start|stop|restart}"
exit 1
esac

In particular, the /opt/d-cache init scripts which are referred to
in the documentation don't start up unless pnfs has started
up and I don't see any script at all to start up pnfs.

There really should be an option in the vdt/DCache install to
make and install the init scripts, similar to what vdt-control --on
does for the Compute Element.

Steve Timm

In the vdt/dCache packaging, are there any init.d scripts available
so that dCache can start automatically after the machine is rebooted?
I'm looking through the docs and do not see any.

Steve Timm
