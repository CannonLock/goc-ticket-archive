    Nov 6, 2006 11:35 PM UTC by leighg 
    All, Leigh, Doug and I met to consider the incident report detailed below.
we will revisit this summary on Friday, after we have had some time to mull.

The summary is that:

1) It seems unlikely that this is a Grid Incident where grid methods and
tools are/will be exploited. The
vulnerability was sniffed OS passwords. The exploit was an OS exploit.

2) There is no evidence that this incident represents a systematic
compromise of OSG sites.  We are leaving this
matter to the site and site security.

3) We draw some lessons learned, which can guide the development of our
own security infrastructure:

LL1)  It is laudable that site examines local activities authenticated
by potentially compromised proxies.
We note that a  true attack against grid middle ware might harvest
proxies at one site and use them at
another, just as sniffed passwords are used to attack other machined.
We do not have the means to look
for suspicious proxy use grid-wide without great drama and  expense.

The use-case of watching a set of "suspicious proxies" at a cost
consistent with investigating a incident report
should be  passed onto those developing grid security tools.  Their
plans should be assessed to see if
this case is supported.

LL 2) This matter re-enforces the need for good security practices by
those offering services.  It seems unfortunate
that a key password was sniffed while cleaning up a machine that is
peripheral to OSG operations.   We have foreseen enumerating a more
complete set of diligences in the OSG Service AUP, and letting that

    drive our plans and procedures.

This incident report re-enforces the view that policy needs to be places
in the Service (or other apropos) AUP.

-- Don
