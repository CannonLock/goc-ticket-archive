    Daniel,

You can change the user (<username> below) that queries the slurm db by doing the following:

1) Create /etc/gip/gip.conf with the contents:
[gip]
gip_user=<username>
2) Changing the ownership of /etc/grid-security/http/httpkey.pem to <username> or giving <username> read access to that file

Give that a try and let us know how it turns out.

Cheers,
