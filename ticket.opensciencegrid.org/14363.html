    Apr 5, 2013 05:48 PM UTC by Suchandra Thapa 
    Hi Amy,

Just to shed some light on the changes that Elizabeth requested, the SGE code previously had some issues with how it configured the globus SGE jobmanager.  It turned out that the SGE jobmanager in GT5 was looking for the reporting log file instead of the directory that the reporting file is located in.  That's why the name of the setting was changed and why osg-configure now tries to verify that the log_file is set to an actual file instead of a directory.  Elizabeth is correct and making the changes that she suggested should resolve the issue.

Suchandra Thapa
