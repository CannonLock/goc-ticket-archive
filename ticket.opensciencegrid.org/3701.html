fix, and response to the ticket went into the ether. It does not help no one actually follows up on
the other side to check whether the problem was resolved and close the ticket from the other side.

From the UCSDT2 sysadmin:

/osgfs is a special area for autofs, it is not a directory. You can
never write into it, you should never ls it.

The correct directory is /osgfs/data which is what $OSG_DATA is assigned
to. You cannot assume that you will have any privileges whatsoever to
any directories above $OSG_DATA. If you want to access $OSG_DATA then
you need to go into /osgfs/data and create the directories there. The
permissions on $OSG_DATA are world writeable.

So to repeat

OSG_DATA = /osgfs/data not /osgfs
