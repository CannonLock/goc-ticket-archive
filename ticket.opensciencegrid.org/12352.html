    Jeff,

All gluex jobs since 6/1 have been using OSG_SQUID_LOCATION as a proxy for all http get requests from worker nodes, if it exists. If this env variable is not defined, the request goes directly to our server.

Even though the above is true, we are still seeing our http get's occasionally being blocked on the FNAL site.  This only applies to FNAL, all other sites are working as expected.  For example, on 7/9 around midday we saw a massive set of jobs rapidly cycling on the FNAL site because of failures of the squid proxy to be able to return results from our server.

-Richard Jones
