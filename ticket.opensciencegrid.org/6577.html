    Mar 25, 2009 07:52 PM UTC by Neha Sharma 
    Suchandra,

I think there could be some configuration issue with your dCache.
There are exactly the same requests to PURDUE, FNAL and UNL and they
have this test successful.  And their dCache version is the same too.
One thing that I noticed  strange from your log is that this file
request were handled as NEARLINE, CUSTODIAL". It should not be. I don't
think you have NEARLINE setup.
Client asked for REPLICA/ONLINE quality, and it might be that you don't
have this REPLICA/ONLINE but force all requests to be at
CUSTODIAL/NEARLINE which dCache may not support for srmCopy pulling from
remote storage.
So, my only recommendation is to check your configuration and see if
there is any CUSTODIAL and/or NEARLINE setup. You might also want to
compare your configuration with PURDUE, FNAL or UNL as their dCache
version is the same or very close.

-- Alex
