    Hello,

I don't think there's anything wrong with your scheduler.  Please check at :
http://dev-scan.grid.iu.edu/
and look for isuhep in the production listing.  There is a known bug with the current
production site_verify script that causes problems with the files in the
$GLOBUS_LOCATION/etc/grid-services directory.  The
url I've sent uses the latest site_verify and will go into production with OSG 0.6.
But that test still shows
the gsiftp test failing.  Have you added the GLOBUS_TCP_PORT_RANGE environment
variable to the
/etc/xinetd.d/gssftp file?  If you are behind a firewall that should be set in both
that file and in the globus-gatekeeper file.  Without it, it'll work sometimes and
not other times as the allocated ports fall within the range and outside of it, resp.
