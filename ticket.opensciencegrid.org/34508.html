    Aug 7, 2017 09:13 PM UTC by OSG-GOC 
    Hi,

first of all here at JINR Tier1 we see such problem with T2_US_Caltech only.
It seems to me, SRM at cit-se.ultralight.org:8443 return broken TURL with "embeded" IPv4 started with FFFF,
my guess, it is due IPv6 is enabled on SRM, so SRM will answer on ::FFFF:..., try:
netstat -anp | grep :8443
Could you try change a preference for resolver on SRM node via /etc/gai.conf?
One can prefer resolver for IPv4 over IPv6 bye changing priority like below:
{{{
cat /etc/gai.conf
label       ::1/128        0
label       ::/0           1
label       2002::/16      2
label       ::/96          3
label       ::ffff:0:0/96  4
precedence  ::1/128        50
precedence  ::/0           40
precedence  2002::/16      30
precedence  ::/96          20
precedence  ::ffff:0:0/96  100
}}}
Machine should be rebooted after.
