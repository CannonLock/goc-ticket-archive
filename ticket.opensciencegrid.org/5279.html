$h_tmp_metric{'globusJobManagerDir'} =
"$h_tmp_metric{'OSG_LOCATION'}/globus/lib/perl/Globus/GRAM/JobManager";

## For VO-SUPPORTED-PROBE

...and creating osg-rsv/bin/probes/worker-scripts/cacert-version.sh with
the following contents:

#!/bin/sh

rpm -q vdt-ca-certs --qf "%{description} version %{version}\n" | sed -ne
's/.*IGTF release \([0-9]*\.[0-9]*\).*version \([0-9]*\)/CA Certificates
v\2 (includes IGTF \1 CAs)/p'

The probe currently succeeds when run by hand.  Let's see what happens
when it is run by rsv...

Description of Problem:
org.osg.certificates.cacert-expiry fails to detect version of installed CA certificates.

On the CIT_CMS_T2 cluster, the CA certificates are managed using the vdt-ca-certs rpm package on the gatekeeper and all worker nodes, not using pacman or the vdt-update-certs software.  The cacert-expiry probe checks for the proper package version using the command 'vdt-version | grep IGTF'.  This command fails to report the actual installed version, since rpm updates of vdt-ca-certs do not get reflected in the output of the vdt-version command.
