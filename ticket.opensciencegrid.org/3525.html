> On May 15, 2007, at 9:33 AM, Murali Ramsunder wrote:
>
>> Hi,
>>
>> I would like to raise this Globus WS GRAM configuration issue. Recently
>> I reinstalled the OSG ITB software stack. Everything worked fine with
>> the install and the WS GRAM started up correctly. However, the server
>> was bound to the private IP (with the correct port) instead of the
>> public IP for the node. I looked through every config file in
>> $VDT_LOCATION and /etc/init.d/globus-ws but did not find anything
>> telling the server process to bind to either internal or external IP for
>> the host. Finally I figured out the problem - the private IP was listed
>> in /etc/hosts and the public IP wasn't. I got it listening on the public
>> IP by placing the public IP for the node ahead of the private IP in
>> /etc/hosts. Placing the private IP ahead of the public IP makes the
>> process bind to the private IP.
>>
>> This parameter should be a configurable value. If the VDT packaging
>> hasn't missed anything, I feel this should be taken to the Globus WS
>> GRAM developers.
>>
>> A second matter: Currently the "port" and "user" is specified in
>> /etc/init.d/globus-ws. Wouldn't it be better if WSC_USER, WSC_PORT, and
>> say a new parameter - WSC_IP or WSC_HOST, be specified in
>> $GLOBUS_LOCATION/etc/<some_config_file> rather than in
>> /etc/init.d/globus-ws. I personally would like all config info
>> consolidated in the $GLOBUS_LOCATION/etc/*
>>
>> Thanks,
>> Murali
>>
>
