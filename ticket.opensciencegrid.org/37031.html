    This may not be much help, but I'm able to run the analysis_test on our T3, so it *should* work.

My initial thought was perhaps your cvmfs area or cache has become corrupt.

On c0118.brazos.tamu.edu try a cvmfs_fsck on /var/lib/cvmfs/shared or just nuke the cache and try the process again manually.
Perhaps nuke the cvmfs cache on your squid as well?

To manually test a node:

(In a user directory on the worker do the following:
source /cvmfs/cms.cern.ch/cmsset_default.sh
mkdir test_it
cd test_it
export SCRAM_ARCH=slc6_amd64_gcc530
scramv1 project CMSSW CMSSW_9_0_0
cd CMSSW_9_0_0/test
vi analysis_test.py
(put in:
import FWCore.ParameterSet.Config as cms

process = cms.Process("CopyJob")
process.load("FWCore.MessageService.MessageLogger_cfi")
process.MessageLogger.cerr.FwkReport.reportEvery = 100

process.source = cms.Source("PoolSource",
fileNames = cms.untracked.vstring("file:/fdata/hepx/store/mc/SAM/GenericTTbar/AODSIM/CMSSW_9_0_0_90X_mcRun1_realistic_v4-v1/10000/887C13FB-8B31-E711-BCE7-0025905B85BA.root")
)

process.out = cms.OutputModule("PoolOutputModule",
fileName = cms.untracked.string('myout.root'),
#     fastCloning = cms.untracked.bool(False),
outputCommands = cms.untracked.vstring('drop *')
)
process.e = cms.EndPath(process.out)


    save.

cmsRun analysis_test.py
)

The cksum for the datafile is:
1703294014 1119717041 /mnt/hadoop/user/uscms01/pnfs/unl.edu/data4/cms/store/mc/SAM/GenericTTbar/AODSIM/CMSSW_9_0_0_90X_mcRun1_realistic_v4-v1/10000/887C13FB-8B31-E711-BCE7-0025905B85BA.root
Make sure that's not corrupt.
