    Mar 20, 2017 11:26 PM UTC by Steven Timm 
    Explanation of what is happening:

1) OSGEDU and CSIU voms servers were already turned off at IU some time ago, both those VO's should be wiped from the VO package altogether.

2) Right now OSG voms server answers on voms1.opensciencegrid.org and voms2.opensciencegrid.org.  (both of which

are at Fermilab).  OSG voms-admin server answers on both of those plus voms.opensciencegrid.org

3) New voms and voms-admin servers for

OSG VOMS are already up at

https://voms.grid.iu.edu:8443/voms/osg

with a database that's synced

4) On the transition day,

voms1.opensciencegrid.org, voms2.opensciencegrid.org, and voms.opensciencegrid.org will all be converted from the real IP addresses they are now, to aliases that all point to voms.grid.iu.edu

5) If it all works we should not have to change the

gums template at all, the voms-admin should just work on voms.opensciencegrid.org just as it is.

Ditto for the edg-mkgridmap file.

6) the /etc/grid-security/vomsdir/osg files need to add an .lsc file that has voms.grid.iu.edu in it because that is the cert that will now be signing the voms attributes.

Is that clear?
