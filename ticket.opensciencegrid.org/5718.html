scope -- related issues. Long'ish mail ahead with some questions for Kyle.

1) Resource_Contacts missing submitter contact

only active resource in such a state. I associated Steven Lee (the
primary admin/security contact) as the submitter. Case closed.

No Action Item.

2) Active resources without any required contacts

Then I looked for resources without a *primary* security and/or admin
contact -- found two: UIOWA-OSG-PROD (already *deactivated resource*)
and FNAL_FERMIGRID_TEST (was *inactive* but not disabled) -- I

Timm et al ask about it we can revisit but it's been inactive for a long
time.

Consquently, all active (i.e active, and not disabled) resources have
the minimum required contacts now. Indeed, there are some inactive
resources without any contacts or some missing contacts but we must
catch those at activation time, if applicable.

No Action Item.

*QUESTIONS TO KYLE
*

1) Max number of contacts (of a certsin rank/type) -- new bug found: I
notice that the web interface is allowing more than 2 admin contacts for

2 but the above resource has 3
b) the number of allowed secondary contacts is 1 but the above

*ACTION ITEM*: While I am ok with discussing increasing the number of

and I will be happy to test.

2) Is there a ticket for the I3_Support SC registration made on Oct 1
2008? I don't see it on Footprints but perhaps I didn't search for the
right keywords (I tried I3_Support and IceCube).

*ACTION ITEM*: If ticket creation for SC registration is not in place,
then put it in place immediately.

1) To be able to generate the above type of information in the form

admin contact -- for example, if we mandate that all resources must have
