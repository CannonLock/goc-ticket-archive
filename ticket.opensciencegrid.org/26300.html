    Mick,

I asked Edgar and Jeff about what may have caused the permissions to change on the frontend. The only thing they could think of is if the frontend was not reconfigured properly since it's an rpm install now. To properly reconfigure the rpm frontends, you should always run the following sequence as root:

service gwms-frontend stop
service gwms-frontend reconfig (or service gwms-frontend upgrade, which is more thorough)
service gwms-frontend start

You might want to try this when you get the chance. It should fix any permissions you may have missed in your manual fix.
