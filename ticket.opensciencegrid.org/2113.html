Alan responded:

This is not the correct behavior. It prevents the client being
installed cluster-wide for all users, for example.

It would be better for the pacman installation to avoid setting the
SRM_CONFIG variable, as the srmcp script itself handles things
correctly if this environmental variable is not set. This would allow
the client installation to be used by all users on a system, rather
than having to have each user install a separate copy of the client.

    
In my tests, if installed by root into a common area, the SMR_CONFIG
gets set to /srmclient/etc/config.xml

Thanks!

Alan

From Alan Sill:

This has the following problem:

Installing OSG:srm-cp on top of OSG:client and setting it up via the
setup.sh leaves the SRM_CONFIG variable defined explicitly to be

SRM_CONFIG = /srmconfig/etc/config.xml

One has to unset this environmental variable to use srmcp so installed.

I am chasing a further inconsistency -- it works as above when
installed on resources that already have /etc/grid-security areas and
host certs (e.g. machines that already run other services), but srmcp
fails to run with a GSI-related error (bad CA) when installed on
resources that do not already have host certificates.

Is a host cert required for a machine that just wants to use the client
to copy files with srmcp?

Note that the installation question with regard to certificate
locations was chosen as "local" in each of the above test cases, and
that the edg-crl-upgraded is running in each case.

Alan

This has the following problem:

setup.sh leaves the SRM_CONFIG variable defined explicitly to be

SRM_CONFIG = /srmconfig/etc/config.xml

One has to unset this environmental variable to use srmcp so installed.

I am chasing a further inconsistency -- it works as above when
installed on resources that already have /etc/grid-security areas  and
host certs (e.g. machines that already run other services), but srmcp
fails to run with a GSI-related error (bad CA) when installed on
resources that do not already have host certificates.

Is a host cert required for a machine that just wants to use the client
to copy files with srmcp?

Note that the installation question with regard to certificate
locations was chosen as "local" in each of the above test cases, and
that the edg-crl-upgraded is running in each case.
