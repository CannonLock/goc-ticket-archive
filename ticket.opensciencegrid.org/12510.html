    Aug 16, 2012 02:29 PM UTC by OSG-GOC 
    It appears this problem is due to several factors:

1) The type of jobs currently running in the cluster do a lot of staging, and the files are predominately on a  storage node that is heavily loaded;
2) There seems be a fair amount of central deletions occurring;
3) An issue with how checksums are handled is adding to the load on the storage node.

In the short term we've reduced the number of running jobs to ease contention in the storage system.  Longer term modifications to the local site mover will be made.  Note also the site will be off-line sometime in the next few days for a migration of the local LFC to one at BNL.

We'll set this ticket to 'on hold' for now and update it later once the above fixes have been implemented.
