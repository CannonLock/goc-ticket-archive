    Dec 5, 2013 01:39 AM UTC by Shawn McKee 
    Hi  Lucy,

That's  great.    Glad you are  able  to  help  out  with  this.

The  updating you mentioned  is how  you configure  your perfSONAR-PS
toolkit  instance  to  *use*  the  mesh-config.   What  we  need is
someone  to  update/create the  USCMS  mesh  configuration itself  (the
tests-us-cms.json  file  you  mentioned.  Simone has  provided   some
good  info about the  update  process  which  I  include  below.
*
""I try to summarize here the steps to follow:

* Join the Task Force mailing list wlcg-ops-coord-tf-perfsonar from
https://e-groups.cern.ch/e-groups/EgroupsSearch.do . This is where we
discuss changes, call meetings, etc ...

* The central configuration is handled through files in CERN AFS,
exposed through HTTP from the URLs you where pointing out. You need rw
access to the AFS location where the config files are. If you send me
your CERN AFS account I can grant you access.

* The central configuration is under
/afs/cern.ch/project/gd/wlcg-ops/perfsonar/conf/central/ . There is is
a directory for the definition of the sites and a directory for the
definition of the tests. I created for you the  two files containing
the sites definitions for the RU cloud and the test definitions for
the RU cloud. Those files are

/afs/cern.ch/project/gd/wlcg-ops/perfsonar
/conf/central/sitedefs/configs/sites-ru-all.conf
/afs/cern.ch/project/gd/wlcg-ops/perfsonar
/conf/central/testdefs/configs/tests-ru-all.conf

To include a new site the process is as follows:


    1) Include the new site in
/afs/cern.ch/project/gd/wlcg-ops/perfsonar
/conf/central/sitedefs/configs/sites-ru-all.conf.
You can clone one of the existing blocks there (I put protvino and
Dubna, which are the Ru sites I found in GOC), it should be simple, if
you have a problem let me know.

2) Build the JSON from the .conf file. For this you need a machine
with perfSONAR installed. You should copy the .conf there and run the
build_json command, like

/opt/perfsonar_ps/mesh_config/bin/build_json -input <file>.conf
-output <file>.json

The corresponding JSON needs to be copied back in AFS under
/afs/cern.ch/project/gd/wlcg-ops/perfsonar/conf/central/sitedefs/jsons
. We are trying to provide the build_json command directly from AFS
but this is not there, so you need to go through the perfSONAR host.

3) Change the test definitions to include the new site. You need to
change 2 files:

/afs/cern.ch/project/gd/wlcg-ops/perfsonar
/conf/central/testdefs/configs/tests-ru-all.conf
/afs/cern.ch/project/gd/wlcg-ops/perfsonar
/conf/central/testdefs/configs/tests-wlcg-all.conf

You can see where I added your two perfSONAR hosts and do the same for
new sites. Be carefull that you might need to add the same host twice
in the same file.

4) Build the JSONs from the test definition files. Same procedure as
for point 2) but this time you should place the new JSON under
/afs/cern.ch/project/gd/wlcg-ops/perfsonar/conf/central/testdefs/jsons

Notice that the testdefs need to access the sitedefs JSONS through
HTTP. So the order of the actions is important, you can not build the
test JSONs before building the site JSONs and storing them in AFS.

5) At this point you should find the two meshes in the WEB URLs

* https://grid-deployment.web.cern.ch/grid-deployment/wlcg-ops/perfsonar
/conf/central/testdefs/jsons/tests-wlcg-all.json

* https://grid-deployment.web.cern.ch/grid-deployment/wlcg-ops/perfsonar
/conf/central/testdefs/jsons/tests-ru-all.json

Your perfSONAR machine should import *both* as explained in the
perfSONAR deployment twiki
https://twiki.cern.ch/twiki/bin/view/LCG/PerfsonarDeployment

This is basically it. If something does not work you can ask for help
sending an email to the mailing list
wlcg-ops-coord-tf-perfsonar@....."

It  takes a  bit  of  careful  work to  properly  create  and  update  the
mesh-config files.   You need to have all the  information about the USCMS
perfSONAR-PS toolkit  instances  and  create (or  update)  the  files
mentioned in  Simone's  instructions  above.

Feel  free  to send  email  and  ask  questions if  needed.      Thanks,

Shawn
