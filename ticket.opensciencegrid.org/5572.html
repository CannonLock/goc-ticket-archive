    Sep 15, 2008 05:26 PM UTC by echism 
    This should be directed to Suchandra Thapa who is the
maintainer of configure-osg.py:

In the case of config.ini where
simplified_srm = False
vo_dir = VONAME

The configure-osg.py substitutes the VONAME
incorrectly.

Instead of going through osg-user-vo-map.txt and taking
the second field, it is taking the first.
Thus, in any case where the username is not the same
as the VO name, we end up getting the wrong path to storage
declared in the GIP.

i.e.  "cmsfrontier" instead of "uscms", "samgrid" instead of "dzero",
"fnalgrid" instead of "fermilab", "ligo" instead of "LIGO", and
the list goes on.  This is a bug.  The configure-osg.py should
be using the same method to detect VO's as the GIP is using.
Contact Brian Bockelman for details.

It is possible to work around this bug by specifying manually
what the storage path should be for the 15-odd VO's where this
is the case, and that is what we have been doing, but
it needs to be fixed.

Steve Timm
