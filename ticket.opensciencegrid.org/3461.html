    May 3, 2007 07:29 PM UTC by Tim Silvers 
    Horst,

I ran a manual site_verify for the new ITB host (osgitb1.nhn.ou.edu) and it is failing Hello World
with error code 22. That suggests that the MIS local user mapping doesn't exist, or the permissions
are not set so that the directory is writable, with which you probably are familiar.

As for disabling services, you're right. Then when you go to shutdown services you can also do that
using vdt-control rather than /etc/init.d/service stop: vdt-control --off MLD How you do it is totally
up to you. :-)

You'll need to do a 'global vdt-control --off' to get everything out of the /etc/init.d, etc. Then
disable the ones you don't want running (tomcat and apache are only used for the cemon, I think).
Then restart with vdt-control --on.

Let us know how it goes.
