    HI Alan/Naresh,

I think setting "advertise_gsiftp = True" is not actually advertising
gsiftp support for your srm (that is actually taken as given). It is
rather intended to advertise a separate disk based SE primarily for
sites that do not have a separate storage element or want to advertise
access to $OSG_DATA, through a gsiftp server
https://twiki.grid.iu.edu/bin/view/Main/OSGConfigurationParameters#GIP

Once you set advertise_gsiftp = false, or OSG_GIP_DISK=0 and rerun the
configuration, you should check
$VDT_LOCATION/gip/etc/osg-info-gip-config/osg-info-static-se.conf. This
file should have only one entry for dn:
GlueSEUniqueID=sigmorgh.hpcc.ttu.edu, instead of the two that currently
exists.

Thanks
Anand

    Jul 29, 2008 09:00 PM UTC by Gabriele Garzoglio 
    mhmm... ress-ops? You mean ress-support?

We have followed this issue so far. The problem was tracked to a
mis-configuration of GIP: CE is bound twice to the same SE. This
generates twice the number of classads from TTU and causes the
out-of-memory error in CEMon. Doubling the memory allocated for CEMon
mitigates the problem AND ReSS automatically discards duplicated
classads. To fix the problem, though, GIP configuration should be changed.

Can you add osg-gip to the ticket?

Thanks
Gabriele

    Hi Alan,

Great! Atleast the site is now reporting something :)

I looked at the gip SE entries and there is some configuration
issue/bug. Specifically there are two entries with the same GlueSEUniqueID

dn: GlueSEUniqueID=sigmorgh.hpcc.ttu.edu,Mds-Vo-name=local,o=grid

one entry is for
GlueSEName: TTU-SIGMORGH:srm_v2
(the bestman SE), while the other is for
GlueSEName: TTU-ANTAEUS:disk

I believe the entry for the later should be the ce-hostname. These
entries can be found in
gip/etc/osg-info-gip-config/osg-info-static-se.conf. This file does get
created by configure_gip. So you can either set OSG_GIP_DISK=0 in
gip-attributes.conf, if you do not want your local disk to be published
as an SE or set OSG_GIP_SE_DISK variable correctly. You will have to
rerun configure_gip after it.

I think the corresponding variables in config.ini are
advertise_gsiftp = false
gsiftp_host = xxx.edu
if you want try that route.

Let me know if this helps.
