    Hi,

A. srm-ping
Where is your CA dir located? If it is not located( or linked)  in the standard place <VDT_LOCATION>/globus/TRUSTED_CA you have to do the following:

1. if you are using latest client package
export  X509_CERT_DIR=<your trusted ca dir>

2. if you are using older client package create a symbolic link from /usr/local/vdt_bestman/globus/TRUSTED_CA
to <your trusted ca dir>

If this doesn't help let me know.

B. Look at

http://www.usatlas.bnl.gov/twiki/bin/view/Admins/BestMan#Space_tokens_for_ATLAS

for space tokens example. I will add some examples to doc soon.

I think if you are not using tokens and not using multiple partitions you do not need to specify cache.

Tanya
