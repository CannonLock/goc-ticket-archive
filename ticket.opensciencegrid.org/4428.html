Marco Mambelli wrote:
Charles,
a clarification about the one cert, multiple node feature:
- a certificate for basename.domain (even if you have no host really
called this way) will work on both basename-XXX.domain and
basename-YYY.domain
- a certificate for basename-XXX.domain will not work on
basename-YYY.domain, it will fail
What you described could be correct or not (Globus-wise), depending on
the DN of the certificate you used.

Cheers,
Marco

On Sat, 5 Jan 2008, Charles Waldman wrote:

Brian Bockelman writes:

Do you have CA certificates installed on the pool node?  dCache has
the directory /etc/grid-security/certificates hardcoded, so it's not
good enough to just have OSG client sitting on disk.

Brian (and others) - after several emails on Friday afternoon (from
Ted and Neha) which indicated that perhaps the problem was due to
no host certs on the pools (this is something I had some doubt about,
I have heard from some sources that pool nodes need host certs and
from others that they do not) - I went ahead and retried the test

    case after putting a host cert on each of the pool nodes.  I saw
exactly the same results.  However, the host certs I used were all
copied over from one of the head nodes - I was using a feature of
Globus that I have heard about (this was discussed at the OSG site
admins meeting at Fermilab last month) where supposedly a host cert
for basename-XXX.domain can be used on hosts basename-YYY.domain -
in particular, I copied the host cert from uct2-grid7.uchicago.edu
to all the uct2-c0XX nodes and the host cert from iut2-grid7.iu.edu
to all the iut2-c0XX nodes.   After doing this I repeated the
test-case and got exactly the same results.

Next step I suppose is to put honest-to-God host certs on the worker
nodes and retry the test.  I am pursuing this currently, will report
back.

Here's a good test - copy your personal cert to the pool which was
doing the download for SRM and run srmcp.  The RemoteGsiftp mover code
has a lot of overlap with srmcp, so they *ought* to either both fail
or both succeed.

Thanks for the suggestion Brian, I will try this as well.
