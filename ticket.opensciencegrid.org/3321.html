which is the problem, but maybe something to do with the filesystem setup.

We have job which creates a new directory in OSG_DATA with 'mkdir -p'. The -p should
mean that it shouldn't complain about target or parent directories existing, but I get:

mkdir -p /opt/osg-temp/engage/workdir

The target directory exists:

$ globus-job-run nest.phys.uwm.edu /bin/ls -l /opt/osg-temp/

drwxr-xr-x  2 engage    grid3       80 Apr  1 10:11 engage

We will try to work around this issue, but as mentioned before, we are worried about
the problem being bigger than just with mkdir.
