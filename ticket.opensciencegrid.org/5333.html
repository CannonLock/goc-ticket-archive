    Aug 13, 2008 05:15 PM UTC by Arvind Gopu 
    Hi Keith, Steve, et al..

With today's OIM updates, now you can register such resources, while
choosing "Misc services -> Hidden CE/SE" as the service. Look for the
misc service option in the drop down menu -- which defaults to Grid
Services (ie CE/SE). I've done the needful for your first resource
FNAL_CDFOSG_1. You can register the others as and when you need to.

Note that on a public view [for eg.
https://oim.grid.iu.edu/pub/resource/show.php], these service end points
won't show up as an CE or an SE ... they'll show up as a hidden CE/SE
... Chris and the Gratia team need to keep that in mind. We could
possibly consider tweaking the mechanism further if there's a dire need
for it. But we don't clearly understand how and why Gratia requires what
it  does ... and we don't want to invest too much more time to
guess-design an infrastructure purely to fit a badly designed software's
needs.

Cheers,
Arvind

    Aug 1, 2008 03:30 PM UTC by Keith Chadwick 
    Tim,

As I discussed on the telephone with Arvind this morning, this
compute element is the first one of several internal FermiGrid
compute elements that FermiGrid will be registering:

FNAL_CDFOSG_1, FNAL_CDFOSG_2, FNAL_CDFOSG_3, FNAL_CDFOSG_4
FNAL_DZEROOSG_1, FNAL_DZEROOSG_2
FNAL_GPGRID_1, FNAL_GPGRID_2

These internal FermiGrid compute elements will not directly accept
jobs from external (to Fermilab) sites (so direct MIS submissions
to them will not succeed).

Access to these internal FermiGrid compute elements is via the Grid
job forwarding mechanisms that run on the FNAL_FERMIGRID compute
element.  Documentation regarding this job forwarding mechanism
is available here:

http://fermigrid.fnal.gov/matchmaking.html

Since these internal compute elements will not directly accept jobs
from external sources, you might as why we are bothering to register
them?

The answer is that we are (starting to) registering these compute
elements in order to support the (future) needs of Gratia to lookup
the compute element in OIM to determine the resource "owner" VO of
that compute element.

This "ownership" information is used in nightly Gratia reports
(I have appended an example below).

At the moment, based on my conversation with Arvind, OIM does not
appear to support this form of registration, but my feeling is

    that Avrind will be working towards it in the (near?) future.

Also, I made a typo when I entered the data - the name should be:

FNAL_CDFOSG_1 (not FNAL_FCDFOSG_1).

-Keith.
