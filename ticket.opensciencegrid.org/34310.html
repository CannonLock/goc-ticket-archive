    Hi again,
Kind of we figured out the problem or set of problems. The sequence of events are like this. About two weeks ago a node with public ip (sandy-1-1) broke. since this node was acting as nat for a second node (compute-1-1) jobs landing there were giving the original problem. During the holiday the power down and later up cause sandy-1-1 to go up for short period of time. Since this is part of hadoop the down part always triggered hadoop "data recovery replication" looks like this last step few day ago caused to trigger some issue in our switch, never understood in the past, like one node going down affecting a third node to loose network connectivity. In the past recovering the down node caused to recover network in a third node. Turn out that this last part is a bug in the switch firmware Dell PowerConnect 2648 related with vlans in ports.

Yesterday we updated firmware, and switch does not block ports anymore, at least don't see the problem again.
Today I turned on our ce, and started receiving jobs. All seem working OK.

Let me know, if the problem persist. Keep this ticket open for a while, to make sure jobs are back to normal.
Thanks,
Eduardo
