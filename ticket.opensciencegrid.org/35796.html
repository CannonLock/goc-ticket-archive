    Jan 12, 2018 11:12 PM UTC by OSG-GOC 
    I would like to open a ticket regarding the following issue on the Tier2 LSST cluster at Bellarmine University (tier2-ogs1.bellarimine.edu).

We have an issue with Condor-CE that we are unable to resolve.  It looks like a routing issue.  So we need help from a Condor expert.   Here is the problem-

telnet tier2-osg1.bellarmine.edu 9619 connects just fine.

We’ve tested condor_ping together with tcpdump and it was not showing TCP packages from response, only request ones were displayed.

We just ran a quick local condor test job on tier2-osg1, and that ran fine, so I'm pretty sure there's nothing wrong with that.

When we ran a condor_ce_trace against the Condor-CE, that hung, so there's most definitely a problem with that.

----
$ condor_ce_trace tier2-osg1.bellarmine.edu
Testing HTCondor-CE collector connectivity.
----
It just hangs instead of keep going.

So this is most definitely a Condor-CE issue, and most likely routing related, since Condor-CE does respond to telnet tier2-osg1.bellarmine.edu 9619 with an initial connection, so it's not a firewall or other
outside or network block problem.

Condor-CE isn't properly forwarding the job to the local Condor scheduler. So it must be a Condor-CE routing issue.  So we need help to track down where this routing issue is.

Thanks
-Akhtar Mahmood
Bellarmine University
