Justin, I have another idea, though I can't quite explain why a "yum update" caused a failure.

Old versions of Bestman only work with an older style of proxy. By default:

- grid-proxy-init makes a new style of proxy that doesn't work with older versions of Bestman.
- voms-proxy-init makes an old style of proxy that does work with older versions of Bestman.

Both programs can be convinced to make the opposite through command-line arguments.

This explains the behavior you saw. You can tell RSV to make an old-style proxy and then it will work with grid-proxy-init instead of voms-proxy-init. In the osg-configure configuration file for RSV, you can set::

legacy_proxy = True

More information at:
https://twiki.grid.iu.edu/bin/view/Documentation/Release3/IniConfigurationOptions#RSV

I hope this helps!

I recently rewrote the nagios-consumer to work properly with RSV.  It was released in 3.7.0.  If you are having problems you should look in /var/log/rsv/consumers.  There are two files of note:
