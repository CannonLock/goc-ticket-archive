    Aug 23, 2017 09:14 PM UTC by Jaime Frey 
    This is a bug in the python bindings for ClassAds. They are aware of the differences between the old and new ClassAds formats, but don't properly handle the string escaping rules of the old ClassAds format. They end up using the escaping rules of new ClassAds. This will result in incorrect values for any ClassAd string literals that contain a backslash. If the backslash is the final character in the string literal, you'll get a parse error.

There's an HTCondor ticket with details here: https://htcondor-wiki.cs.wisc.edu/index.cgi/tktview?tn=6382
The bug that Carl mentioned previously is unrelated: https://htcondor-wiki.cs.wisc.edu/index.cgi/tktview?tn=6384
