    Hi,

Tracing the bug, the process stalls executing the test ./csh.test.
The strace reveals: fcntl(6, F_SETLKW, {type=F_RDLCK, whence=SEEK_SET, start=0, len=0}

Goggling the problem, it seems a concurrency bug in the tcsh, which requires a header line in .history file, but may leave the file empty then tries to rewrites the .history file.
I think this version of tcsh is not safe to be executed in a cluster/NFS (without nfslock) environment.

As a workaround, I've created a .history with root permissions that should avoid this issue.

Let me know if the problem persists.
Best regards,
Gabriel
