    Jan 4, 2011 12:44 AM UTC by Soichi Hayashi 
    Marco,

Hi. I was gone most of the December, and I am finally back! I apologize for the TWiki annoyance. Handling of x509 certificates by web browsers aren't very matured and it is always a struggle to get this working correctly.

The way it works now is that, for view-only access, user should access twiki via http:// URLs (like.. http://twiki.grid.iu.edu/bin/view). They can view any pages on twiki without being prompted of any password / certificate.

When they decide to edit a page, they will click on "Edit" button, and the user will be first redirected to https:// version of the same page, then Apache will request x509 certificate. If user doesn't provide the certificate, it will display user/pass authentication dialog (which would never authenticate anyone). Due to how x509 plugin on Twiki is implemented, I haven't found a way to suppress this user/pass dialog.

> I did not authorize Firefox to send automatically my certificate each time the site is asking for it,
Is there a particular reason why you can't make it automatically select your certificate?

> .. have the login session lasting more than a couple of minutes (not to have to resend the certificate all the times)
Yes, we can increase this. It is currently set to 300 seconds (SSLSessionCacheTimeout  300) Do you have a particular value that you'd like me to set? If you can set your certificate to automatically select cert, then this value should be irrelevant since your browser will automatically re-send the certificate if necessary.

Thanks,
Soichi
