    > [root@gc ~]# cat /opt/ce-1.2/apache/logs/mod_jk.log
> [Tue Sep 06 12:44:28.571 2011] [13324:3086403280] [error] ajp_validate::jk_ajp_common.c (2259): can't resolve tomcat address localhost
> [Tue Sep 06 12:44:28.572 2011] [13324:3086403280] [error] ajp_validate::jk_ajp_common.c (2262): invalid host and port localhost 8943

What I see here is that Apache is trying to contact Tomcat via a network connect. It's connecting to "localhost", and it can't resolve localhost to an IP address. This means your network setup is not right, and you need to fix it.

From the commandline, you should be able to do the following:

% host gc.ucatolica.edu.co
SOME-IP-ADDRESS

% host SOME-IP-ADDRESS
gc.ucatolica.edu.co

% host localhost
127.0.0.1

Do all of these work? How have you set up host/IP resolution? What's in your /etc/hosts file?
