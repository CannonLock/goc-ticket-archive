Terrence Martin wrote:
message may not tell you what you need to know in the event of a HD error.

Run this command

dmesg

Look for any HD related errors, like sector errors or block remappings. dmesg dumps the internal kernel log. To
get this to a file on disk add

kern.*                                                  /var/log/kern

to the /etc/syslog.conf file

The line is probably commented out and points to the console. I find it more useful to send it to disk. If you want
to you can also send it to a central log server.

Terrence

On Wed, 7 May 2008, Jon Bakken wrote:

I am pretty sure that your problem is due to disk i/o errors - that's  what
usually causes the "pool disabled due to problems in the repository'
error message.    These errors can be temporary, and when you go
to look at the problem, it's not there anymore.

During our disk evaluations, many many vendors failed to sustain  dcache's
multiple simultaneous reading/writing threads for a pool, and this was  the error
that I would see.   You never see this with normal disk tests (iozone
for example).

Typically, these i/o errors are logged in /var/log/messages.   Check  there.
