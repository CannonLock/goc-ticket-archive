    Oct 9, 2006 09:12 PM UTC by leighg 
    Shaowen update us with this details with the GROW VO..

to be forwarded to support centers with the Please support Grow message..

Eric mentioned to me that you requested concise information about GROW OSG
applications during today's operation teleconference. I've come up with a
short description as follows.

-----------------------
Currently, GROW has two major applications that have been running on OSG
production infrastructure. One application is called spatial interpolation
that is widely used in geographic information systems to create 3D
representations from 2D input. GIScience literature provides a number of
algorithms for spatial interpolation. These methods become computationally
intensive when large datasets are treated. GROW researchers develop advanced
spatial interpolation algorithms using OSG. The other application is for
reliability testing of mechanical systems in Mechanical Engineering. This
mechanical engineering application sweeps a large number of parameters to
discover the optimal configurations that would make mechanical systems
reliable.
-----------------------

Should there be additional information needed, please let me know.

This has been done for a couple of VO's - NanoHub and Ligo - mostly
through the Monday Ops meetings and emails to the various resources'
support centers.  What we've been trying to do is to have sites honor
their grid3-vo-user-map.txt file since it is the only way we currently
have of tracking VO usability on a site.  Basically, if they say they
support a VO then they should make it work or not say they support it.
Easy enough in concept but difficult to keep up with, particularly with
sites running GUMS since that is a completely additional step.

Another source of problems

I'm not sure I understand this - all of our monitors, including the one
that monitors VOMS servers, use grid-proxy-init.  I had always assumed
that voms-proxy-init was something one uses on a VOMS server but I don't
really know.

We have similar ranges set on our boxes.  I know our environment works
as I can submit to remote sites (i.e. UERJ) without problem.


    This is odd.  STAR-SAO PAULO says it supports GROW but not UERJ.  I'm
assuming that you work more directly with them but that is not very
efficient for the 32 sites that say they support your VO.  What we did
with the other VOs is confirm problems with the site using the information
from the scanner then grouped the sites by support center and emailed
those support centers.  Then follow up as necessary in the Monday Ops
meeting.  Its time consuming but it did produce results.
