no, I don't think we've found a victim yet. =)

I've found another issue we could possibly address at the same time, though --
in addition to the one that Jaime mentioned the other day:

----
There's another part to the lsf.pm fix that I haven't seen in the
ticket. The module assumes that when bjobs reports that it has no
knowledge of a job ("Job <123> is not found"), it will exit with exit
status 255. Parag and I observed it exiting with exit status 0 in this
case. So lsf.pm needs to examine the output of the tool, not just the
exit status.
----

I noticed yesterday when I played around with it in order to get MPI jobs
working on the OSCER Sooner cluster that lsf.pm also needs to add a line like

#BSUB -a mvapich

to the submit script, otherwise the job will fail with some error about
PJL_something not defined.

Currently, there's no section in lsf.pm which addresses "#BSUB -a".

The question is just, how do we discover the possible settings of that?
For now I just hardwired it into our lsf.pm with

print JOB '#BSUB -a mvapich', "\n";

but of course that's not a good solution; I got the possible settings here

    from an example submit script which said this:

----
# Possible options are:
#   -a mpich2
#     Use MPI over Gigabit Ethernet (slow interconnection network).
#     Use this when MPI_INTERCONNECT is set to either p4 or gige.
#   -a mvapich
#     Use MPI over MVAPICH, a particular implementation of the MPI
#       libraries that runs over the high performance Infiniband
#       interconnection network).
#     Use this when MPI_INTERCONNECT is set to ib and MPI_VENDOR isn't set.
#   -a openmpi
#     Use MPI over OpenMPI, another implementation of the MPI
#       libraries that runs over the high performance Infiniband
#       interconnection network).
#     Use this when MPI_INTERCONNECT is set to ib and MPI_VENDOR is
#       set to openmpi.
----

But I'm assuming that varies from implementation to implementation, right?

Any ideas how to automate this?

The other thing I had to hardwire in our lsf.pm is this:

$mpirun = ". $lsf_profile && /lsf/7.0/linux2.6-glibc2.3-x86_64/bin/mpirun.lsf";

since the LSF jobmanager install script only looked for "mpirun" in the PATH,
but we're using "mpirun.lsf" here.

Thanks a lot,
