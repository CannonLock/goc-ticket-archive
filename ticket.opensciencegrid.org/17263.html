    In the Gratia collector/mysql instance there is a set of mysql procedures that
update the summary tables for each JobUsageRecord updated:
* add_JUR_to_summary
* del_JUR_from_summary

When trying to remove excess CPU for some JobUsageRecords, a bug was
uncovered causing the del_JUR_from_summary procedure to NOT update
BatchPilot jobs at all.   While investigating this, other little inconsistencies
were uncovered.  It appears that over time when changes were made to the
add_JUR_to_summary they were not propagated to the del_JUR_from_summary.
The del_JUR_from_summary is a rarely used procedure.

Anyway, a new version has been made available and needs to be implemented in
advance of the normal release process for the Gratia collector in order to correctly
remove excessive CPU data from several sites which are causing the Gratia reports/graphs
to show really abnormal usage.

These procedures  are contained in this file:
/usr/share/gratia/sql/summary-procedures.sql

These changes only became  urgent to implement when it was discovered that
the BatchPilot jobs could not be removed. This are the changes made to the procedures:
http://sourceforge.net/p/gratia/svn/5077/
http://sourceforge.net/p/gratia/svn/5131/
http://sourceforge.net/p/gratia/svn/5132/

Note, that with these changes, new trace table entries will be created whenever
CPU exceeds Wall time for a job.  This is to allows for an easier check for
excessive CPU being recorded in Gratia.  This has been a real problem for many
months now.

This should only need be implemented on the primary MySql database of the Collector.
Not the replica.

On gr12x5

    1. Backup the current /usr/share/gratia/sql/summary-procedures.sql
2. Download the latest version of summary-procedures.sql
(actually Steve already has it downloaded yesterday)

On the Gratia collector admin UI:
1. Disable database updates

On gr12x5:
1. Update the procedures as root in the gratia database
mysql  <args> </usr/share/gratia/sql/summary-procedures.sql

On the Gratia collector admin UI
1. Enable database updates

Let me know if you think I missed anything or if you have more questions

John Weigand
