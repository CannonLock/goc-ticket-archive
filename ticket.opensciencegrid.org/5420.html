    Aug 15, 2008 10:25 PM UTC by echism 
    Hello,

We have decided to change how the VDT distributes Certificate
Authority (CA) Certificates.

There are two basic changes:

1) When you do a new, fresh installation of the VDT, it will no longer
install CA certificates by default. Instead, you will have to edit
a configuration file to point at an available CA certificate
distribution and run vdt-update-certs in order to get the CA
certificates.

2) The Open Science Grid Grid Operations Center (GOC) will start
distributing CA certificates for OSG sites, and the VDT's
installation will not be the recommended one for OSG sites.

Why are we doing this? The VDT installation (outside of the OSG
software stack) is intended to be a grid-agnostic software stack, but
the set of Certificate Authorities you trust is a combination of the
grid you a member of and what CAs you personally trust. By making
these changes, we do two things:

a) We require users to think--even if briefly--about what set of CAs
they trust.

b) The VDT team will no longer be in the business of recommending CAs
to Open Science Grid sites. Instead, that duty will belong to the
OSG Security Team, who will develop the CA distribution and host it
on the GOC's web site.

For the foreseeable future, the VDT will provide a convenience
distribution of CA certificates that will be the set of IGTF CA
certificates. (Please note that this means we will not be distributing
the IGTF and TeraGrid CAs anymore, just the IGTF.) However, you (the

    user) will be required to select this distribution if it is what you
want instead of another distribution. The VDT Team will provide this
as a convenience, but not as a recommendation.

Here is the way we'll phase this in:

August 18th:
VDT 1.10.1 will be able to install certificates from a configurable
location, which could be the GOC or the VDT or somewhere else.
We'll update vdt-update-certs to provide this functionality. The
configuration file for vdt-update-certs will not have a location in
it, and the script will default to the VDT convenience CA
distribution. No change in default behavior for VDT users.

August 22nd:
Same change as August 18th, but for VDT 1.8.1.

September 8:
A) New installations of VDT 1.10.1 will not install certificates by
default, and will require configuration to do so.

B) The VDT convenience distribution of CAs will be moved to a new
web location, and users will be required to change their
configuration to make a choice in what CAs they trust.

C) The VDT convenience distribution of CAs will be changed from the
current set (IGTF + TeraGrid) to be just IGTF.

Note: VDT 1.8.1 and older will continue to install CA certificates by
default in a new installation (unlike VDT 1.10.1), but after September
8th will only have the IGTF CA Certificates in fresh installations or
updates.
