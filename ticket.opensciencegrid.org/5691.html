    Oct 3, 2008 10:12 PM UTC by echism 
    Hi,

The cdf and cms VOMS servers are now replicated* to voms.fnal.gov on an hourly basis.

The cdf VOMS server on voms.fnal.gov was previously a snapshot of the cdf VOMS server
at CNAF from late June.  Its connection string has been added to the OSG vomses file
distributed with the OSG vo pacman package since that time.  This connection string
has not changed.

The cms VOMS server has never been replicated to voms.fnal.gov prior to this date.
The connection string for this VOMS server is the following:

"cms" "voms.fnal.gov" "15015" "/DC=org/DC=doegrids/OU=Services/CN=http/voms.fnal.gov"
"cms"

GOC, please add this line to the next vomses file for the next release of the vo
pacman package.

Cheers,
Dan

*We can only replicate the DNs of the users who have been issued certificates of the
CAs accepted in the OSG.  The master cdf and cms VOMS servers have additional, and in
many cases expired, CA certificates and users who have certificates issued by these
"extra" CAs.
