    May 9, 2014 07:30 PM UTC by Carl Edquist 
    Yes, here's the relevant part I see from the log:

Resource_List.ncpus=1
Resource_List.neednodes=1:ppn=8
Resource_List.nodect=1 Resource_List.nodes=1:ppn=8

The old version always gave priority to ncpus, if it was specified at all; but apparently ncpus sometimes gets specified as 1 and other fields (procs or nodes:ppn) are really the ones we should care about.  The fixed version will only consider ncpus if it's > 1, so it will use (nodes * ppn) = 8 instead.  ... So I think that will address the problem here.  The fixed version is available now in osg-testing and will go out in next Tuesday's release.

Thanks,
Carl
