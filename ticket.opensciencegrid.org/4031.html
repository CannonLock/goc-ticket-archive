Dear Ruth.
I think that CEMON and ReSS tool that Gabriele is working on are very
important. Probably the most important step in the OSG development at
present time. These tools have potential to transform OSG from a set of
computer farms to a grid. Right now we are blindly  submitting jobs to
OSG sites hoping that there are enough batch slots for them, and
empirically searching sites where we can run jobs. The first time I see
tool that can give me list of sites where I can run jobs based on
parameters of the job and my VO. And most valuable would be possibility
to know at the submission time how many batch slots the site have
available for my VO. This would make running production on OSG much
more efficient. From this point of view it should be a priority in GOC
policy. It may be for the future release of OSG, but seems it should be
supported by GOC.
What do you think about this?

Regards.
Nikolay.

Rob,
I'm sorry, but it is too easy to dismiss this ticket this way...

CEMon was an optional component for OSG 0.6.0 because it was new. Now, both
interoperability with EGEE and OSG resource selection depend on it. We have to push
sites to modernize their information infrastructure or we will not have a usable grid.

If this is achieved by making CEMon a mandatory component, then probably we have to
fight this political battle. Evidently, considering it an optional component does not
give us enough leverage.

I kindly ask to reopen ticket 4031.
Gabriele

    Sep 12, 2007 06:17 PM UTC by Rob Quick 
    Nikolay,

From the OSG CE Installation:

"The CEMon (CEMonitor) service is responsible for providing information coming from
the Computing Element (CE) and is being implemented as an optional component in the
OSG Compute Element (CE) architecture in support of the ReSS (Resource Selection
Service) OSG project."

With CEMon being an optional component of OSG, the GOC can not make sites install
CEMon. I assume each of these sites has chosen not to run the optional component. I
apologize about the delay in this repsonse.

I am closing this ticket,
Rob
