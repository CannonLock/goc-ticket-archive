    [by Soichi Hayashi]

I have my condor installed via RPM, and I have following in /etc/osg/config.d/20-condor.ini

condor_location = DEFAULT

When I run osg-configure -v I get following error message.

CRITICAL Error in GratiaConfiguration while parsing configuration
CRITICAL Exception: Can't get value for condor_location in Condor section and no default given
Error in GratiaConfiguration while parsing configuration
You may be able to get more details rerunning /usr/sbin/osg-configure with the -d option and/or by examining /var/log/osg/osg-configure.log

I changed /etc/osg/config.d/20-condor.ini to contain

condor_location = /usr/bin/condor

And now osg-configure -v works. I am not sure if this is expected or not, but I think it's strange that I can't leave the condor location DEFAULT if I didn't change it.
