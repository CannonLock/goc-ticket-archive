    Sep 26, 2012 06:31 PM UTC by OSG-GOC 
    Thanks Pavel.

We tracked down this unusual error.  It was the result of trying to setup the federated Xrootd configuration in our newly updated dCache instance.   We had included some xrootd and name-to-name plugins in /usr/local/share/dcache/plugins on our xrootd door node and one other door node.  This apparently caused a problem for  the gridftp  doors on those same nodes.   We have backed out  those changes at around 1:10 PM Eastern time today (Sep 26 2012).   The "rsaEncrypted" errors seem to be gone now.

Summarizing our status:
1) We had signifcant problems with individual pools getting overloaded when certain analysis skim jobs were running in large numbers
2) We upgraded our dCache from 1.9.12-21 to 2.2.4-1 on September 24 2012.
3) We had some residual issues related to the significant changes in dCache that we had to correct (path changes for our LSM configuration was the main one)
4) Changes last night for FAX caused problems with dCache and were backed out.
5) Some doors had incorrect iptables settings that were blocking some gridftp connections (also fixed today).

We are optimistic that things should again be functional.  If they look OK we should close this ticket.

Shawn
