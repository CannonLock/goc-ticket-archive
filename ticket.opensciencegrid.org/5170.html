What happens if you submit from gate02 itself? That is, when there is
no network in the way.

Is there a firewall between the two computers?

Do all jobs fail, or just some of them?

The container-real.log is not relevant, because it is for the Globus
container, which is not used for globus-job-run.

There is a useful log file, but it's hard to find it. It's the GRAM
job manager log. Here's what you should do.

1) Figure out what user you will be when you submit a job.

2) Go into that user's log directory. If the user is not submitting
jobs, delete all gram user logs. I forget the precise name, but it
should be clear.

3) Submit a job. Watch for a new gram job manager log to be created in
the user's home directory.

4) Watch the file for a bit. Kill the job, send the log file.

By the way, is this home directory on a shared file system? Is that
file system functioning smoothly? No NFS problems?
