Open Science Grid FootPrints wrote:
> [Duplicate message snipped]
    Jun 4, 2010 08:30 PM UTC by Mariachi 
    Hi Bockjoo,

I think the issue is the "Match FQAN" setting.

The user's VOMS proxy has Role=analysis, but the dzerouser mapping
setting has "Match FQAN"=exact, but does not have a Role defined. So the
mapping is null.

I think what you want is to make Match FQAN=vogroup. This will ensure
that the user is a member of /dzero/users, but won't care if they have
any Role set or not.

The gridmapfile has an entry, because you have "Accept Grid Proxies" set
to true. The way the gridmapfile is made, is each user's DN (and only
their DN) is run through the system to see what they map to. So in this
context they get mapped.

Hope this makes sense.

BTW, there has always been a tension in GUMS between providing enough
flexibility to express complicated policies and it being complicated
enough to produce unexpected behavior. We've had several cases here as
well where the behavior was mysterious. In almost every case we
eventually figured out that GUMS was doing exactly what we asked--we
just didn't realize what we had asked it to do. :-)

Let me know if switching to 'vogroup' doesn't fix the mapping problem.
You could also set it to 'ignore', since in this case there should be no
difference.

Cheers,

--john
