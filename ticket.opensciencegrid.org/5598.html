    All,

This came to the GOC from Michael Thoman at CalTech...

Description of Problem:
OSG ca-certificates rpm file is poorly named. The convention for
naming rpm files is <name>-<version>-<release>.<arch>.rpm, where
<name> exactly matches the Name: field in the rpm .spec file. The CA
certificate package is named 'osg-ca-certificates-...noarch.rpm',
while the Name: field in the rpm .spec file is 'osg-ca-certs'.

This isn't a major problem, it it would be nice to maintain
consistency with the way that most major Linux vendors name their rpm
packages.

It is GOC ticket number 5598,

Rob
