    I believe I've encountered an argument parsing issue with srm-ping-probe v.1.6 on the commandline.

$ ./srm-ping-probe --version
srm-ping-probe v.1.6
Arvind Gopu [http://grid.iu.edu]

The inline help describes the serviceURI argument as:

hostname[:port][/service]

However, when I pass the following invocation:

./srm-ping-probe --verbose -u se1.accre.vanderbilt.edu:6288/srm/v2/server

the following commandline is constructed from the arguments:

CMD: [/gpfs0/local/x86_64/osg/srm-client-fermi//bin/srmping -retry_num=0 -debug -2 -x509_user_proxy=/tmp/x509up_u35741 srm://se1.accre.vanderbilt.edu:8443 -webservice_path=srm/managerv2 2>&1 ]

The working invocation should be:

/gpfs0/local/x86_64/osg/srm-client-fermi//bin/srmping -retry_num=0 -debug -2 -x509_user_proxy=/tmp/x509up_u35741 srm://se1.accre.vanderbilt.edu:6288 -webservice_path=srm/v2/server

The probes were downloaded from: http://rsv.grid.iu.edu/downloads/OSG_RSV_Probes-2.3.7.tar.gz

Is there a newer version of this probe, or is this a known issue?
