    Doug,

To answer your question about how other sites keep up, the vast majority are following the official install instructions at https://twiki.grid.iu.edu/bin/view/Documentation/Release3/InstallCvmfs which automatically makes all repositories available on clients without any intervention at the sites.  Since you're exporting cvmfs over NFS, the automounting does not work for you because there's no way to have an NFS client access trigger an automount on an NFS server; you need to pre-mount and export all the repositories.

If you only need to support NOvA, you can mount only the few repositories that Art just posted here.  If you want to be able to support all OSG experiments, there is a way, although I don't know of any site that has done this yet so it will require a small amount of script development on your part.  There is a script that will give you the names of all the current opensciencegrid.org repositories.  It's not currently in an rpm that's convenient for you to use nor installed in a cvmfs repository, but you can get it from
https://raw.githubusercontent.com/opensciencegrid/oasis-server/master/bin/print_osg_repos

Please let me know if you decide to use that for cvmfs over NFS.   If you do, it would be good if we at least documented it in case someone else wanted to do it and maybe also package it better.

I am also interested in hearing if you have verified that the bandwidth from a cvmfs-over-NFS server is adequate for your supercomputer.  I haven't heard of another supercomputer site that is successfully using that in production.  Most of them use another mechanism to regularly update a copy of all files from cvmfs to their NFS servers, using a procedure David Lesny dubbed "Stratum-R".   I can give you more details if you're interested.

Dave
