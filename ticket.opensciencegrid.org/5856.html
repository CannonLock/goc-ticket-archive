New Work Log Entry : The following was e-mailed to the Requester:

greenc@.... sent this Notes To Requester:

OK, we have a miscommunication here.

The original ticket is, "Gratia not reporting for NYSGRID_CORNELL_NYS1"

This pertains to the fact that we have not been receiving probe data
from your gatekeepers. This has *nothing* to do with the "Gratia
Reporting" package, which is only necessary if you wish to *receive* and
produce reports on Gratia Accounting data. The *probes* are what should
be sending accounting data to us.

Please, abandon your attempts to install Gratia Reporting.

Running remote site diagnostics indicates that you have pbsAcctLogDir in
$VDT_LOCATION/gratia/probe/pbs/urCollector.conf set to:

/opt/osg-1.0.0/globus/var/globus-pbs-log

which is an empty directory.

You need to set this to the server_priv log directory of your PBS
master, eg:

/var/spool/pbs/server_priv/accounting

If this is not on the gatkeeper, you need to find some way (preferably
NFS mounting) to make these data visible to the gatekeeper.

Thanks,
Chris.
