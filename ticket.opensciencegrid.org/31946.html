    Nov 22, 2016 01:45 AM UTC by OSG-GOC 
    We did run into some additional problems during the installation, though not related to the earlier kernel panic.   But perhaps there's an additional mismatch between the net-boot install done on PDSF and the way osg services are configured that you can help us identify.  Starting this ticket to help investigate.

The installation is done as an add-on to a net-boot of a base system &, for example, initially did not save the rpm database.  As a result, osg-configure did not see the installed rpms and bypassed the configuration of the htcondor_gateway.   We fixed this by including the rpm database, so now the osg-configure does run that configuration section,    however it doesn't work & seems incomplete. My thinking is there may be something else fundamental (like the missing rpm database).  In particular neither the schedd nor the JobRouter are configured to run:

condor_ce_config_val DAEMON_LIST
MASTER COLLECTOR, SHARED_PORT

which appear to be set in the defaults:

/usr/share/condor-ce/config.d/01-ce-collector-defaults.conf
/usr/share/condor-ce/config.d/03-ce-shared-port-defaults.conf

I see that they are set in /etc/condor-ce/condor_config:

grep DAEMON_LIST /etc/condor-ce/condor_config
DAEMON_LIST = MASTER COLLECTOR SCHEDD JOB_ROUTER

but that seems to be completely ignored as the only services running on start/restart are those shown by "condor_ce_config_val DAEMON_LIST":

condor_master -pidfile /var/run/condor-ce/condor_master.pid
condor_procd -A /var/lock/condor-ce/procd_pipe -L /var/log/condor-ce/ProcLog -R 1000000 -S 60 -C 492
condor_shared_port -f -p 9619
condor_collector -f -port 9619

Unfortunately, I don't have access to a working install to compare with.   We're running in SL6.3 & the set of -ce- rpms installed are:

htcondor-ce-2.0.2-1.osg33.el6.x86_64
osg-base-ce-3.3-7.osg33.el6.x86_64
osg-base-ce-sge-3.3-7.osg33.el6.x86_64
osg-ce-3.3-7.osg33.el6.x86_64
htcondor-ce-view-2.0.2-1.osg33.el6.x86_64
osg-htcondor-ce-3.3-7.osg33.el6.x86_64
osg-ce-sge-3.3-7.osg33.el6.x86_64
osg-htcondor-ce-sge-3.3-7.osg33.el6.x86_64


    I stop/start the service via:

service condor-ce start|stop|restart|status

I just now stopped the service, moved the old MasterLog file and started it again.  You can down load the newly produced MasterLog at:

http://portal.nersc.gov/project/osp/pdsfgrid/MasterLog
