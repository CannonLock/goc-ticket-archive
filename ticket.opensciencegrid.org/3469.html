----STEPS TO FIX THE ISSUE fwd begin-----

The change you made in that osg-user-vo-map.txt file apparently gets over-written by
another script. So you need to do a bit more fixing.. Sorry about the delay in
updating you on this ...

John Rosheck in the GOC did some testing and has come up with a solution to fix this
"Engage" vs. "engage" issue: upgrading vo-0.6. Can you give this a whirl, and let me

    know how it goes?

Here are the steps suggested by John:

##################
# (Setup pacman path)
# . /grid/mypacmandir/setup.sh
# cd $VDT_LOCATION

# (Make copy of setup.sh in case ... John's paranoid and loosing this file is very bad)
# cp setup.sh /root/my-ce-setup-date.sh
# (supposed to make pacman save it too)
# pacman -allow save-setup

# (actually update the CE installation)
# pacman -update OSG:vo-0.6

# (then recreate the files in $VDT_LOCATION/monitoring, etc)
# edg-mkgridmap

Note: Which VO's a site supports is now controlled by commenting out the actual "group
voms://" lines for each VO in the file $VDT_LOCATION/edg/etc/edg-mkgridmap.conf. I
know this works for edg-mkgridmap.

##################

Also, Carl, if your site is using GUMS then you need to edit your gums config file,
and copy over the Engage entry from the new gums template; also please copy the
hostgroup line from the template file.

------ END FWD --------------
