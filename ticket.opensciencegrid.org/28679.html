    Hi Baosong,

Thank you for the explanation. In Clemson, we use PBS to manage jobs. You can find some sample job scripts to run on our cluster at http://citi.clemson.edu/palmetto/pages/userguide.html. I think you are submitting jobs to Open Science Grid using Condor, so you may find some samples from the following resources:
https://twiki.opensciencegrid.org/bin/view/Documentation/Release3/NavUserMain
https://support.opensciencegrid.org/support/solutions/articles/5000633410
https://portal.xsede.org/OSG-User-Guide

For the OSG jobs, the jobs will be route to a gateway node named osg-ce.clemson.edu. The gateway node runs condor_ce server which takes OSG jobs, converts them to PBS jobs scripts, and then submit to our local PBS jobs. If the resulted PBS job script meet the PBS requirement,  the job should be able to run successfully unless there is a systematic issue on our site.

Because Clemson mainly contributes infrastructure resources to OSG and our team mainly focus on infrastructure support, you may first contact the OSG technical support team to see if they can provide better assistance. The OSG support team may be able to connect you to other OSG users with successful experiences of using Clemson resources.

    Thank you, Bo.

I looked at the log files attached to this ticket. The logs indicated three likely causes for the job failures.

First, the job complains memory allocation failure and ulimit command was not permitted. This can be fixed by explicitly specifying the memory requirement in the job script. The Palmetto cluster at Clemson is managed by PBS which enforces the resource usages like wall clock time and memory usages. The user needs to specify how long the job will last and how many memory the job needs. Otherwise, the default value 30 minutes wall clock time and 1GB memory will be used.

Second, the job complains that the command globus-url-copy command was not found. The program is available under the osg-wn-client installation tree. It can be found at $OSG_GRID/usr/bin. The job script many need to include the full path.

Third, the job complains that the kinit command was not found. I have no answer for this one.  The script might not need Kerberos authentication anyway since osg jobs are assumed to use the grid certificate for authentication.

Please let me know if the jobs could succeed after the user makes corresponding changes.

Thanks,

Xizhou
