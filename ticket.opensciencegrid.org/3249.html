Hello GOC,

I'm in the process of installing OSG-0.6.0 for the STAR-BNL site, but I'm hitting a
problem requesting a certificate.
I think the problem is that my gatekeeper requires a proxy for outbound http (or ftp)
connections, but it looks like cert-request
($VDT_LOCATION/cert-scripts/bin/cert-request) doesn't honor the proxy environment
variables that I have set:

[root@stargrid02 temp]# env |grep -i proxy
http_proxy=http://squid.sec.bnl.local:;3128
FTP_PROXY=http://squid.sec.bnl.local:;3128
ftp_proxy=http://squid.sec.bnl.local:;3128
HTTPS_PROXY=http://squid.sec.bnl.local:;3128
https_proxy=http://squid.sec.bnl.local:;3128
HTTP_PROXY=http://squid.sec.bnl.local:;3128

I tried editing cert-request by adding "$ua->env_proxy;" in two places as shown below:

...  line 212
my $ua,$req,$res;
$ua = LWP::UserAgent->new;
#Added the following to account for BNL http proxy (W. Betts)
$ua->env_proxy;
#End addition by W. Betts
$ua->agent("RequestScript/" . $pkgversion . " ");
$req = HTTP::Request->new(POST => $baseURL);
... ~line 790 (in sub get_osgvolist)

    my $ua = LWP::UserAgent->new;
#   Added by W. Betts for BNL proxy server.
$ua->env_proxy;
#End addition by W. Betts
$ua->agent("RequestScript/" . $pkgversion . " ");
my $req = HTTP::Request->new(GET => $url);

I also tried being more explcit by using the following instead of env-proxy:
$ua->proxy('http', 'http://192.168.1.4:3128/');

In all cases, there is a long pause (~a couple of minutes) when I execute my request
("cert-request -ou s -dir . -host stargrid02.rcf.bnl.gov -service http -label
stargrid02-http") then it proceeds and gets to the input stage for a registration
authority, and it will go no further:

[root@stargrid02 temp]# cert-request -ou s -dir . -host stargrid02.rcf.bnl.gov
-service http -label stargrid02-http
<snip (key generation, admin name, etc)>
input your complete phone number: 6313443285

BTW, I already have a host cert and a cert for ldap from my existing 0.4.1
installation -- I'm only trying to get a certificate for http for CEmon, so this
isn't an immediate show stopper.  I suppose I could use one of the existing certs for
http in the meanwhile.
