    Oct 20, 2014 06:18 PM UTC by Kevin Hill 
    Hi,
This patch provides support for an extension (TLS_FALLBACK_SCSV) that will prevent clients and servers that can do TLS v1 or better from being tricked into falling back to SSLv3 or lower.  TLS_FALLBACK_SCSV does need to be supported on both the client and server sides, so probably won't help clients that need SSLv3 right now. Therefore I'd still recommend trying to get SSLv3 turned off where possible.

-kevin

Brian,

According to this script installed by osg-pki-tools,
[/usr/lib/python2.6/site-packages/osgpkitools/OSGPKIUtils.py]
ssl_context = M2Crypto.SSL.Context('sslv3')

It seems that our osg-pki-tools is forced to use sslv3. I believe this should be updated to

ssl_context = M2Crypto.SSL.Context('sslv23')

[from https://github.com/pulp/pulp/pull/1244/files]
> # Despite the confusing name, 'sslv23' configures m2crypto to use any available protocol in
> # the underlying openssl implementation.

I've tried it with "sslv23", and I was able to issue host certificate through OIM-ITB. Could you investigate and see if we need to apply this patch ASAP?
