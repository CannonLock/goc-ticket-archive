    Mar 14, 2012 10:38 PM UTC by Matyas Selmeci 
    Looks like the schedd from condor-cron is sending a conflicting ad to the collector. To work around this, you should add a file into /etc/condor-cron/config.d called 'condor_host' containing the single line:
CONDOR_HOST =
and restart the condor-cron service. This will prevent condor-cron from sending ads to the collector, but the condor-cron specific tools (such as condor_cron_q) will still see it.

If this works, then there is an issue with the default condor-cron configuration that OSG ships that we'll have to resolve.
