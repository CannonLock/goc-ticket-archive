    Feb 4, 2016 03:37 PM UTC by Vince Neal 
    Notification was sent out on 2/2/2016.  Are there any outstanding action items for this ticket? If I may assist please let me know.

Greetings All,

WHAT ARE WE DOING?
The following OSG virtual organizations (Gluex, Icecube, Star, Snoplus and OSG) are being transitioned from the DigiCert certificate authority (CA) to the CILogon OSG CA.

WHEN WILL THIS OCCUR?
Tuesday, Feb. 9; 8 a.m. to 4 p.m.

WHAT IS THE IMPACT TO YOU?
• As part of this transition, on Tuesday, Feb 9 from 8 a.m. to 4 p.m., our VOMS server will be switched to use CILogon OSG CA certificates, hence its distinguished name (DN) will change.
• If the vo-client or osg-ca-certs packages on the node you are using have not been updated to version 63-1 (vo-client) or version 1.50-1 (osg-ca-certs), you will see voms-proxy-init requests. Make sure you have these package versions installed.
• Distinguished names for certificates issued by the new CILogon OSG CA have been uploaded into the VOMS databases for users who currently have certificates issued by the DigiCert CA.

WHAT DO YOU NEED TO DO?
• Please upgrade osg-ca-certs to at least version 1.50-1.
• Please upgrade the vo-client package to version 63-1 when available from OSG 3.3 repo starting Tuesday, Feb. 9.

OSG Grid Operations Center (GOC)
Email/Phone: goc@...., 317-278-9699
GOC Homepage: http://www.opensciencegrid.org/ops

    Hi GOC,

Below, I am describing work that needs to be done (and by whom) to get OSG VO transitioned to CILogon OSG CA in February

1) (Scott/Kyle) Change certificate used on voms.grid.iu.edu (which hosts the replica of OSG VOMS server) to new one issued by CILogon OSG CA

To get the new certificate (for voms.grid.iu.edu) before transition actually happens, use the OIM ITB instance (instead of production one) -- via web interface (http://oim-itb.grid.iu.edu/oim/certificate) or use command line tools (osg-pki-tools)

2) Upload new CILogon DNs for all users that currently have Digicert CA issued certificates

My understanding is there will be more than 1 person who will be doing this.

MIS/CSIU - Kyle/Rob
OSG - Neha -- I volunteer myself for this as OSG VOMS DB at IU is simply syncing the contents of OSG VOMS DB at Fermilab. I have already uploaded DNs for fermilab, CDF, DUNE VOs, so am familiar with the process. But, OSG VO managers/admins should confirm to me that they are OK with me doing this. If not ok, please let me know who will be doing this at Fermilab.

This step involves running a script (provided by OSG Security) that will look up all existing (user/host/service) Digicert DNs in VOMS DB and add their CILogon OSG CA equivalent. Digicert DN entries will NOT be affected.

Sole purpose here is user convenience. Users will still need to request new CILogon OSG CA certificates ( when their current Digicert CA cert expires or is about to expire), but they won't have to request membership to OSG ( or mis/CSIU) VO with their new DN (because it would have uploaded in VOMS DB already)

Neha will email the script and instructions on how to use it - by end of day 01/13/16

3) (Kyle/Scott) Communication with users/VO members

For all VOs that are affected by certificate DN change on voms.grid.iu.edu, please communicate with user community telling them what this transition means and what they need to do.

Users -- should know that --  current Digicert CA certs will continue to work until expiration; procedure to request/renew certs will remain same and it is only the backend CA that is changing; post transition date, certs will be issued by CILogon OSG CA; new DNs will be already uploaded in VOMS DB for their convenience  (everything I mention in 2 above) etc.
Admins -- should install CILogon OSG CA cert on all relevant nodes ( upgrade osg-ca-certs to atleast 1.50-1) before Feb 9 and upgrade vo-client to version XY-Z when available from OSG 3.3 repo starting Tue, Feb. 9. "

4) (Scott/Kyle) -- Coordinate with OSG Software team on changes to vo-client package ( Feb 9 release)
