    Apr 29, 2011 03:57 PM UTC by rynge@.... 
    We can talk on the phone about it, but this is a bigger issue than just
creating a file. The problem is that non of the infrastructure (VOMS,
VOMRS, ...) collects this information. This is not scalable for two reasons:

1. It should be the user who picks their field. The VO admins only have
a small insight in what actual research is, and I feel uncomfortable
guessing and classifying users because the lack of information.

2. If OSG wants to collect the data, tooling/infrastructure should be
updated to support the collecting and aggregation. Currently the VO now
has to keep a second database, and try to keep it in sync, which quickly
becomes a mess.

I think this is something that need to be bubbled up to either a blue
print or some other requirement gathering group.

I will create a new file but due to (1), this will take some time.
