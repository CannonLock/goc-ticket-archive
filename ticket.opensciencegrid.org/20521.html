    With clean LDIF the data is still not being forwarded to BDII.
Burt suggests that this might be a host name reverse lookup problem.  i.e. FNAL_GPGRID_1 has got two IP addresses,
fnpcosg1.fnal.gov which is registered in OIM as the official resource IP for FNAL_GPGRID_1, and fnpc10x1.fnal.gov (from which the GIP
data is actually coming to is1.grid.iu.edu) which is not.

GOC staff, can you please look at this and verify that this is what is happening?  If need be, we can shift the IP address that is sending
the packets but we would prefer not to if we don't have to.
