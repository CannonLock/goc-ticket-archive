    Jan 14, 2009 04:18 PM UTC by VDT 
    Hi Steve,
In the 1.10.1n release, configure_gratia was changed with some
suggestions from the Gratia developers.  Specifically, there was a
concern that the 3 host fields in the ProbeConfig would revert to
localhost:8443 if "configure_gratia --probe <probe type>" is called
without supplying the --report-to option.

So they suggested adding a new command line option,
--force-probe-config.  Now, the hosts are only set if either:
a) this is the first time configuring this gratia probe
b) the --force-probe-config flag is passed

I am guessing that what happened for you is that the first time
configure_gratia was called (probably from configure-osg.py upon
update/installation), the host was set to localhost:8443.  After that
point, it will not change unless you supply --force-probe-config.

I am going to make a few changes to configure_gratia for the upcoming
1.10.1s release to be more verbose if it is not changing the hosts so
that this change is more apparent to a user.  Sorry for the confusion.

-scot
