There is a problem with how the deletion service is attempting to use the most
recently created dump file.

An example:

The deletion service is attempting to delete:

srm://gk03.atlas-swt2.org:8443/srm/v2/server?SFN=/xrd/datadisk/rucio/SAM/testfile-DEL-ATLASDATADISK-1422508192-fb58f875b9af.txt

This file does not exist.

In the local filesystem the file is found at:
/xrd/datadisk/SAM/testfile-DEL-ATLASDATADISK-1422508192-fb58f875b9af.txt

The base path for the DATADISK token is:
/xrd/datadisk

The SAM file is written outside of the rucio area.

The dump file being used by the deletion service lists the file as:
SAM/testfile-DEL-ATLASDATADISK-1422508192-fb58f875b9af.txt

For some reason the deletion service is prepending a "rucio" to this path when
trying to delete it.
