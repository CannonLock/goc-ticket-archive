On Fri, 8 Dec 2006, Gabriele Garzoglio wrote:

Hi Tibor,
in general, the statuses reported by the monitoring system are a translation from the local batch
system statuses. In the case of osg, the jobs at the forwarding node are submitted to condor-g for
further handling by OSG. The status reported are the ones coming from condor-g and are translated
as follows:

map_local_id_and_status () {
awk 'BEGIN {
JIM_JOB_SUBMITTED="submitted"
JIM_JOB_PENDING="pending"
JIM_JOB_ACTIVE="active"
JIM_JOB_FAILED="failed"
JIM_JOB_DONE="done"
JIM_JOB_SUSPENDED="suspended"
JIM_JOB_UNSUBMITTED="unsubmitted"
}
{
id=$1;
condor_status=$2;   /*1 = Idle, 2 = Running, 3 = Removed, 4 = Completed, and 5 = Held.*/
if   (condor_status==1) {jim_job_status=JIM_JOB_PENDING}
else if (condor_status==2) {jim_job_status=JIM_JOB_ACTIVE}
else if (condor_status==3) {jim_job_status=JIM_JOB_DONE}
else if (condor_status==4) {jim_job_status=JIM_JOB_DONE}
else if (condor_status==5) {jim_job_status=JIM_JOB_SUSPENDED}
else {jim_job_status="UNKNOWN"}
printf("JobId=%s Status=%s\n",id,jim_job_status)
}'

}

If you are interested at the mapping for different batch system, you can look at the specific batch
system handlers in sam_batch_adapter/etc/handlers/

Cheers
Gabriele
