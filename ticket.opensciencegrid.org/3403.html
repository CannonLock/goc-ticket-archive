Short Description=OSG batch environment does not pick up GLOBUS_LOCATION

Status=Service Desk

This is from a (relatively) clean install of 0.6.0 on CMS-BURT-ITB

[burt@zork ~/060]$ globus-job-run cmsitbsrv01/jobmanager-fork /bin/bash -c 'echo
$GLOBUS_LOCATION; grep ^GLOBUS_LOCATION
$OSG_LOCATION/monitoring/osg-job-environment.conf'
/fnal/products/osg/v0.6.0/globus
GLOBUS_LOCATION="/usr/local/osg/wn_client/v0.6.0/globus"

[burt@zork ~/060]$ globus-job-run cmsitbsrv01/jobmanager-condor /bin/bash -c
'printenv' | grep GLOBUS_LOCATION
GLOBUS_LOCATION=/fnal/products/osg/v0.6.0/globus

--

I know osg-job-environment.conf works for other variables -- if I drop FOO=BAR in
there, I get it in the job environment. Somehow GLOBUS_LOCATION is being overridden
somewhere?
