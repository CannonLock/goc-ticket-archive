    John [adding Rob Quick to ticket],

If I understand correctly, you make a good point. Without publishing active/inactive history for a resource, it's hard to tell when it was flipped from inactive to active or vice-versa -- correct?

Note that active flag (and disabled flag) can only be set by GOC staff.

I suspect we need to consider one or more of the following options -- thinking out aloud, please don't make any changes yet but do provide feedback you might have to the points below!

1) You should consider using OIM log to figure out if some flag was flipped - if necessary, we can allow a separate user or service certificate to be able to access OIM logs https://oim.grid.iu.edu/oim/log -- would this help?
-- Perhaps we (GOC) should publish these logs as a miscellaneous MyOSG view so there is one-stop shopping for a user like you?
AND/OR
-- Perhaps we need to add active/inactive log for resources just like we did for VOs per Gratia's request.

2) Recommend/require to admins that they request inactivation of a resource only if the resource is going to be down for maintenance for > n days (say 30 days). This will prevent arbitrary flipping of the flag.

Meanwhile, we are working on figuring out the feasibility of removing the active flag from all entities but resource (and possible VO if Gratia is using the active flag).
