    Mar 6, 2014 02:09 PM UTC by marian.babik@.... 
    Technically, SAM recomputation changes all CRITICAL statuses to UNKNOWN
statuses (so adjusting the underlying data first), i.e. it is re-painting
red to grey, leaving green parts untouched for the affected period. The
time spent in UNKNOWN status is not taken into account when computing A/R
numbers, so after adjusting the statuses, A/R numbers usually improve. At
the same time, time spent in UNKNOWN status is also part of the reports
(for T1s below the graphs, for T2s as 3rd column after A/R), of course in
effect of any re-computation this number will increase.

For additional details on reports, instances, metrics and aggregations you
can link this page
https://tomtools.cern.ch/confluence/display/SAMDOC/WLCG+Critical+Metrics

M.
