    Oct 7, 2011 03:59 PM UTC by OSG-GOC 
    [by James Weichel]

There is a case where the search for document reference when doing a document move fails to find them all.  When a reference is of the form: [[ReleaseDocumentation/CertificateGetCmd][Get Cert with unix commands]] the link will not get updated if CertificateGetCmd is moved to a different web.

If the link is instead [[ReleaseDocumentation.CertificateGetCmd][Get Cert with unix commands]], then the link does get updated.

Also links of the same form in INCLUDES %INCLUDE{"Documentation/DocumentationTeam/DocConventions" section="Header"}% do not get updated while if they are written %INCLUDE{"Documentation/DocumentationTeam.DocConventions" section="Header"}%, they do get updated.

Since we will be moving some of the documents that are not release dependent from ReleaseDocumentation to Documentation, this is a problem.

If this could be fixed easily, you should do it.  If not, we may need to search documents for this error and fix them manually.
