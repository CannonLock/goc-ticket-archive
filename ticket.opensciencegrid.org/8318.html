    Hello,
I've got jobs failing at Purdue-RCAC.  There seem to be two problems with two independent clusters at the site.

1.) on the "Miner" cluster I'm getting this error when trying to use curl:
curl: error while loading shared libraries: libssl.so.4: cannot open shared object file: No such file or directory

The curl being used was: /opt/osg/curl/bin/curl

In the last job run this error occurred 41 times on miner-a127, plus a handful of times on a dozen other nodes that jobs got scheduled to.
We've seen problems with curl at FNAL before, I don't know if this ticket might have any info that is relevant to the Purdue-RCAC cluster
https://ticket.grid.iu.edu/goc/viewer?id=8025

2.) Several nodes in the Prospero cluster seem to be out of disk space, or possibly our group user ran out of quota on the filesystem for GRAM_SCRATCH or /tmp
I had 88 failures at prospero-017.rcac.purdue.edu
and 57 at prospero-004.rcac.purdue.edu
plus a few others.

The job script was using 'find -delete' to clean up after itself, but it looks like this argument isn't supported by the version of find on that cluster, so it might have caused our jobs to leave behind files and fill disks/quotas.
I'll accept suggestions for alternate methods to have a job clean up after itself that would be universally supported on OSG platforms.

Thanks!
