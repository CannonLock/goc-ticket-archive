    First note that you have a very old version of BeStMan.  You should consider upgrading at some point to a newer version (BeStMan2 can be installed while still using pacman).

Look at $VDT_LOCATION/bestman/conf/bestman.rc.  There should be CertFileName and KeyFileName lines that contain your service cert.  This is likely the certificate you need to modify.  You may also want to look at $VDT_LOCATION/bestman/sbin/bestman.server to make sure that CERTPATH,KEYPATH,GUMSCERTPATH,GUMSKEYPATH are also updated.

    Jul 13, 2012 06:04 PM UTC 
    Are you running BeStMan on your SRM instance?  Are you using the pacman version or the RPM version?
I would guess that the service has stopped for some reason.  You should be able to see the process running with "ps -ef | grep -i bestman".

With pacman, you can start it with either "vdt-control --on" followed by the name of the process (bestman or bestman2).  With the RPM, "service bestman2 start".
You may also have to start the gridftp server if that is off too.

You may also want to check the logs in $VDT_LOCATION/vdt-app-data/bestman2/logs (pacman) or /var/logs/bestman2 (RPM) to see if an error is preventing the process from starting up.

OSG Software
