Summary*        : Location of PBS accounting log directory is mis-configured.
Notes           : Hi,

Sorry for the delay in replying.

Your OSG installation is currently misconfigured: the pbsAcctLogDir entry in $VDT_LOCATION/gratia/probe/pbs/urCollector.conf is currently set to the empty directory:

/opt/osg-1.2.8/globus/var/globus-pbs-log

It needs to be set to the server_priv/accounting directory of your PBS installation (usually /var/spool/pbs/server_priv/accounting or similar). It is usually root-readable only. You can do this directly in the urCollector.conf file, but to avoid overwriting during a subsequent configure step you should make sure that $VDT_LOCATION/globus/etc/globus-pbs.conf has the correct location fro the server_logs directory and the configuration system will deduce the correct location of the server_priv/accounting directory.

eg:

[root@d0osg1x1 etc]# cat globus-pbs.conf
log_path=/var/spool/pbs/server_logs

Leads to a deduced value of:

pbsAcctLogDir = "/var/spool/pbs/server_priv/accounting"

after re-running configure-osg.

GOC: please cc: VDT and OSG configuration experts. I am unsure of the provenance of globus-pbs.conf: how it is created (automatically or manually); if the former, where does it get its answer from; if the latter, where is it documented that this file needs to exist? Even if this particular issue is resolved, I would still like to know the answer to the above question.

Thanks,
Chris.
