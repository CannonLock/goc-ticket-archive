    Jan 15, 2009 04:54 PM UTC by Parag Mhashilkar 
    Hi,

What is the OSG version installed on the machine? Please note that
starting with OSG v1.0, newer versions of cemon changed the unit of
policy rates from ms to sec. So 30000 is every 500 min which is quite
long.

In case, you do have OSG v1.0, edit
$GLITE_LOCATION/etc/glite-ce-monitor/cemonitor-config.xml and change the
rate to be 600 from 30000

Also, check if you have reasonable value (in sec) for executionDelay

<sensor id="sensor-osg-ce"
[...]
<property name="executionDelay" value="600" />

</sensor>

Save the changes, and restart tomcat.

These values are supposed to be 600 sec by default for OSG v1.0. Did you
change it copied the config from older OSG installations?

Let me know if this helps, if not,

1. Enable debug logging for CEMon -
$VDT_LOCATION/tomcat/v55/webapps/ce-monitor/WEB-INF/classes/log4j.properties
Change following two parameters to look like this ...
log4j.logger.org.glite.security=error, fileout, info, warn
log4j.logger.org.glite=info, fileout, error, warn
2. Restart tomcat and let is run for 20 mins atleast
3. Mail/upload the config file cemonitor-config.xml and upload the log
file (GLITE_LOCATION/var/log/glite-ce-monitor.log) so that I can go
through them.

    
PS: Please address future tickets to me rather than Tanya. The ress
support address remains same (ress-support@....)
