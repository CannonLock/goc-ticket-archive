    I see you've also set lcmaps_log_level to 5.  That's the important one and should be sufficient (and give us a lot of messages we don't need which is fine).   But there should be a lot more messages.  By default /var/log/messages is not set up to get info or debug level messages.  I recommend using the instructions at https://twiki.grid.iu.edu/bin/view/Documentation/Release3/InstallGlexec#10_Glexec_log_files to create a separate /var/log/glexec.log that gets all log levels.  Don't leave it at a high debug level on all worker nodes though because it can generate a lot of messages.  It takes up quite a bit of space and can slow down the system, especially if there's no minus sign ("-") at the beginning of the file name in /etc/syslog.conf because then each message is flushed separately to the log.
