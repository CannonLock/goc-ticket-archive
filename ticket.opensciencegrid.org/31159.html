    Hello,

all AGLT2 services are now switched to srm. We investigated the deletion with https last weak. The reason for these errors is, that in case of non-existing file, dCache retruns permission error, which is not correct. In such case, the deletion is re-tried again and again. This seems to be dCache problem related to https protocol. Please try to switch to newest version of dCache so that we can proceed deletion with https asap.
