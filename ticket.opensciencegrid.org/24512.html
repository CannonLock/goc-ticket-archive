    Mar 19, 2015 01:54 AM UTC by OSG-GOC 
    There is something odd happening here.

I have looked at several of the files and in each case:

The file is related to the SUSY group.
We have a copy of the file in PRODDISK but not in GROUPDISK.
The file was produced at SWT2_CPB, recently (March 7th).
The file was NOT deleted from the GROUPDISK area.
After production there have been no attempts to access the file in
PRODDISK.
For some reason the file is considered to be in GROUPDISK.

Either RUCIO thinks that the file got moved or the production system
broke down
and registered the wrong location for the job outputs.  We are
favoring a
problem with rucio.

When tokens first started this would happen because DQ2 would see the
replicas
in PRODDISK and accept them for GROUPDISK.  This would actually work
because the
registrations included the PFN.  Now with Rucion, it seems that the
files should
have been copied from PRODDISK to GROUPDISK but were not.

I believe an example dataset is:
mc12_8TeV.233629.MadGraphPythia_AUET2B_CTEQ6L1_pMSSM_QCD_9258867.merge.NTUP_SUSY.e3553_a220_a263_a264_r4540_p1328_tid04993194_00
