    Mar 25, 2008 01:13 PM UTC by Tim Silvers 
    Globus requires a proxy to be valid for more than 3 (or is it 6?) hours for jobs to run is what Arvind
has discovered in his testing with RSV probes. Since I was off yesterday I'd planned to renew the
proxy for the GridScan daemon this morning. Even though it was still good until almost 10AM
Globus wants a proxy to be valid for at least three hours or it will not run jobs, go figure. I haven't
found this documented anywhere yet, but know I know that I shouldn't let the proxy get that close
to expiring. I might look at having the script automatically renew the proxy or send email if it gets
this close to expiration.

Scans are passing now and VORS will catch up in the next few hours as scans are run and the VORS
interface refreshes. I will monitor VORS and the scans just to be sure, but I'm confident this issue
is resolved.

Tim
