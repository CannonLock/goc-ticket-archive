    Hi all,

Sorry, thought I had already replied to this one.

Currently, adding a FQAN to a 'banned' user group disables the account.
All the under-the-hood checking is already implemented.

The name of the banned group needs to be defined at the global level of
the gums.config, e.g.:

<gums version='1.3' allowGridmapFiles='true' bannedUserGroups='banned'>
<persistenceFactories>
.
.
.

I'm not sure if VDT is adding this element to the template by default or
not. But doing that, and creating the manual group as part of the setup
would just leave adding users to site admins.

And we could fairly easily allow definition of the banned group via the
web interface.

Further functionality of updating a banned group from a URI is partially
implemented, and was the subject of lots of discussion with Mine and
Alain some time last year (I think).

Cheers,

--john
