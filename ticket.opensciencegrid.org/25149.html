    Hi,

I am not sure DPM is really affected, but I have new CMS users who can't access my DPM, and they are not in the lcgdm-mapfile.
They are not in that file, because it has not been updated since april 1st.
And it's not been updated because dzero breaks the tool with its voms server, which contains an empty value.

The error I get is:
Can't call method "getData" on an undefined value at /usr/libexec/edg-mkgridmap/edg-mkgridmap.pl line 685.

I added debug to the tool, and narrowed this down to dzero.
Off course, the tool is doing a wrong asumption that voms only contains non empty entries, but if you look here :
https://voms.fnal.gov:8443/voms/dzero/services/VOMSCompatibility?method=getGridmapUsers&container=%2Fdzero

You'll notice this :

<getGridmapUsersReturn xsi:type="soapenc:string">/DC=gov/DC=fnal/O=Fermilab/OU=Robots/CN=fifegrid/CN=batch/CN=Reinhild Y. Peters/CN=UID:peters</getGridmapUsersReturn>

<getGridmapUsersReturn xsi:type="soapenc:string"/>

That last entry is empty and is causing the mkgridmap breakage.
If I edit the mkgridmap tool and change this :

#$subject = $sn->getFirstChild->getData;
#push(@Subject, $subject);
my $subjChild=$sn->getFirstChild;
if($subjChild) {
$subject = $subjChild->getData;
push(@Subject, $subject);
}

Then the mkgridmap tool works.
So, as a first and quicker work around, can dzero clean up its voms ?
Then the tool can be fixed too, with the time it requires to cleanly fix it and deploy the change...
