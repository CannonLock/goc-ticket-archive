    Mar 21, 2013 08:41 PM UTC by Matyas Selmeci 
    There looks to be some confusion here. osg-ca-certs-updater is _not_ actually an alternative to using one of the ca-certs RPMs. In fact, it requires one of them to be installed since what osg-ca-certs-updater does is to periodically update one of the ca-certs RPMs.
The ca-certs RPMs are: osg-ca-certs, osg-ca-certs-compat, igtf-ca-certs, igtf-ca-certs-compat.

*osg-ca-scripts* _is_ an alternative to using one of the RPMs, and in fact cannot be installed at the same time as one of the ca-certs RPMs. According to the osg-profile.txt, that is the package is currently installed.

The osg-ca-scripts package provides a script called "osg-ca-manage" to set up your CAs one time, and a service called "osg-update-certs-cron" to periodically update these certs. That's probably the cron job you'll want to enable.

For more information, please see the https://twiki.grid.iu.edu/bin/view/Documentation/Release3/InstallCertAuth
The osg-ca-scripts package corresponds to "Option 2" on that page, whereas using one of the ca-certs RPMs along with osg-ca-certs-updater corresponds to "Option 1" on that page.

It's unfortunate that these two different ways of handling CA certs are named so similarly.

Hope this helps.

-Mat
