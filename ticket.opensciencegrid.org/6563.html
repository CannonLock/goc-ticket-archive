    Hey,

We ask to make a best-effort.  For example, the MIT site has 11
different hardware clusters - too much to list separately - and I
believe they group them into 2-3 generic ones.  Try to identify the
biggest subgroups; it doesn't have to be exactly right.

The important things to have approximately right are:
1) The actual number of CPUs (we estimate site size to be around (# of
CPUs) * (calculated CPU power)
2) Realistic CPU model names so we can guess the computing power.

I realize that Condor pools can be very dynamic, meaning the actual
number of CPUs can fluctuate.  Try to take it to within 10% if you can.

    Hey,

The GIP info from NYSGRID_RIT:

http://is.grid.iu.edu/data/cemon_processed_osg/grid.rc.rit.edu.processed?which=NYSGRID_RIT

shows that they have configured their CPU model as "Opteron" and the clock speed as "2300000000".

The CPU model should be taken from the output of /proc/cpuinfo (we keep histories of CPU models) and not just the generic "Opteron".  An example from another site would be "Intel(R) Xeon(TM) CPU E5310 @ 1.60GHz".

The clock speed should be in MHz, not Hz.  Thus, the right number is "2300".

This should be fixed in their config.ini and configure-osg.py re-run.

Thanks,

Brian
