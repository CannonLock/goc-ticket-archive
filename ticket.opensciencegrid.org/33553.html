    May 12, 2017 03:01 PM UTC by OSG-GOC 
    Hi,

problems with CE at Purdue are now fixed, so pilots are running again.
I have been therefore able to troubleshoot the original issue reported in the
ticket.
The directory /tmp/ has no permissions to write for unprivileged users. That
was almost clear from the error message, but now I can see it.

$ ls -lh /
total 1.7M
...
...
drwxr-xr-x 7 root root 4.0K May 12 10:25 tmp
...
...

However, it is unclear to me if the site expects or not jobs to be able to
write under /tmp/
They have in the job environment these contradictory variables:

OSG_WN_TMP=/tmp
TMP=/var/condor/execute/dir_3728274
TMPDIR=/var/condor/execute/dir_3728274

If site does not want to grant access to /tmp/, I think maybe $OSG_WN_TMP
should also point to /var/condor/execute/dir_XXXXXXX

On the other hand, I think the creation of a subdirectory underneath /tmp/ is
still hardcoded in rucio.
So rucio tries to create /tmp/usatlas1/ unconditionally, irrespective the
content of any environment variable.

That's everything I have for now.

Jose
