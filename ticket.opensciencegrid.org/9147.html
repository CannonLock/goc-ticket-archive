    Hi,

OSG Storage group is working with LIGO Binary Inspiral team in order to help to access and use opportunistic storage advertised by the sites. According to BDII information UMISSHEP SE (umiss005.hep.olemiss.edu ) supports LIGO VO. All the standard srm tests are working fine but it looks like the resource failed to advertise mount point information correctly. According to BDII SE has a file system mounted on the worker nodes but it looks BDII MountInfo information is icorrect.
GlueCESEBindMountInfo: /misc/storage005,/storage005

mount_info that is defined in config.ini under SE section allows to map path accessible through your SE (e.g /osgremote/osg_data/ligo...) to the specific partition on the worker node, so if you have on your worker node mounted file system  /storage005 and you can access the data in SE from the worker node using
/storage005/osgremote/osg_data/... path

your mount point should have
mount_point = /storage005,/

Could you please correct config.ini file? You don't need to do restart any services. The right information will propagate to BDII in about ~10 minutes.

Please let me know if you have any questions.

Thanks a lot,
Tanya
