    Jun 3, 2014 04:04 PM UTC by Marco Mambelli 
    Majid,
if there is no directory Condor is not writing the per job files. It should have been complaining with an error.

To send the missing data you have to run the probe against the Condor history file. Use something like:
/usr/share/gratia/condor/condor_meter --history --start-time="2014-05-21" --end-time="2014-05-30" --verbose

Set start-end time to cover all the outage or more. Sending duplicates is not a problem, Gratia will discard them.
If you have a non standard Condor installation make sure that the environment is set correctly and the condor_history command works returning the right jobs.

Here the instructions form the OSG documentation:
https://twiki.grid.iu.edu/bin/view/Documentation/Release3/TroubleshootingGratia#6_6_Reporting_old_Condor_data

Cheers,
Marco
