    Vince and Brian are right. Both certificates are valid and they don't have affect each other at all. In fact, both certificates possess the same DN (they were produced in different endpoints).

Now, did the private key from the first request (in production) get overwritten for the private key from the second request? If so, you would need to keep the "correct" certificate.

You should check if the hostkey.pem corresponds with the certificate. You can use this openssl commands:

$openssl x509 -noout -modulus -in [host_certificate.pem]
$openssl rsa -noout -modulus -in [hostkey.pem]

Both modulus output should match.
