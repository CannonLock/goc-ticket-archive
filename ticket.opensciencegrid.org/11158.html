The CA packages are *not* same as s/w packages, we just change the
content (i.e. the CA files included). So there is no need to go through
ITB for these and the packages get released directly to production and
it has been so for a long time and we have discussed this with GOC. I
don't see any real reason for changing it. In fact there might be need
for us to immediately release a package to production due to urgent
security issue and requiring to go through ITB could be problematic.

Also as I have mentioned multiple times in the past the packages for
production and itb are different because metadata files, so even if we
release to ITB we have to make a separate release for production anyway.

Anand

On 10/20/2011 07:34 AM, Horst Severini wrote:
Hi Alain,
Anand should probably comment, so I've added him to the list.

You can always switch to the non-ITB CA certs.
Okay, I just changed to the OSG certs with
----
vdt-ca-manage setCAURL --url osg-new
vdt-ca-manage refreshCA
vdt-ca-manage fetchCRL
----
And that seems to have worked:
----
