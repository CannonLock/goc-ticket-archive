    Jun 10, 2008 11:59 PM UTC by grog@.... 
    I believe this was an issue with ownership that was probably induced
by updated server certificate a couple of months ago.  The
certificate and key used by CEmon were linked to the root-owned /etc/
grid-security/host*.pem files instead of the daemon-owned
container*.pem files.  The relevant entry for the cert/key in
cemonitor-config.xml reads:

<service id="main-service"
description="cemonitor service"
sslkeyfile="/etc/grid-security/http/httpkey.pem"
sslcertfile="/etc/grid-security/http/httpcert.pem"
sslkeypasswd=""
sslCAfiles="/opt/osg-ce-0.8.0-r1/globus/TRUSTED_CA/*.0"
>

These are the paths to the links to the respective container*.pem files.

Suchandra said that UC_Teraport was actually showing up through CEmon
[http://is.grid.iu.edu/cgi-bin/show_source_data?
which=UC_Teraport&source=cemon], but did notice there was an error in
the GIP Validator [http://gip-validate.grid.iu.edu/production/
UC_Teraport/index.html].

There were permission errors in the glite-ce-monitor.log:

10 Jun 2008 17:34:19,239
org.glite.security.trustmanager.ContextWrapper - ContextWrapper
initialization failed: Identity reading failed: /etc/grid-security/
http/httpkey.pem (Permission denied)

But these went away after relinking to the container*.pem files.

Until today, the catalina.log and archives were empty, but is now
being written to.  There was one WARN entry since the certificate/key
correction:

    
Jun 10, 2008 6:40:53 PM org.apache.catalina.core.StandardHostDeployer
install
INFO: Processing Context configuration file URL file:/opt/osg-
ce-0.8.0-r1/tomcat/v5/conf/Catalina/localhost/ce-monitor.xml
log4j:WARN No appenders could be found for logger
(org.apache.catalina.startup.TldConfig).
log4j:WARN Please initialize the log4j system properly.

Attached are the files you requested.  Let me know if the cert/key
correction had any effect.
