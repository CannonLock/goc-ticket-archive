We have set up a set of validations as a part of the Gratia/WLCG Interface that allow us to proactively identify (as much as possible) any conditions that  might cause problems with the accounting data being mis-reported for Tier1 and 2 sites with MOU agreements.

In MyOSG/OIM, each of the USCMS-FNAL-WC1 resources (USCMS-FNAL-WC1-CE1/SE) are showing a WLCG Information Accounting Name of US-FNAL-CMS  as seen here:

https://myosg.grid.iu.edu/rgsummary/index?datasource=summary&summary_attrs_showwlcg=on&summary_attrs_showservice=on&gip_status_attrs_showtestresults=on&downtime_attrs_showpast=&account_type=cumulative_hours&ce_account_type=gip_vo&se_account_type=vo_transfer_volume&bdiitree_type=total_jobs&bdii_object=service&bdii_server=is-osg&start_type=7daysago&start_date=09%2F21%2F2011&end_type=now&end_date=09%2F21%2F2011&rg=on&rg_210=on&gridtype=on&gridtype_1=on&service_1=on&service_5=on&service_2=on&service_3=on&service_central_value=0&service_hidden_value=0&active=on&active_value=1&disable_value=1

The EGI REBUS Topology shows USCMS-FNAL-WC1-CE as the Accounting Name:
http://wlcg-rebus.cern.ch/apps/topology/
This is what WLCG/EGI reporting will only be using in the future as we understand it.
It should cause no problems in the current environment.

Can this be changed  in OIM to reflect the correct name (USCMS-FNAL-WC1-CE) for each of these resources?
