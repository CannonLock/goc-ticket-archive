Hello,

Yes, that is quite likely.  They'll need to have port 2119 open via inetd
to get access to the globus gatekeeper.  Also, there is a range of port
for globus that must be opened for globus to work generally.  These are
defined using the environment variable GLOBUS_TCP_PORT_RANGE.  Sometimes
this is found in /etc/xinetd.d/globus-gatekeeper, sometimes in
$VDT_LOCATION/setup.sh.  But that port range will need to be opened too.

Please let me know if you have any more questions.
