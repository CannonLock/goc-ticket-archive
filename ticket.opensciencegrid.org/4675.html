On Wed, 2008-02-20 at 14:49, Neha Sharma wrote:

Hi Xin

Here they are. These are basically what Ted sent earlier.

- Download the latest vdt tarball
- Untar it
- Do a clean shutdown - Stop all currently running services,
including
Postgres and PNFS
- Restart postgres on all relevant nodes

- Restart PNFS on PNFS node

- Run the config-node.pl and install.sh (don't need to run
config-node.pl if you want to use an existing site-info.def. In
this
case, copy old site-info.def to new location)
- Copy the old (I assume working) PoolManager.conf file. If
needed,
edit it. Check settings in dCacheSetup file.
- Start dcache-core, dcache-pool on relevant nodes

Here is sample upgrade procedure (assuming its a 1 node install):

o cd /usr/local/
o wget
http://vdt.cs.wisc.edu/software/dcache/server/dcachetest.tar.gz
o tar -zxvf dcachetest.tar.gz
o /opt/d-cache/bin/dcache-pool stop
o /opt/d-cache/bin/dcache-core stop
o /opt/pnfs/bin/pnfs stop
o /etc/init.d/postgresql stop

o make sure no dcache related processes is running

o /etc/init.d/postgresql start
o /opt/pnfs/bin/pnfs start

o cd dcachetest/install
o cp ../../dcacheold/install/site-info.def .
o ./install.sh

o check that authorization mechanism is configured correctly - it
will
be combination of /opt/d-cache/etc/dcache.kpwd,
/etc/grid-security/storage-authzdb, /opt/d-cache/etc/dcachesrm-
gplazma.policy. (since it is an upgrade, this should be correct
already)

o Also make sure relevant directories exist under PNFS (since it
is an
upgrade, these should already exist)

o check /opt/d-cache/config/dCacheSetup to make sure
"srmImplicitSpaceManagerEnabled=no" and
"srmSpaceManagerEnabled=yes"
(these are default values, but no harm in double checking)

o /opt/d-cache/bin/dcache-core start
o /opt/d-cache/bin/dcache-pool start

o try couple of test transfers

This should do it. Let me know if you have any questions/problem

-Neha
