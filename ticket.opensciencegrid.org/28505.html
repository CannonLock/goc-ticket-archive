    Feb 16, 2016 02:52 PM UTC by Kyle Gross 
    Jeny & Richard,

I understand that OSG was also transitioned.  I only mentioned it because I wanted people to be looking in the right place.  When I looked for a gluex certificate, there wasn't one to be found.  It was only when I looked under the OSG VO that I found Richard's certificate.

>>> How can one tell what vo a user cert was registered to, besides looking it up in the voms?

With regards to how I know, I know because I am an admin for the OSG CA web interface.  I see that  I can also look on your certificate ticket (https://ticket.grid.iu.edu/12982) which tells us that this certificate was originally requested in the OSG VO.  Requesting a certificate has nothing to do with the certificate being listed in a VOMS server.

>>> In my understanding, my user cert is osg-wide and when I get approved as a member of a vo, my user cert gets added to the voms db for that vo.

That is correct, but that is a separate issue from issuing the certificate.

>>> If that is correct, a user cert does not have a vo-centric definition when it is issued, does it?

A user always requests a certificate within the context of a VO so that we can make sure that trusted individuals that we know also know the user.  This is how we approve certificates for various users that we don't know.

Hopefully that clears things up.
