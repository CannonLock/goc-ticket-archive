    Hi Derek, all,
there is an initial (=in the XML generated by the probe) value of VOName that is the short nome and I don't know how is generated. Normally Condor has the FQAN and other probes should have nothing.

The current logic is: if there is an initial value do not override it
This is why at the end the short name is kept even if all the job is done to look the content of the certinfo file

I changed it in something like: if there is an initial value and it is a short name then if a FQAN is found in a certinfo file or querying htcondor-ce, then replace the initial name with the FQAN.
This is a better behavior and will fix the problem at the UNL sites.

I attached a modified xml_utils.py (/usr/lib/python2.6/site-packages/gratia/common/xml_utils.py) with the new logic.
I tested it locally but have only CEs w/ condor
Could you try it?
