    Jan 20, 2016 11:12 PM UTC by Jeny Teheran 
    Hello,

According to the OSG CA transition schedule, Gluex and Star VOs are going to be transitioned on February 9th, 2016. These VOs are in charge of their VOMS servers and an update of the VOMS servers DNs is required. A new vo-client package version -v63- needs to be included in the next OSG Software release 3.2.35 / 3.3.9.

- Gluex VOMS server new DN:
"/DC=org/DC=opensciencegrid/O=Open Science Grid/OU=Services/CN=voms/gryphn.phys.uconn.edu"

- Star VOMS server new DN:
"/DC=org/DC=opensciencegrid/O=Open Science Grid/OU=Services/CN=vo.racf.bnl.gov"

For the .lsc files "/vomsdir/Gluex/gryphn.phys.uconn.edu.lsc" and "/vomsdir/star/vo.racf.bnl.gov.lsc" in the vomsdir, the issuer needs to be updated:
"/DC=org/DC=cilogon/C=US/O=CILogon/CN=CILogon OSG CA 1"

Please, let me know if more information is required.
