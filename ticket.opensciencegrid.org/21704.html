    I know I am not supposed to get involved but this is kind of an anomaly.

The latest June data should be up in APEL now.  Whenever the EGI cron runs to summarize it, it will be available in the EGI portal and you can grab it Cath.

To explain (as this has never happened timing-wise in forever)... The interface is designed to send a file of "deletions" (zeroes out all data from the previous run) as explained here in item 1 here...
https://twiki.grid.iu.edu/bin/view/Accounting/ApelWlcgDevelopersCorner#LCG_py

What happened this weekend is that we had failures on the upload to APEL.
They were timeouts on the APEL side.  This is normally no big deal.  It happens
periodically.  It is not really critical to immediately re-run the interface
as the next day's run will refresh it (we send monthly summaries up).

HOWEVER, it had to happen on the 5th day of the month.  We only run the interface for the first 5 days to accomodate sites that might be doing some "catchup".  The interface evidently sent the deletes for June up OK.  Then 5 minutes later, the SSM interface hiccup'd (timed out) and the updates never made it.

I did not notice this.  I re-ran an hour ago and all should be available.

Sorry for the confusion.
John Weigand
