Hi,

In my recent osg3 installation I ran across an issue with configuring RSV, notably running the default gridftp tests.

The /etc/osg/config.d/30-rsv.ini file states:
; The gridftp_hosts options lists the FQDN of the gridftp servers that the RSV
; GridFTP metrics should monitor.  This is be a list of FQDNs separated by a
; comma, for example:
; gridftp_hosts = gridftp.example.com, gridftp2.example.com, gridftp3.example.com
;
; If this option is commented out then the value of ce_hosts will be used.
; If you want to disable the GridFTP probes then uncomment this entry and
; set it to DEFAULT
gridftp_hosts = gk01.atlas-swt2.org

I have seen that if gridftp_hosts entry is commented out, no gridftp tests are performed.  I explicitly set gridftp_hosts to the CE name.

If I uncomment the entry and set it to DEFAULT, it will not run the gridftp tests (as stated).

The installed rsv packages are:
rsv-3.7.4-1.osg.el5.noarch
rsv-core-3.7.4-1.osg.el5.noarch
rsv-metrics-3.7.4-1.osg.el5.noarch
rsv-consumers-3.7.4-1.osg.el5.noarch
osg-configure-rsv-1.0.14-1.osg.el5.noarch
