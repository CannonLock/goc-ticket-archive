Steve is right; don't move pacman-based installations.  Soft-links
and locations are resolved during the installation steps and coded
into various pieces of the installations, making it very hard to move
them.

If you really want to transfer from one location to another, one way
to do so is to make the new directory and use the OLD_VDT_LOCATION
variable to point to the former location while you reinstall the same
packages into the new location.  In more detail, to do this, set up
your current installation, do a vdt-control -- off to shut it down,
cd to its base directory and do a pacman -history to see what you had
put in and in what order.  Then log out and back in to get a fresh
environment, set the OLD_VDT_LOCATION to point to the area that
contains your current installation, set up pacman, cd to the
directory for your new installation and do the pacman -get steps for
whatever you had installed before.

It's basically the same as a new installation, but you are not
changing versions.  You will get any lettered updates automatically,
so there is some benefit for doing this if you need to move and
update at the same time.

If nothing changed in the old installation, you may be able to move
it back to its original location and follow the steps outlined above
to recover it.

Hope this helps.

Alan

On Fri, 2007-11-16 at 21:20 +0000, goc@.... wrote:
Support Centers,

There is a new VO package which includes the updated entries in the vomses file for the CERN
VOMS changes. It also includes a new format for the gums.template that will allow both versions
1.1 and 1.2 of GUMS to function properly.

All OSG resources are encouraged to upgrade.

For resources that have updated to 0.8.0 please use:

pacman -update OSG:vo-package

For resources at OSG 0.6.0 please use:

pacman -update OSG:vo-0.6

For resources still using 0.4.x we are no longer producing packages for 0.4 installation please
update your OSG middleware to the current 0.8.0 stack.

For OSG Client users please use (Note: This not for wn-client and works for both 0.6 and 0.8
releases.):

pacman -update OSG:vo-client-0.6

    
Please contact the GOC with any questions.
