    Scott,

I agree.

One possible clarification - OSG distinguishes between RAs for approving user certificates in a VO and Grid Admins for approving host/service certificate in a DNS domain (e.g. uwm.edu).

I think you might just need OSG host certificates for the cluster, but frankly if someone goes the the enrollment process, they might as well become an RA as well to cover any future contingencies.

The process to become either is documented at the following URL. Basically one reviews the training and then makes a request that will be approved by a LIGO VO Managers in OIM (Kent or Brian Moe at this time).

https://twiki.grid.iu.edu/bin/view/Operations/OSGPKITrustedAgent

Let me know if you have any questions about the process,

Von

    Hi Von,

If LIGO can register an RA that would be helpful. I do recommend it be
Brian Moe.

At this time LIGO continues to use certificates issued by CILogon for
our users to access the LIGO Data Grid clusters. We issue host and
service certificates for those cluster machines from the LIGO CA that
we operate.

To continue to have the UWM cluster be available as an OSG resource,
however, we need the DigiCert certificates. That is the only use case
right now. Note that the OSG interfaces for our cluster are dedicated
nodes--they are only used for OSG and not for LIGO access.

Another path would be for OSG to recognize the LIGO CA(s). We can talk
about that for the future, but right now the fastest path is for Brian
to become an RA in order to get DigiCert certificates for the sole
purpose of providing access for OSG to the UWM cluster.

Thanks,

Scott
