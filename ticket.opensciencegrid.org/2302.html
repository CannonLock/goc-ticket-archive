I would like to figure out how to move some some mailing lists to opensciencegrid.org.

There are currently two VDT mailing lists I want to move. They are slightly different
from each other:

1) vdt-discuss@....

This is a straightforward mailing list. I would like to take the membership of that
mailing list and give it to you for a vdt-discuss@.....

Right now the list requires me to approve anyone that joins the list. I don't need
that to continue, but if possible I would like to require people to be members before
they can send to it, just to cut down on the spam.

2) vdt-support@....

This is mostly a straightforward mailing list. It can be thought of as a forwarding
mail address that forwards to vdt-support@..... Our ticketing system uses
vdt-support@.... as it's return address, so all mail to the ticketing system
passes through this mailing list.

However, a few people are also subscribed to this mailing list. It's a bit weird that
it serves this dual purpose, but I think we'll keep it that way.

We would like to move this to vdt-support@...., including the entire
membership.

I would like to be careful about moving these lists, because these are well-known

    addresses. I would prefer something like this:

1) Advertise that on some date (September 1, 2006?) the membership of these mailing
lists will be moved. If people don't want to be moved, they can unsubscribe.

2) Move the mailing lists on September 1, 2006. Keep the old mailing lists (that's my
responsibility).

3) Kill the old mailing lists later, maybe December 2006. Again, my responsibility.

Does this sound reasonable? Is it something we can do?

Thanks!
-alain
