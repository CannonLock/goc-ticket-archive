    Oct 15, 2014 08:49 PM UTC 
    Dave Lesny  pointed  out the fix that  is  needed.

"This is what I use in Connect in the job setup for DQ2 to fix the "multi" user issue

# DQ2 uses $USER to create a unique /var/tmp/.dq2$USER/TiersOfAtlas.py
[[ -z $USER ]] && export USER=$(whoami)

# Setup DQ2 and Rucio Client
source /cvmfs/atlas.cern.ch/repo/sw/ddm/latest/setup.sh --quiet"

Bob Ball  implemented  this  fix  this  morning  for  AGLT2  and  the  MCORE  failure  rate  has dropped significantly since  then.

We will  watch  for  a  bit  more  but  suspect  this  should  have  fixed  the problem.
