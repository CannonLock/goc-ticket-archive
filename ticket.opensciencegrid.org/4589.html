Sebastien,

I believe the edg-mkgridmap service is not enabled.
%vdt-control --list
will tell you if the service is enabled.

If not, you can enable the service using vdt-register-service.
%vdt-register-service --name edg-mkgridmap --enable
Then you can turn on the service using
%vdt-control edg-mkgridmap --on

https://twiki.grid.iu.edu/twiki/bin/view/ReleaseDocumentation/ComputeElementInstall#Starting_services
should give you more information about these commands.

Anand

>>> Since you are not using GUMS, check the following:
>>> 1. Are all the VO's you want to support listed in
>>> $VDT_LOCATION/edg/etc/edg-mkgridmap.conf
>>> 2. Have you created the local mapping accounts for the other VO's.
>>> $VDT_LOCATION/monitoring/osg-undefined-accounts.txt, should give you a

>>> accounts and run
