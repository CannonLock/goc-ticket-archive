Hay all,

We use Will's script to write protect full dcache pools. This relies "info" command
of the pool cells in the dcache admin interface reporting correct disckspace
usages. However today, I noticed several of our pools went offline because Diskspace
usage reported by dcache admin interface is under 90% while the actual diskspace
usage(du) is already above 99%. I've ran a script to sum up the sizes of all the files
in the pool/data directory, I saw the sizes is 9G more than the used space reported
by dcache admin interface.

Looks the info command of the pool cells in the dcache admin interface is
reporting incorrectly. Has anyone seen this before?
