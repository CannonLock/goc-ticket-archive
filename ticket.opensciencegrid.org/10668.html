> Error:  Could not fetch ads --- can't find collector

This is not the problem. There are two comments about condor_status:

1) Condor Cron does not run a Condor collector because it's not needed.
If you are running Condor separately from Condor cron, then it might be
a problem.

2) All of the command for Condor cron have been renamed. For instance,
you would use condor_cron_q instead of condor_q. This lets us run a
separate instance of Condor for Condor cron that doesn't interfere with
a pre-existing installation.

> detailsData: FAILED Attempt to execute remote job:
>  [/sharesoft/osg/ce-1.2/globus/bin/globus-job-submit
> hepcms-0.umd.edu/jobmanager   /bin/sleep 300 2>&1  ]
> ERROR: GRAM Job submission failed because the job manager failed to
> open stdout (error code 73)

An aged FAQ from one of the Globus staff has a pretty good explanation
of the possible errors:

http://www.mcs.anl.gov/~bacon/guide-faq.html#stdout

Does that help?
