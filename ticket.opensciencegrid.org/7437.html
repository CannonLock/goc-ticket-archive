    Hi,

I've been creating some summaries recently of resources by OS type.

There are some guidelines here for publication of OS types:

http://goc.grid.sinica.edu.tw/gocwiki/How_to_publish_the_OS_name

which were previously approved by OSG via the inter-operations group.

Now
http://cern.ch/straylen/summary-oses.txt
is a report from these publications that is generated once per day.

The following sites look to be invalid and I comment on each of them:

1) LIGO_UWM_NEMO and FLTECH are publishing  (CentOS, 5)
where as it should be (CentOS, 5.n)

2) RENCI-Engagement is publishing (Linux, 2.6.18-6-686)
and I expect it should be something else but see (*).

3) Vanderbilt  is publishing (Red Hat Enterprise Linux, 5) and it should
probably be (RedHatEnterpriseAS,5) (See **)

4) Purdue-Steele and NWICG_NDCCL are publishing (Red Hat Enterprise Linux,5.3)
and it should probably be (RedHatEnterpriseAS,5)

5) BNL-ATLAS is publishing (Red Hat Enterprise Linux,4) and it should
probably be (RedHatEnterpriseAS,4)

6) rutgers-cms is publishing (Scientific Linux CERN SLC,4.7) and it
should probably be (ScientificCERNSLC, 4.7)

7) UCD is publishing ( Scientific Linux CERN SLC, 5.3) and it should probably

    be (ScientificCERNSLC, 5.3)

8) UCR-HEP, osu-cms and umd-cms	are publishing (Scientific Linux SL, 4.5)
and it should probably be (ScientificSL, 4.5)

9) USCMS-FNAL-WC1 is publishing (Scientific Linux SL, 5.2) and it should
probably be (ScientificSL, 5.2)

10) AGLT2 is publishing (Scientific Linux SL,5.3) and it should probably
be (ScientificSL, 5.3)

11) FNAL_FERMIGRID and FNAL_GPGRID_1 are publishing (Scientific Linux SLF, 5.3)
this may well be correct. See (*)

12) GLOW is publishing (linux-sl-fermi-4.4, Linux) and I expect incorrect

(*) For any OS it may well be that new values have to be added to the
table. Instructions exist how to collect the information. You can
let me know if something is missing and I will add it the table
and subsequent monitoring.

(**) Given the list is built up statically for new OSes I am checking with
someone with access to RHEL 5 box exactly what the values are.

Many Thanks.
Not massively urgent or anything just want to go in the right direction.
Steve
