Subject: Question about latest vo-client release: inconsistent hostname

Hi GOC,

I wonder if there is a bug in version 40 of the vo-client. I'm not sure, mind you, but I see something inconsistent.

This release added alice. In the edg-mkgridmap.conf, I see:

# USER-VO-MAP alice ALICE -- 37 -- Latchezar Betev (latchezar.betev@....)
group vomss://voms.cern.ch:8443/voms/alice alice

Note the hostname: voms.cern.ch.

But in the vomses file, I see:

"alice" "voms114.cern.ch" "15000" "/DC=ch/DC=cern/OU=computers/CN=voms.cern.ch" "alice"

Note the different hostname, which doesn't match the DN. Moreover:

% host voms114.cern.ch
Host voms114.cern.ch not found: 3(NXDOMAIN)

My hunch is that the hostname in the vomses file should be voms.cern.ch, not voms114. Even more convincing, <https://voms.cern.ch:8443/voms/alice/configuration/configuration.action>; reports:

VOMSES string for this vo:
"alice" "voms.cern.ch" "15000" "/DC=ch/DC=cern/OU=computers/CN=voms.cern.ch" "alice"

Perhaps the vo client should be updated?

-alain
