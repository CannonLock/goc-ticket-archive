    Feb 19, 2017 02:04 AM UTC by OSG-GOC 
    Hi Feng,

These are errors that aren't doing any harm, but it might be something for DDM
to pay attention to. What happens is:

1) Files are created with >231 characters in their name. Notice that the name
below has repeats in it and looks like it might be a bug.
2) When the file is deleted with our SRM, it reports the error below.
3) The "requested service not available" is misleading. The SRM is up the whole
time. Also, the files are actually successfully deleted, in spite of the
message.
4) We've gotten similar bunches of errors in the past few days.

Summary:

a) These errors do no harm and will stop when the extra long named files are
all deleted.
b) It suggests that the rucio team might want to prevent the creation of files
with file names greater than 231 characters. Note that files with greater than
256 characters will break many file systems anyway.

Regards,

- Saul
