> When something like this comes up should we be checking with you
> or should this type of error be reported through GGUS?

Probably with both since the issue can be either a malformed
signing_policy file (for which the IGTF would be responsible), or a
bug in parsing it (which would mean the GGUS route).

In this case, there is indeed a fault in the EACL file -- this
means a new distribution is needed to consistently fix this.
I apologize to all sites, as I realize this will means more work
for everybody. And it confirms that both the originating CA as well
as the registrar (or at least two people) have to review the input.

    That process was not followed during this insertion, and unfortunately
this triggers visible errors instead of just a non-functional CA.
In addition, this error was also not caught by the EGEE certification
process where it happily passed ...

Anyway, the 1.34 releases fixes this, and is now available from the
EUGridPMA distribution web site.

DavidG.
