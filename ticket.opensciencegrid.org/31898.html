    Hi,

At the RACF at BNL, we now control and administer the domain `sdcc.bnl.gov`. As the network in which administrative hosts is isolated, we also need dual-interface nodes to bridge networks; one of the interfaces on these nodes will reside on the domain `sdcc.bnl.local`. Testing host certificate generation for these domains has shown that our automated request and approval process (via OSG PKI tools) fails if the host or alt name is in the `sdcc.bnl.local` domain, as well as if the host name and requested alt name are equal and in the  `sdcc.bnl.gov` domain.  We assume that we need to "register" the domains `sdcc.bnl.gov` and `sdcc.bnl.local` with OSG for certificate creation -- if this is correct, how do we proceed?

Example requests include:
https://oim.grid.iu.edu/oim/certificatehost?id=8937
https://oim.grid.iu.edu/oim/certificatehost?id=8938

Strangely enough, request '8937' says the request was approved and the certificate was "issuing", but then returned an error from the CA:
Failed to sign certificate -- CertificateProviderException :: From CILogon: Unknown status code from cilogon: 500

As we need to roll out infrastructure for these new domains quickly, and several hosts will require certificates, we'd like to clarify and address this issue as soon as possible.

Thanks,
