This is occurring in OSG 0.4.1./VDT 1.3.10.

The gt4prima authorization process is not working when a delegated
proxy is
used. It is picking up an appended value for the DN in the call to the
GUMS server and is getting 'denied'.

For example, the normal proxy DN used is:
/DC=org/DC=doegrids/OU=People/CN=John Weigand 458491

When a delegated proxy is used, the DN used is:
if voms proxy:
/DC=org/DC=doegrids/OU=People/CN=John Weigand 458491/CN=proxy
if grid proxy:
/DC=org/DC=doegrids/OU=People/CN=John Weigand 458491/CN=782526324

The delegated proxy appears to be used in the ReliableFileTransfer
service
of globus-ws.


    The gt4prima code has been fixed and will shortly be committed to CVS
for
distribution.

The purpose of this ticket is to track the distribution of this fix into
the next release or for inclusion as an OSG patch for the 0.4.1 release.
