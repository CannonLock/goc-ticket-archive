    May 15, 2015 04:14 PM UTC by Jaime Frey 
    The “couldn't set globus thread model” error happens when the schedd attempts to configure the Globus libraries to not use pthreads and the associated locking (since HTCondor processes are single threaded). From my reading of the Globus code, the error means that something in the process has already configured the Globus libraries in pthread mode.

That could happen via an explicit call to globus_thread_set_model() or by activating a Globus module when GLOBUS_THREAD_MODEL=pthread is set in the environment. Since the HTCondor code doesn’t make any other calls into the Globus libraries, I suspect some library that’s being dlopen()d is to blame.

Can you do an lsof on a condor_schedd process that has shown this error and look for an entry for a libglobus_thread_pthread library file? That’s the sign that Globus has been initialized in pthread mode.

— Jaime Frey
