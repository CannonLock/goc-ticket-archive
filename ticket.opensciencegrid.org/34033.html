    Jun 6, 2017 07:49 PM UTC by Derek Weitzel 
    Hi Bockjoo,

A discrepancy is normal.  See the image here:

https://twiki.grid.iu.edu/twiki/bin/viewfile/Documentation/Release3/HTCondorCEOverview/ce_otherbatchsystem.png

When the job is first submitted, it will have xcount = X which is set by the factory.  And RequestCpus = 1, which is default for HTCondor.  The job router then translates the job for the local scheduler, changing RequestCpus = xcount.  So I would expect it to look something like this:
...
xcount=8 RequestCpus=1
xcount=8 RequestCpus=8
...

Both of these will actually represent the same job, but the first is the direct submission from the factory, and the second is the transformed job.

-Derek
