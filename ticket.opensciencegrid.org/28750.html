    My suggestion is to fix this service by doing the following:

1) change psmad.grid.iu.edu to be only perfsonar2.grid.iu.edu, removing perfsonar1.grid.iu.edu from the VIP alias
2) change psomd.grid.iu.edu to be only perfsonar1.grid.iu.edu, removing perfsonar2.grid.iu.edu from the VIP alias
3) Run maddash-server only on perfsonar2.grid.iu.edu and disable omd:   '/sbin/service omd stop; chkconfig omd off'
4) Run OMD/check_mk only on perfsonar1.grid.iu.edu and disable maddash-server:  '/sbin/service maddash-server stop; chkconfig maddash-server off'

This will remove the need to have both omd/check_mk and maddash-server running on both hosts and prevent confusion when problems are observed.
