    Sep 19, 2013 06:42 PM UTC by Terrence Martin 
    Hi, Brian and I had a look at this again this morning. Brian poked around and realized that the issue is that given we are not using host certs at UCSD when GUMS goes to map a user it checks the pilot cert from the SSL connection. The problem is the SSL connection only forwards the DN, not the extension, so GUMS dutifully refused to perform that mapping for NoVA which required an extension. The fix we put in is to allow NoVA users to map even if they have not VOMS extension and place that mapping at the end of our maps.

This is similar to how CMS is setup at UCSD, and has been for a very long time where you get an account in CMS if you are in the VO, even without an extension.

I am starting up a node with NoVA preference to monitor, hopefully things will run.

Sorry for the delay, it was an odd issue interaction.
